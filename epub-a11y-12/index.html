1. 引言

1.1 概述

本节为非规范性内容。

本规范，EPUB 可访问性，处理 EPUB 生态系统中的两个关键需求：

  • EPUB 出版物可访问特性的可发现性；以及
  • 可访问 EPUB 出版物的评估和认证。

提供可访问性元数据有助于对 EPUB 出版物的可用性做出知情决定。它允许消费者查看内容的可访问特性，并决定 EPUB 出版物是否适合他们的需求，而无论其是否达到 可访问性认证的门槛。至少，所有符合本规范的 EPUB 出版物 都满足 2. 可发现性中描述的可访问性元数据要求。

虽然一直都可以创建具有高度可访问性的 EPUB 出版物， 但本规范为认证内容可访问设定了正式要求。这些要求 提供了一套清晰的准则，用于据此评估内容并允许质量认证。一个 可访问的 EPUB 出版物是指满足 3. 可访问出版物中描述的 可访问性要求的出版物。

本规范还在 4. 分发中处理分发对内容的可访问性和可发现性 的影响。

本规范不针对 EPUB 的某一个单一版本。它适用于 符合任何版本或配置文件的 EPUB 出版物，包括该标准的未来版本。

理想情况下，这些指南有助于评估任何基于开放 Web 平台 技术构建的数字出版物，尽管确保这种应用超出了本规范的范围。

注

有关形成本规范的决策的更多背景，请参阅 EPUB 可访问性常见 问题。

1.2 支持文档

本节为非规范性内容。

以下支持文档提供资料性指导，有助于理解和 实现本标准：

理解 EPUB 可访问性

本文档假定读者对 EPUB 3 [epub-3] 和 WCAG 2 [wcag2] 都具有较高程度的熟悉度。它期望 标准实现者理解如何将关注 Web 内容的 WCAG 2 应用于 EPUB 出版物，后者更像是一个打包的网站，并作为单一 文档阅读。

理解 EPUB 可访问性 1.2 [epub-a11y-understand] 文档 更详细地说明了 如何在 EPUB 出版物的语境中理解成功标准——它们何时 适用或不适用、在出版物中应查找什么等等。它还解释了本文档引入的其他 EPUB 目标。

EPUB 可访问性技术

本规范对 EPUB 出版物的可访问性要求采用抽象方法，类似于 WCAG [wcag2] 将其 可访问性指南与实现这些指南的 技术分离的方式。这种方法使指南即使在 格式演进时仍能保持稳定。

为促进这种方法，配套的 EPUB 可访问性技术 1.2 [epub-a11y-tech-12] 文档概述了一致性技术。这些技术解释了如何满足 本规范对不同版本 EPUB 的要求。

1.3 国际化

本节为非规范性内容。

本规范旨在解决用户的可访问性需求，而不受其阅读 语言的影响。[wcag2] 中定义的原则和成功标准也是如此。 目标是确保用户无论其偏好的阅读方式如何，都能充分获取出版物的 信息。

与此同时，创作文本的语言和书写约定会影响 满足可访问性要求所需的技术。例如，EPUB 要求支持 Unicode 文本 [epub-3]， 这确保可以使用正确的字符数据（即避免 需要使用文本图像）。虽然这是一个重要特性，但仅靠它本身 通常不足以确保文本在任何给定语言中完全可访问（例如，还可能需要有关 方向性、强调、发音等的额外信息）。

因此，也可能存在满足可访问性要求的特定语言或文化 实践。这些实践是在本规范及其技术中定义， 还是在其他地方定义（例如，在 WCAG 技术或特定语言最佳实践建议中），将取决于 这些问题是 EPUB 特有的，还是广泛影响所有 Web 内容。

1.4 适用于较旧 版本的 EPUB

本节为非规范性内容。

本规范适用于任何 EPUB 出版物，无论其符合哪个版本的 EPUB。

例如，即使 EPUB 2 [opf-201] 是在本规范之前产生的， 因而没有 提及本规范，仍然可以创建符合可访问性和 可发现性要求的 EPUB 2 出版物。不过，理想情况下，EPUB 2 出版物应升级到最新 版本的 EPUB 3，以获得最先进的可访问性特性和技术。

请注意，EPUB 2 不支持本规范中定义的所有元数据表达，因为它 没有与 refines 属性 [epub-3] 等价的机制。如果无法避免需要 refines 属性的元数据表达， 则这些陈述中会存在一定程度的歧义（即， 表达之间的关系可能只能通过其在包文档 元数据中的位置来显现）。

1.5 术语

本规范使用 EPUB 3 中定义的术语 [epub-3]。

它还定义了以下术语：

辅助技术

本规范使用 [wcag2] 中的辅助技术 定义。

在 EPUB 的情况下，辅助技术并不总是独立于 阅读系统的单独应用程序。阅读系统通常 集成独立辅助技术的特性，例如文本转语音 播放。

2. 可发现性

2.1 引言

本节为非规范性内容。

与网页不同，EPUB 出版物通过许多渠道分发 供个人消费——这种模式使 EPUB 成为电子书和其他类型数字出版物的成功格式。 然而，这种模式的一个后果是，关于出版物可访问性的 具体细节必须随出版物一起传递。

例如，一个聚合来自出版商和作者内容的在线书店，不会知道 每个提交内容中的制作质量，除非出版商通过 元数据告知它们。

因此，确保任何相关方都可以发现 EPUB 出版物的可访问特性是 一个主要关注点。EPUB 出版物可以有多组充分的访问模式，具体取决于为 支持以另一种模式阅读而提供的替代内容。例如，如果出版物中的 所有图像都提供了替代文本和描述，则它既会具有默认的文本和视觉充分访问 模式，也会具有纯文本的充分访问模式。

同样，不满足本规范可访问性要求的内容， 并不必然不能满足个别用户的需求。

只有通过提供丰富的元数据，用户才能决定内容是否适合 他们。

2.2 包元数据

所有 EPUB 出版物 MUST 在包文档中包含 [schema-org] 可访问性元数据，以公开其可访问属性，无论这些出版物是否也满足 可访问性或优化 要求。

EPUB 出版物 MUST 包含以下可访问性元数据：

  • accessModeSufficient — 一组或 多组足以在无重大信息损失的情况下消费 内容的访问模式。EPUB 出版物可根据其所包含内容的类型， 为其消费提供多组充分访问模式（即，与 访问模式不同， 该属性会考虑任何并非广泛可访问内容的替代内容，例如为 音频内容包含转录稿）。

  • accessibilityFeature — 有助于内容整体 可访问性的特性和 适配（例如，替代文本、扩展描述、字幕）。

  • accessibilityHazard — 内容呈现的任何潜在 危害（例如， 闪烁、运动模拟、声音）。

EPUB 出版物 SHOULD 包含以下 [schema-org] 可访问性元数据：

  • accessMode — 处理或感知 内容所必需的人类感官感知系统或认知能力 （例如，文本、视觉、听觉、触觉）。

  • accessibilitySummary — 一段人类可读的 可访问性摘要， 用于补充而非重复其他可发现性元数据。该摘要还 描述任何已知缺陷（例如，缺少扩展描述、特定 危害）。

EPUB 出版物 MAY 包含本节未规定的其他 [schema-org] 可访问性元数据。

注

本规范假定一致性和可发现性元数据将以 人类可读的方式处理并呈现，从而除非有额外 信息需要表达，否则不需要摘要。然而，这种处理并非在所有市场领域都可用， 尤其是在图书馆系统中。因此，该限制会影响是否包含 摘要以及在其中说明什么。

注

有关可与这些属性一起使用的已批准术语完整列表，请参阅 用于可发现性的 Schema.org 可访问性属性 词汇表 [a11y-discov-vocab]。

注

请参阅 在 EPUB 包文档中表达 可访问性元数据 指南，以获取关于这些属性以及如何在不同版本 EPUB 中包含它们的更多 信息。另请参阅 在 分发记录中包含可访问性元数据 [epub-a11y-tech-12]，以获取关于在其他格式中包含 可访问性元数据的更多 信息。

2.3 链接的元数据记录

可访问性元数据也可以包含在链接记录 [epub-3] 中（即，从 link 元素引用的元数据记录），但仅在链接记录中包含 此类元数据并不满足本规范的可发现性 要求。

3. 可访问出版物

3.1 引言

本节为非规范性内容。

EPUB 构建于开放 Web 平台之上，HTML、CSS、JavaScript 和 SVG 是用于 内容创作的核心技术。利用这些技术，作者可以通过应用既有的 Web 可访问性技术（例如尽可能使用原生元素和控件，并使用 [wai-aria] 角色、状态 和属性增强自定义交互式内容）来创作具有高度 可访问性的 EPUB 出版物。此外，只要可能，EPUB 社区还会把 出版可访问性需求加入这些标准——例如，通过创建 [dpub-aria-1.1] 角色模块。任何熟悉 Web 可访问性的人都不 需要学习新的可访问性框架来使 EPUB 出版物可访问。

制作可访问 Web 内容的主要来源是 W3C 的 Web 内容可访问性 指南（WCAG）[wcag2]，它为可访问内容建立 基准。WCAG 定义了四个 高层内容原则——内容应可感知、可操作、可理解且稳健。 这些原则也是创建可访问 EPUB 出版物的核心，因此本规范建立在 WCAG 中所完成的大量工作之上并不令人意外。

本节定义如何将 WCAG 中定义的一致性准则 应用于 EPUB 出版物。它还添加了EPUB 出版物 特有的其他要求，并定义了一致性报告 要求。

按照本节要求创作的 EPUB 出版物，将对具有各种阅读需求和偏好的用户 具有高度可访问性。

3.2 与 WCAG 的关系

本节为非规范性内容。

WCAG [wcag2] 及其相关技术 对 Web 内容可访问性的问题和解决方案提供了广泛覆盖，涵盖 从多媒体到交互式内容再到结构化标记等方方面面。它们代表了 本规范构建其上的基础。

本规范不重复这些文档中引入的要求或技术，因为 这有破坏两个标准之间兼容性的风险（例如，使指导不同步或 发生冲突）。与此同时，尽管本规范没有逐项指出这些要求， 但它并不降低这些要求在创建可访问 EPUB 出版物中的重要性。

相反，本规范定义如何将 WCAG 应用于 EPUB 出版物——它是一组 Web 文档，而非单个页面——并 添加一组附加要求。这些要求与 WCAG 中涵盖的要求相比 并非更重要或更不重要；它们只是 EPUB 出版物必须遵循的要求。（每项要求都会在其相应章节中解释它与 WCAG 的关系。）

EPUB 可访问性技术文档 [epub-a11y-tech-12] 中的技术也是如此。它覆盖了 EPUB 出版物特有的技术，或 需要在 EPUB 出版物语境中澄清的技术。这并不意味着其余 WCAG 技术不适用。

因此，尽管可以在没有深入了解 WCAG 一致性的情况下阅读本节， 但实现本规范的可访问性要求需要理解 WCAG。

因为本规范添加了不属于 WCAG 的要求，EPUB 出版物可以 符合 WCAG 而不符合本规范。

3.3 WCAG 一致性

3.3.1 WCAG 一致性 要求

为符合本规范，EPUB 出版物：

  • MUST 至少满足 WCAG 2.0 [wcag20] 的要求， 但强烈 RECOMMENDED 满足 最新 W3C WCAG 2 推荐标准的要求。

  • MUST 对所选择的任何 WCAG 2 版本， 满足 A 级的要求，但强烈 RECOMMENDED 满足 AA 级的要求。

    注

    尽管本规范不要求符合 AAA 级， 但在制作可访问 EPUB 出版物时，鼓励遵循 AAA 成功标准中详细说明的实践。

这些要求所提供的报告灵活性，是为了确保本规范可以 适用于强制要求可访问性的地区，同时不否定或取代 这些地区生效的要求。

例如，本规范将基线要求设为 WCAG 2.0 A 级，主要是为了 为较旧内容提供向后兼容性，并在没有正式要求的地方提供灵活性以鼓励 可访问制作的采用。然而，大多数可访问性从业者并不 认为该级别提供了高度可访问性。

理想情况下，EPUB 出版物应符合最新版本 WCAG 2 的 AA 级，但地方和 国家法律，或采购方或分发方要求，将定义它们必须满足的正式阈值。

注

可访问性立法要求的示例包括欧盟的 Directive 2019/882 以及美国的 1973 年康复法第 508 节。EPUB 出版物需要满足的不仅仅是基本的 A 级成功标准，才能 符合这些法律。

跟上 WCAG 的步伐有助于持续增强用户访问能力。随着 Web 技术的变化和改进，以及对阻碍访问条件的认识不断演进，该 标准会添加新要求。满足这些额外要求有助于确保 EPUB 出版物采用最新技术。满足较旧版本的要求 虽然仍有帮助，但可能导致不够理想的阅读体验。

同样，法律框架和政策通常将 AA 级一致性作为 可访问性的基准，因为该级别提供了最大范围且现实可 实现的改进。当只满足 A 级一致性时，多个用户 群体的可访问性会受损，从而导致不够理想的阅读体验。

区域政策也可以施加自己的可访问性要求。这些要求 并不总是由 WCAG 覆盖（例如，特定语言方面的考虑）。在某些情况下，地区 可能会规定低于 A 级的最低一致性。如果出版物达到区域可访问性标准， 但未满足 A 级要求，则不能声明符合本 标准。

注

W3C 可访问性指南工作 组目前正在开发 WCAG 3。由于该版本可能代表与 WCAG 2 的重大背离，本规范的未来版本将处理 与其相关的一致性要求。一旦 WCAG 3 稳定并被广泛认可，即鼓励采用， 但符合新版本并非本标准的要求。

3.3.2 评估 WCAG 一致性

3.3.2.1 页面和出版物

WCAG 原则 [wcag2] 关注对单个网页的 评估，但 EPUB 出版物更接近 WCAG 所称的一组网页：“[a] collection of web pages that share a common purpose” [wcag2]。

因此，在评估 EPUB 出版物的可访问性时，不能 孤立地审查单个页面——或者 EPUB 3 中所谓的 EPUB 内容文档。相反，完整的 EPUB 出版物 MUST 按 WCAG 指南 对内容是否可感知、可操作、可理解和稳健进行评估。

例如，如果 EPUB 内容文档中的内容排序正确， 但spine 中的文档排序不正确， 这并不足够。同样，为每个 EPUB 内容文档包含标题，是对为出版物提供 标题的补充：如果缺少任一项，整体可访问性都会下降。

EPUB 可访问性技术 [epub-a11y-tech-12] 提供了更多 关于将这些指南应用于 EPUB 出版物的信息。

3.3.2.2 应用 一致性准则

在评估 EPUB 出版物时，WCAG 一致性准则 [wcag2] 按如下方式应用：

  • 在确定是否符合某个一致性级别时，整个 EPUB 出版物 MUST 满足所声明级别的一致性要求。
  • EPUB 出版物 MUST NOT 使用 EPUB 的 回退机制来提供符合要求的替代版本 [wcag2]，因为用户没有 可靠方式访问这些回退。如果 EPUB 出版物使用回退，则主要 内容及其回退内容 MUST 满足所声明一致性级别 的要求。EPUB 特定的回退机制包括 manifest 回退 [epub-3]、bindings [epub-3] 以及 通过 epub:switch 元素 [epub-3] 进行内容切换。
  • “完整 页面”要求 [wcag2] ——即在提出一致性声明时 不能排除页面的部分内容——适用于 EPUB 出版物中的每一个 EPUB 内容文档 （即，它们都必须完整符合所声明的一致性级别）。

3.4 EPUB 要求

3.4.1 页面导航

3.4.1.1 概述

本节为非规范性内容。

静态分页内容仍然无处不在，因为纸质书在普通阅读大众和教育环境中 仍是最常消费的图书媒介。纸质书也不是 静态分页的唯一来源：固定版式数字出版物中也存在静态页面边界。

因此，在使用静态分页内容的环境中，非视觉读者相较于其同伴会面临不利， 因为他们无法轻松定位出版物中的相同位置 （例如，如果教师要求学生都翻到某一特定页面）。

包含页面边界位置有助于弥合这种差距，确保选择 可重排媒体不会让这些用户处于不利地位。

在没有静态分页等价物的可重排出版物中，提供页面导航也有帮助。 这些出版物由 阅读系统默认分页并非静态，因为它 会根据 viewport 大小和用户的 字体设置而变化。因此，如果没有静态引用，同一 EPUB 出版物用户之间协调位置可能会很复杂。

包含页面导航代表实现 多种方式成功 标准 [wcag2] 的一种方法，因为它 为用户提供了另一种有意义的访问内容方式（例如，除 目录、线性阅读顺序和任何其他导航辅助之外）。

鉴于页面导航在混合纸质/数字环境中的重要性，包含此特性的要求 比仅作为满足多种方式成功标准的许多方式之一时 具有更高优先级。

注

有关在 EPUB 出版物中包含页面导航的更多信息，请参阅 页面导航 [epub-a11y-tech-12]。

3.4.1.2 适用性

当以下任一情况为真时，EPUB 出版物 SHOULD 包含页面 导航：

  • 该 EPUB 出版物是静态分页出版物的动态分页等价物 （例如，包含在纸质/数字捆绑包中）；
  • 该 EPUB 出版物在可合理预期同时使用两个版本的环境中 作为静态分页出版物的替代品提供（例如， 教育环境）；或
  • 该 EPUB 出版物由与静态分页出版物相同的工作流生成， 并且该工作流允许跨 格式保留分页位置。

没有静态分页等价物的 EPUB 出版物 MAY 包含 页面导航。

只有当 EPUB 出版物包含页面导航时，本节定义的目标才适用于它。

3.4.1.3 目标
3.4.1.3.1 分页来源
理解分页 来源 分页来源的技术
目标

标识静态分页位置的来源。

满足此目标

当 EPUB 出版物包含与出版物的静态分页版本对应的 分页符标记 和/或页面列表时，该出版物 MUST 在包 文档元数据中标识 该来源。

然而，如果这些特性被添加到仅数字版 EPUB 出版物中，该 出版物 MUST NOT 指定来源（即，不要 将当前出版物标识为其自身分页的来源）。

3.4.1.3.2 页面列表
理解页面列表 页面列表技术
目标

提供到静态分页位置的导航。

满足此目标

EPUB 出版物 MUST 包含页面列表。

页面列表 SHOULD 包含指向从来源复制的所有内容页面的链接 （即，它不必包含空白页或数字版本中未 复制内容的链接）。

页面列表 MUST 包含指向内容中所有分页符 标记的链接。

鼓励在页面列表中包含指向来源中所有页面的链接， 无论其是否被复制，但这不是一致性 要求。

3.4.1.3.3 分页符
理解分页符 分页符技术
目标

提供静态分页位置。

满足此目标

在 EPUB 出版物中包含分页符标记是 OPTIONAL。

如果 EPUB 出版物包含分页符标记，则分页符标记 SHOULD 标识从来源复制的所有页面（即，空白页和数字版本中未 复制的内容不需要标记）。

鼓励为来源中的所有页面包含分页符标记， 无论其是否被复制，但这不是要求。

此外，如果在内容的同步文本-音频播放中朗读页码 （例如，EPUB 3 媒体叠加 [epub-3]），则分页符标记 MUST 在播放指令中标识（例如， 在媒体叠加 SMIL 文件中使用 epub:type 属性 [epub-3]）。

3.4.2 同步 文本-音频播放

3.4.2.1 概述

本节为非规范性内容。

提供同步文本-音频播放有助于解决各种用户需求。它不仅 能够从 EPUB 出版物开头到结尾实现无缝的视觉和听觉阅读体验， 也有助于只需要音频播放的用户 （例如，看不见文本或因晕动症而无法视觉阅读的用户），或只从带文本高亮的阅读中受益的用户 （例如，有阅读障碍的读者）。

与纯线性聆听体验不同，带同步文本-音频播放的 EPUB 保留了用户在出版物中导航的能力，例如通过 目录导航，并且还引入了以音频为中心的阅读特性，如短语导航以及 控制朗读内容哪些部分的方式。

为了为用户提供对内容呈现的更大控制，EPUB 出版物需要 包含结构和语义，以便 阅读系统具备启用这种用户体验所需的上下文。 借助更丰富的上下文，阅读系统可以提供跳过干扰主要叙事的次要 内容，以及让用户从表格等深层嵌套 结构中退出的能力。

向同步文本-音频播放添加结构和语义，大体上属于 信息与关系 成功 标准 [wcag2] 的目标。没有 结构化且具有语义意义的播放 序列，其效果就是剥夺用户对内容进行丰富导航的能力。

3.4.2.2 适用性

带同步 文本-音频 播放的 EPUB 出版物 MUST 符合 [epub-3] 中的所有要求。无需 满足 [epub-3] 中定义之外的任何 附加要求即可符合本规范。

然而，为了最大限度提高同步文本-音频播放对具有不同 阅读需求的人群的有效性，强烈鼓励 EPUB 出版物满足下一节中定义的 OPTIONAL 目标。

注

这些要求仅在 EPUB 出版物包含同步文本-音频 播放时适用；它们不要求所有 EPUB 出版物都包含同步播放 才能符合要求。

3.4.2.3 目标
3.4.2.3.1 完整性
理解完整性 完整性技术
目标

确保所有文本内容都可通过音频获得。

满足此目标

EPUB 出版物 MUST 包含 对所有可见文本内容以及所有视觉媒体文本替代内容的 同步音频播放。

3.4.2.3.2 阅读顺序
理解阅读顺序 阅读顺序技术
目标

确保同步文本-音频播放与逻辑阅读顺序相匹配。

满足此目标

同步文本-音频播放指令的顺序 SHOULD 同时反映：

  • spine中所引用 EPUB 内容文档的顺序；以及
  • 其各自 EPUB 内容文档内每个元素的顺序。

如果 EPUB 出版物使用不同的排序，该排序 MUST 仍然产生 内容的逻辑播放。

3.4.2.3.3 可跳过性
理解可跳过性 可跳过性技术
目标

使用户能够自动跳过内容。

满足此目标

同步文本-音频播放指令 SHOULD 标识所有可跳过的 结构。

3.4.2.3.4 可退出性
理解可退出性 可退出性技术
目标

使用户能够自动从结构化内容中退出。

满足此目标

同步文本-音频播放指令 SHOULD 标识所有可退出的 结构。

3.4.2.3.5 导航 文档
理解导航 文档 导航 文档技术
目标

确保当阅读 系统呈现 EPUB 导航文档中的导航辅助时， 可以进行听觉播放。

满足此目标

EPUB 出版物 SHOULD 包含 对 EPUB 导航 文档 [epub-3] 的同步文本-音频播放。

3.5 一致性报告

3.5.1 引言

本节为非规范性内容。

评估者通过在包文档中表达 元数据属性，报告 EPUB 出版物的可访问性一致性。

该元数据同时确立：

  • 所达到的一致性级别；以及
  • 关于谁执行了评估的信息。

该元数据使用 DCMI Metadata Terms [dcterms] 中的属性和 EPUB 可访问性词汇表中的属性组合，如以下 章节中更详细解释的那样。

虽然任何个人或团体都可以执行一致性评估——只要他们具备 根据本规范的 WCAG 和 EPUB 要求评估 EPUB 出版物的知识和工具—— 但用户需要能够相信评估是以全面方式执行的，并且 一致性声明不受结果自利因素影响。

为解决这种信任问题，一些地区指定地方机构负责所有 评估，在这种情况下，由谁执行评估没有灵活性。在不存在此类 机构的情况下，鼓励评估者获得并引用任何适用的凭据，以增强公众对其 评估的信心。

注

由于每种元数据格式在可表达内容方面都是独特的，本规范不强制规定 如何在 EPUB 包文档之外表达一致性元数据。

确保内部和外部可访问性元数据表达之间的一致性，是 作者、出版商和分发商的责任。有关分发对 可访问性影响的讨论，请参阅 4. 分发。

3.5.2 出版物一致性

为了表明符合本规范的可访问性要求，EPUB 出版物 [epub-3] MUST 在其 元数据节中指定一个 conformsTo 属性 [dcterms]， 其值在空白规范化 [xml] 后，必须 精确匹配（即， 大小写和间距都匹配）以下模式：

EPUB Accessibility 1.2 - WCAG WCAG-VER Level WCAG-LVL

其中：

WCAG-VER

指定出版物所符合的 WCAG 版本号。该值 MUST 为 2.0 或更高版本。

WCAG-LVL

指定出版物所符合的 WCAG 一致性级别（例如，A 或 AA）。

示例 1：基本一致性声明

在此示例中，EPUB 出版物元数据表明该内容符合 EPUB Accessibility 1.2 规范，级别为 WCAG 2.2 Level AA。

<package …>
   <metadata …>
      …
      <meta 
          property="dcterms:conformsTo"
          id="conf">
         EPUB Accessibility 1.2 - WCAG 2.2 Level AA
      </meta>
      …
   </metadata>
   …
</package>

以下字符串可有效用于标识对本 规范的一致性

  • EPUB Accessibility 1.2 - WCAG 2.2 Level A
  • EPUB Accessibility 1.2 - WCAG 2.2 Level AA
  • EPUB Accessibility 1.2 - WCAG 2.2 Level AAA
  • EPUB Accessibility 1.2 - WCAG 2.1 Level A
  • EPUB Accessibility 1.2 - WCAG 2.1 Level AA
  • EPUB Accessibility 1.2 - WCAG 2.1 Level AAA
  • EPUB Accessibility 1.2 - WCAG 2.0 Level A
  • EPUB Accessibility 1.2 - WCAG 2.0 Level AA
  • EPUB Accessibility 1.2 - WCAG 2.0 Level AAA
注

WCAG 3.0 将引入新的级别名称 （目前为 Bronze、Silver 和 Gold）。这些名称很可能会在 字符串模式中取代 A、AA 和 AAA，但一致性将在该规范成为 W3C 推荐标准后处理。

3.5.2.1 其他一致性 声明

本节为非规范性内容。

要求包含 dcterms:conformsTo 标识符并不阻止 EPUB 出版物符合其他标准，包括其他可访问性标准和 指南（例如，覆盖特定自然语言问题的规范）。EPUB 出版物可以有多个 dcterms:conformsTo 陈述。

例如，如果 EPUB 出版物只满足 WCAG 一致性要求（即，不 完全符合本规范），元数据可以包含一个 dcterms:conformsTo 属性，其值为 一致性声明的必要组成部分 [wcag2] 中定义的一致性 URL。

注

由于 [wcag2] 未 定义如何在 URL 中指定一致性级别，因此 需要一种替代方式来关联此信息（例如，它可以 在可访问性摘要中表达）。

相反，为了表明 EPUB 出版物不满足任何可访问性标准 （例如，为了声明豁免），元数据可以包含一个 dcterms:conformsTo 属性，其值为 “none”。

注

有关声明豁免的更多信息，请参阅 EPUB Accessibility exemption 属性 [epub-a11y-exemption]。

3.5.3 评估者信息

3.5.3.1 基于声明数量的表达差异

本节为非规范性内容。

如何向 EPUB 出版物添加评估者信息取决于所作一致性声明的 数量。

大多数出版物只会包含一个 dcterms:conformsTo [dcterms] 属性，并带有内容的可访问性声明。 在这种情况下，不需要使用 EPUB 3 的元数据关联机制 [epub-3] 来连接各种元数据 陈述。

考虑以下示例元数据：

<metadata …>
   …
   <meta
       property="dcterms:conformsTo"
       id="conf">
      EPUB Accessibility 1.2 - WCAG 2.2 Level AA
   </meta>
   <meta
       property="a11y:certifiedBy">
      Foo's Accessibility Testing
   </meta>
   <meta
       property="a11y:certifierCredential">
      A+ Accessibility Rating
   </meta>
   <meta
       property="dcterms:date">
      2021-09-07
   </meta>
   <link
       rel="a11y:certifierReport"
       href="reports/a11y.xhtml"/>
   …
</metadata>

由于只有一个一致性声明，可以推定所有额外的评估 信息都与其相关。

注

如果希望让元数据连接明确化，如下一部分所述， 即使只有一个声明也仍然允许这样做。

只有当存在多个 dcterms:conformsTo 陈述时， 才必须使用 refines 属性 [epub-3]。如果不将元数据 陈述链接在一起，信息就不再易于机器处理，这可能导致 其向用户呈现时出错。

以下示例显示与前一示例相同的元数据，但这次所有内容都 通过使用 refines 属性连接起来，因为出版商添加了 对其内部标准的一致性声明。

<metadata …>
   …
   <meta
       property="dcterms:conformsTo"
       id="internal">
      Acme Publishing's EPUB Standards
   </meta>
   …
   <meta
       property="dcterms:conformsTo"
       id="conf">
      EPUB Accessibility 1.2 - WCAG 2.2 Level AA
   </meta>
   <meta
       id="certifier"
       property="a11y:certifiedBy"
       refines="#conf">
      Foo's Accessibility Testing
   </meta>
   <meta
       property="a11y:certifierCredential"
       refines="#certifier">
      A+ Accessibility Rating
   </meta>
   <meta
       property="dcterms:date"
       refines="#certifier">
      2021-09-07
   </meta>
   <link
       rel="a11y:certifierReport"
       refines="#certifier"
       href="reports/a11y.xhtml"/>
   …
</metadata>

refines 属性现在允许建立返回 EPUB Accessibility 标准一致性声明的路径，从而将所有评估者信息 链接到它。

以下章节更详细解释评估者元数据的用途， 以及在需要时如何使用 refines 属性。

3.5.3.2 评估者姓名

包文档元数据 MUST 包含一个 a11y:certifiedBy 属性， 该属性指定评估 EPUB 出版物的主体名称。

如果作出多个一致性声明，则将评估者与其 关联 [epub-3] 到其声明。

注

如果一个组织评估 EPUB 出版物，用户通常会想知道该 组织的名称。本规范不鼓励包含执行评估的 个人姓名而非组织名称，因为 这可能削弱用户对该声明的信任。

3.5.4 重新评估一致性

本节为非规范性内容。

注

以下指导仅为帮助确定何时需要新的评估而提供。 它不是本规范的一致性要求。

EPUB 出版物的一致性评估有效期有多长，是一个复杂问题。与不断 演进的网站不同，EPUB 出版物在初次出版后通常不会再次更新。因此， 未修改的 EPUB 出版物将始终符合其最后一次评估。

然而，在出版领域，发布 EPUB 出版物的更新版本以修正 作品中的错误和错字，以及定期发布新版是很常见的。由于并非所有 对 EPUB 出版物的更改都会实质性改变其可访问性，这使得 何时执行新的评估，以及完整还是部分重新评估 是否足够的问题变得复杂。

作为一般规则，当对 EPUB 出版物的结构和 功能进行实质性更改时，应重新评估内容，例如：

  • 修改 EPUB 内容文档中标记的性质或顺序；
  • 添加或修改传达信息的图像；
  • 修改影响可读性的格式（例如，对比度）；以及
  • 添加或修改交互式控件、表单等。

如果 EPUB 出版物包含与上一版本实质上相同的标记和内容， 则很可能只需要评估新的修改，以重新确认一致性。

如果自 EPUB 出版物上次 发布以来，本规范或 [wcag2] 的更新版本已经发布，则建议进行新的评估，以确保符合最新 标准。即使在这种情况下，也可能不需要完整的重新评估。例如，除非标准在 方法论或一致性方面发生重大变化，否则可能只需要检查新增 或修改的成功标准。

相反，当只对 EPUB 出版物进行非实质性更改时， 该出版物不需要重新评估，例如：

  • 修正文中的排印错误（即，标记未更改）；
  • 添加或修改装饰性图像；
  • 修改不影响内容理解或不 改变文本显示的格式；以及
  • 修改包文档 元数据。

具备评估 EPUB 出版物可访问性资格的个人，需要判断 更改是否具有实质性。例如，编辑可能没有意识到 看似微小的格式更改所带来的影响。

即使在非实质性更改的情况下，如果可访问性标准已经发生变化，本规范也建议进行 更新评估（完整或部分）。

强烈鼓励考虑比这里描述的方法更渐进的方式。 等待内容变更后再审查和更新 EPUB 出版物的可访问性， 可能会使其缺少近期改进。例如，出版商可能会优先定期 审查其最畅销的 EPUB 出版物，以确保其对尽可能广泛的受众保持最大可用性。

3.6 定向可访问性

本节为非规范性内容。

虽然 WCAG [wcag2] 提供了一套通用指南 用于使内容广泛可访问， 但符合要求的内容并不总是对特定用户群体最优。相反，针对 特定需求或阅读方式优化的内容，往往恰恰因为面向 特定受众而不符合 WCAG。

例如，带同步 文本和音频的 EPUB 出版物可以包含完整的内容音频录音， 但将文本内容限制为只有主要标题。在这种情况下，EPUB 出版物 可供需要听取内容的用户消费（即，他们可以聆听完整出版物并在标题之间导航）， 但对任何听不到音频的人都不可用。

换言之，当 EPUB 出版物面向特定阅读方式时， 未达到 WCAG 一致性级别并不意味着该出版物对于预期 受众的可访问性有所降低。

虽然为面向特定读者需求的出版物定义要求超出了 本规范的范围，但 A. 优化出版物标准指南提供了一些最佳实践，用于 以符合本文档目标的方式创建此类标准。

3.7 反馈

虽然预期出版物会在认证其可访问性之前进行评估，但由于 产出的内容数量极大，且检查所有标记很复杂，有时 问题会漏到最终产品中。可访问性报告也可能不够 详细，无法让每个用户确定内容是否适合其 需求。

为协助用户报告他们遇到的任何可访问性问题，或进一步了解 EPUB 出版物的可访问性，EPUB 出版物 MAY 使用 a11y:contactEmail 属性包含 可访问性联系邮箱。

此字段中的邮箱应将用户反馈和问题直接指向负责 出版物可访问性的团体或 个人。它 SHOULD NOT 是通用 反馈地址。

<meta property="a11y:contactEmail">
   a11y-info@example.com
</meta>

4. 分发

本节为非规范性内容。

注

虽然 EPUB 出版物不必满足本节中的 建议即可符合本规范，但某些司法辖区要求 EPUB 出版物遵循类似实践。例如，Directive 2019/882 就包含对欧盟境内分发的数字出版物的类似要求。

创建可访问的 EPUB 出版物并不 保证用户能够以可访问的方式获取或消费该出版物。 根据其分发方式，其他因素会影响其整体可访问性。例如， 用于查找和获取内容的可访问界面是分发过程的必要组成部分， 搜索和查看可访问性元数据的能力也是如此。

虽然分发过程的许多方面超出了本规范的范围，但仍有一些 可控制的因素。例如，虽然出版商通常无法控制应用于其 EPUB 出版物的 数字版权管理（DRM）方案的可访问性，但他们确实 可以控制对其 EPUB 出版物应用哪些使用权。因此，即使 DRM 方案可能允许 出版商阻止对出版物文本的访问，该出版商也需要注意不要应用 这种限制，因为它可能会阻止辅助技术朗读文本 的能力。

为尽量减少分发对可访问性的影响，本规范建议遵循以下 分发实践：

  • 不要施加会损害辅助技术访问的限制；以及
  • 在分发 EPUB 出版物所需的记录格式 （例如 [onix] 或 [marc21]）支持此类 元数据时，在其中包含可访问性元数据。
注

本节中的指导并不限制 EPUB 出版物通过 不可访问的分发渠道进行分发。例如，分发商应用了会损害 内容访问的 DRM 的情况。

5. 隐私与安全

创作可访问内容不会为用户引入任何新的隐私或安全 考虑。满足可访问性要求关乎最佳地使用可用技术，本规范 不引入任何新特性。

包含可访问性元数据同样不会引入安全或隐私问题，因为 描述一个 EPUB 出版物只提供其适合不同用户群体的 一般概念。

另一方面，在阅读系统、 书店以及任何其他可以构建用户画像的界面中使用可访问性元数据， 可能会侵犯用户隐私。例如，存储并预测用户最可能消费的内容 类型，或预测如何最好地启动其播放，可能看起来有帮助，但建议开发者 不要进行此类画像分析，除非已获得用户明确许可，并且提供 可轻松删除该画像的方式。

即使用户同意应用程序维护有关其可访问性需求的信息， 开发者也 SHOULD 确保这些信息保持私密 （例如，不与第三方广告商共享该信息，甚至不与原始出版商共享）。

开发者 SHOULD NOT 存储或挖掘用户在 根据内容可访问性特征搜索内容时所执行搜索类型的信息。此类信息可用于 间接描绘用户的能力。

A. 优化出版物标准 指南

本节为非规范性内容。

优化的 EPUB 出版物是指为解决特定 可访问性需求而创建的出版物，如 3.6 定向可访问性中所讨论。

虽然这些出版物不能符合本规范的要求，但其可访问发现和 一致性报告的一般模型可以作为模板，用于确保用户在偏好此类内容时 能够找到优化内容。

特别是，建议在任何优化标准中将以下实践整合为正式要求：

  • 符合要求的 EPUB 出版物满足本规范的发现元数据 要求，以便其可访问属性可由 机器读取。

  • 符合要求的 EPUB 出版物按照 [dcterms]，在 conformsTo 属性中标识其遵循的标准，以便将该信息提供给 用户。如果 这不可行：

    • 由于该标准未定义一致性值，建议符合要求的 出版物包含一个 conformsTo 属性，其中包含一个 URL [url]，指向该 标准公开可用的位置，以便用户可以查阅该优化的具体细节。
    • 由于标识符不足以让用户理解一致性（例如， 该标准未公开可用），建议符合要求的出版物在可访问性摘要中提供关于其优化的 附加信息。
注

有关优化标准列表，请参阅 新链接待定。

B.2.1 certifiedBy

certifiedBy 属性的定义
名称： certifiedBy
描述： 标识负责测试和认证 EPUB 出版物可访问性的 主体。
允许的值： xsd:string
基数： 一个或多个

描述： 标识一项凭据或徽章，用于确立在关联的 certifiedBy 属性中标识的主体 认证内容可访问的权限。
允许的值： xsd:string
基数： 零个或多个
扩展： a11y:certifiedBy

<meta
    property="a11y:certifiedBy"
    id="certifier">
   Accessibility Testers Group

    property="a11y:certifierCredential"

certifierReport 属性的定义
名称： certifierReport
描述： 提供指向由关联的 certifiedBy 属性中标识的 主体创建的可访问性报告的链接。
允许的值： xsd:anyURI
基数： 零个或多个

D. 变更日志

本节为非规范性内容。

请注意，此变更日志仅标识自 EPUB Accessibility 1.1 以来的实质性变更——即那些可能影响 EPUB 出版物一致性的变更。

有关已处理的所有议题列表，请参阅工作组的 议题跟踪器。

自 EPUB Accessibility 1.1 以来的实质性变更
  • 2026 年 3 月 26 日：澄清只有在存在多个一致性声明时，才需要使用 refines 属性将 可访问性元数据陈述链接在一起。参见 议题 2846。
  • 2025 年 9 月 5 日：向评估者凭据字段添加说明，指出虽然 URL 是有效值， 但其显示通常需要额外处理。参见 议题 2112。
  • 2025 年 9 月 4 日：添加 a11y:contactEmail，以允许 EPUB 创建者包含 用于获取更多可访问性信息和表达关切的联系邮箱。参见 议题 2702。
  • 2025 年 6 月 26 日：将 accessMode 改为推荐属性，并将 accessModeSufficient 改为必需属性， 以更好地反映它们在判断图书是否可读方面的有用性。参见 议题 2679。
  • 2025 年 6 月 2 日：明确评估日期必须符合 ISO 8601。参见 议题 2725。
