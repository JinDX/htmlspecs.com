1. 简介

本节为非规范性内容。

目前已有一个 Web API 能将 HTMLVideoElement 放入画中画窗口（requestPictureInPicture()）。这限制了网站定制画中画体验（PiP）的能力。我们希望通过提供始终置顶窗口里的完整 Document 来扩展这一功能。

这个新窗口类似于通过现有 open() 方法在 Window 上打开的同源空白窗口，但有一些细微区别：

  • PiP 窗口会浮在其他窗口之上。

  • PiP 窗口不会比打开它的窗口存活更长。

  • 网站无法设置 PiP 窗口的位置。

  • PiP 窗口不可导航（任何改变到新文档的 `window.history` 或 `window.location` 调用都会关闭 PiP 窗口）。

3. 安全注意事项

3.1. 安全上下文

此 API 仅限于 [SECURE-CONTEXTS]。

3.2. 欺骗

要求用户代理在 DocumentPictureInPicture 窗口上提供足够的界面，防止恶意网站利用浮窗功能欺骗其它网站或系统界面。

3.2.1. 定位

用户代理必须阻止网站设置窗口位置，以避免网站故意将窗口定位到可能让用户误以为属于别页界面的地方。特别是 moveTo() 和 moveBy() API 必须在文档画中画窗口中禁用。

3.2.2. 来源可见性

要求用户代理始终向用户明确显示正在控制 DocumentPictureInPicture 窗口的来源，确保用户知道内容来自哪里。例如，用户代理可以在窗口标题栏显示网站的来源。

3.2.3. 最大尺寸

用户代理应限制文档画中画窗口的最大尺寸，防止网站用始终置顶窗口遮盖整个屏幕把用户锁定在画中画窗口里。这也有助于防止欺骗用户桌面。

3.3. IFrame 内联框架

本 API 仅适用于 顶层可遍历对象。但 DocumentPictureInPicture Window 仍可包含 HTMLIFrameElement，包括 跨域 HTMLIFrameElement。

4. 隐私注意事项

4.1. 指纹识别

当 PiP 窗口关闭再重新打开时，用户代理可重复之前窗口的大小和位置，以优化用户体验。但建议用户代理不要跨来源复用大小或位置，否则可能被恶意网站用于识别用户。

5. API

[Exposed=Window]
partial interface Window {
  [SameObject, SecureContext] readonly attribute DocumentPictureInPicture
    documentPictureInPicture;
};

[Exposed=Window, SecureContext]
interface DocumentPictureInPicture : EventTarget {
  [NewObject] Promise<Window> requestWindow(
    optional DocumentPictureInPictureOptions options = {});
  readonly attribute Window window;
  attribute EventHandler onenter;
};

dictionary DocumentPictureInPictureOptions {
  [EnforceRange] unsigned long long width = 0;
  [EnforceRange] unsigned long long height = 0;
  boolean disallowReturnToOpener = false;
  boolean preferInitialWindowPlacement = false;
};

[Exposed=Window, SecureContext]
interface DocumentPictureInPictureEvent : Event {
  constructor(DOMString type, DocumentPictureInPictureEventInit eventInitDict);
  [SameObject] readonly attribute Window window;
};

dictionary DocumentPictureInPictureEventInit : EventInit {
  required Window window;
};

一个 DocumentPictureInPicture 对象允许网站创建和打开新始终置顶的 Window，以及监听与该 Window 打开与关闭相关的事件。

每个 Window 对象都有一个关联的 documentPictureInPicture API，即在创建 DocumentPictureInPicture 实例的同时创建。

documentPictureInPicture 取值步骤如下：

  1. 返回 this 的 documentPictureInPicture API。

每个 DocumentPictureInPicture 对象都有一个关联的 最后打开的窗口，它是初始值为 null，并在 requestWindow() 方法步骤时设置的 Window 对象。

window 取值步骤如下：

  1. 令 win 为 this 的 最后打开的窗口。

  2. 如果 win 不为 null 且 win 的 closed 属性为 false，则返回 win。

  3. 返回 null。

requestWindow(options) 方法步骤如下：

  1. 如果 文档画中画支持 为 false，抛出 "NotSupportedError" DOMException。

  2. 如果 this 的 相关全局对象 的 navigable 不是 顶层可遍历对象，抛出 "NotAllowedError" DOMException。

  3. 如果 this 的 相关全局对象 的 navigable 的 是否为文档画中画布尔值为 true，抛出 "NotAllowedError" DOMException。

  4. 如果 this 的 相关全局对象没有 临时激活，抛出 "NotAllowedError" DOMException。

  5. 如果 options["width"] 存在且大于零，但 options["height"] 不存在或为零，则抛出 RangeError。

  6. 如果 options["height"] 存在且大于零，但 options["width"] 不存在或为零，则抛出 RangeError。

  7. 消耗用户激活，参数为 this 的 相关全局对象。

  8. 令 win 为 this 的 最后打开的窗口。如果 win 不为 null 且 win 的 closed 属性为 false，则 关闭 win 的 navigable。

  9. 用户代理可选地 关闭任何现有的画中画窗口。

  10. 令 pip traversable 为结果：创建新顶层可遍历对象，参数为 this 的 相关全局对象 的 navigable 的 活动浏览上下文 和 "_blank"。

生成的 Document 的 URL 为 `about:blank`，但其 文档基础 URL 回退为调用 requestWindow() 的发起者的 URL。有些浏览器对普通 `about:blank` 弹窗不实现此回退行为，相关讨论见 whatwg/html#421。建议实现者在文档画中画窗口按规范继承此行为，以避免更多兼容性问题。

  1. 将 pip traversable 的 活动文档的 模式设置为 this 的 相关全局对象 的 关联 Document 的 模式。

  2. 将 pip traversable 的 是否为文档画中画布尔值设为 true。

  3. 如果 options["width"] 存在且大于零：

    1. 可选：如值过大或过小可限制或忽略 options["width"]，以获得用户友好窗口尺寸。

    2. 可选：将 pip traversable 的 活动浏览上下文窗口设为视窗左右边距距离为 options["width"] 像素。

  4. 如果 options["height"] 存在且大于零：

    1. 可选：如值过大或过小可限制或忽略 options["height"]，以获得用户友好窗口尺寸。

    2. 可选：将 pip traversable 的 活动浏览上下文窗口设为视窗上下边距距离为 options["height"] 像素。

若 options["preferInitialWindowPlacement"] 存在且为 true，则用户代理可以利用这一提示，以更倾向于类似第 13、14 步的行为，而不是参考之前关闭的 pip traversable 窗口的位置信息或大小。

  1. 如果 options["disallowReturnToOpener"] 存在且为 true，则用户代理应不在画中画窗口显示允许用户返回原窗口的界面元素。

无论是视频还是文档画中画，用户代理常会显示一个按钮让用户返回原页面并关闭画中画窗口。大多数情况下此操作合理（尤其对于把视频放回主文档的画中画窗口），但对文档画中画窗口并不总适用。disallowReturnToOpener 是网站提示用户代理该操作是否适合其特定的文档画中画体验。

  1. 将 pip traversable 的 活动浏览上下文 的窗口配置为浮于其他窗口之上。

  2. 将 this 的 最后打开的窗口设置为 pip traversable 的 活动窗口。

  3. 在this的相关全局对象上，使用 DOM 操作任务源 排队全局任务，触发名为 enter 的事件，事件类型为 DocumentPictureInPictureEvent， 并将其 window 属性初始化为 pip traversable 的 活动窗口。

  4. 返回 pip traversable 的 活动窗口。

窗口大小可由网站配置，初始位置由用户代理决定。

enter

当 PiP 窗口打开时，在 DocumentPictureInPicture 上触发。

6. 概念

6.1. 文档画中画支持

每个用户代理都拥有一个 文档画中画支持 布尔值，其值为 实现自定义（并且可能会根据用户偏好而变化）。

6.2. DocumentPictureInPicture 窗口

每个 顶层可遍历对象 都有一个 是否为文档画中画 布尔值，其默认值为 false，但可在 requestWindow() 方法步骤中设置为 true。

6.3. 关闭文档画中画窗口

等获得足够共识后并入 close。

修改 close 第 2 步，"如果为 toUnload 检查卸载是否被用户取消的结果为 true，则返回。"，修改为：

  1. 如果 traversable 的 是否为文档画中画 布尔值为 true，则跳过此步。否则，如果为 toUnload 检查卸载是否被用户取消 的结果为 true，则返回。

6.4. 关闭现有画中画窗口

关闭现有画中画窗口的操作：

  1. 对于用户代理的 顶层可遍历对象集合 中每一个 top-level traversable：

    1. 如果 top-level traversable 的 是否为文档画中画 布尔值为 true，则 关闭 top-level traversable。

    2. 如果 top-level traversable 的 活动文档 的 pictureInPictureElement 不为 null，则以 top-level traversable 的 活动文档 运行 退出画中画算法。

    3. 对 top-level traversable 的 活动文档 的每个 navigable 后代 navigables：

      1. 如果 navigable 的 活动文档 的 pictureInPictureElement 不为 null，则以 navigable 的 活动文档 运行 退出画中画算法。

6.5. 单一画中画窗口

任意 顶层可遍历对象 在同一时刻最多只能打开一个文档画中画窗口。如果 顶层可遍历对象 的 活动窗口的 documentPictureInPicture API 的 最后打开的窗口非 null，且尝试再打开新的文档画中画窗口，用户代理应按照 requestWindow() 方法步骤关闭现有 最后打开的窗口。

但是否所有 顶层可遍历对象画中画窗口只能有一个，取决于具体实现与平台。因此，当已存在画中画请求，而仍有 顶层可遍历对象的 是否为文档画中画布尔值为 true 或 活动文档 的 pictureInPictureElement 不为 null 时的行为，留作实现细节：用户代理可 关闭现有画中画窗口，也可允许创建多个画中画窗口。

6.6. 原始文档或 PiP 文档销毁时关闭 PiP 窗口

给定 Document document，关闭所有关联的文档画中画窗口操作：

  1. 令 navigable 为 document 的 node navigable。

  2. 如果 navigable 不是 顶层可遍历对象，则中止步骤。

  3. 如果 navigable 的 是否为文档画中画布尔值为 true，则 关闭 navigable并中止步骤。

  4. 令 win 为 navigable 的 活动窗口的 documentPictureInPicture API 的 最后打开的窗口。

  5. 如果 win 不为 null 且 win 的 closed 属性为 false，则 关闭 win 的 navigable。

等获得足够共识后并入 destroy 。

向 destroy 末尾添加第 10 步：

  1. 关闭所有关联的文档画中画窗口，参数为 document。

这确保当具有打开的文档画中画窗口的页面关闭时，其 PiP 窗口也会关闭。

6.7. 原始文档或 PiP 文档导航时关闭 PiP 窗口

等获得足够共识后并入 navigate。

修改 navigate 的步骤 16.3，"针对 navigable 的活动窗口，在导航与遍历任务源上排队全局任务，以终止 navigable 的活动文档"，并在其后插入步骤 16.4：

  1. 在 navigable 的 活动窗口 上，针对 导航与遍历任务源，排队一个全局任务，以 终止 navigable 的 活动文档，并针对 navigable 的 活动文档 关闭所有关联的文档画中画窗口。

  2. 如果 navigable 是 顶层可遍历对象，且其 是否为文档画中画布尔值为 true，则中止这些步骤。

这确保当具有打开的文档画中画窗口的页面被导航时，其 PiP 窗口也会关闭。也确保当文档画中画窗口里的文档被导航时，画中画窗口会关闭。

6.8. 调整 PiP 窗口大小

虽然以编程方式调整文档画中画窗口大小非常有用，但由于该窗口始终置顶，因此无约束调整大小可能被滥用、造成干扰或侵扰。为缓解此类问题，且不完全禁用窗口调整大小 API，我们会让这些 API 在文档画中画窗口消耗用户手势。

等获得足够共识后并入 resizeTo()。

在 resizeTo() 第 3 步之后增加新步骤："如果 target 不是脚本创建（而不是用户行为）的 辅助浏览上下文，则返回。"：

  1. 如果 target 的 顶层可遍历对象 的 是否为文档画中画 布尔值为 true，则：

    1. 如果 this 的 相关全局对象没有 临时激活，抛出 "NotAllowedError" DOMException。

    2. 消耗用户激活，参数为 this 的 相关全局对象。

等获得足够共识后并入 resizeBy()。

在 resizeBy() 第 3 步之后增加新步骤："如果 target 不是脚本创建（而不是用户行为）的 辅助浏览上下文，则返回。"：

  1. 如果 target 的 顶层可遍历对象 的 是否为文档画中画 布尔值为 true，则：

    1. 如果 this 的 相关全局对象没有 临时激活，抛出 "NotAllowedError" DOMException。

    2. 消耗用户激活，参数为 this 的 相关全局对象。

6.9. 聚焦打开窗口

画中画窗口能重新聚焦其打开的标签页通常是有用的，比如窗口尺寸较小时无法满足用户体验。我们对 focus() API 做了修改，使其在画中画窗口聚焦打开方时可获得系统级焦点。

等获得足够共识后并入 focus()。

在 focus() 第 3 步后增加新步骤："对 current 运行 聚焦步骤"：

  1. 如果 current 是 顶层可遍历对象，则：

    1. 令 pipWindow 为 current 的 活动窗口的 documentPictureInPicture API 的 最后打开的窗口。

    2. 如果 pipWindow 不为 null 且 pipWindow 的 相关全局对象有 临时激活，则：

      1. 消耗用户激活，参数为 pipWindow 的 相关全局对象。

      2. 给予 current 系统焦点。

给予打开方系统焦点不一定要关闭文档画中画窗口。如果网站希望聚焦后立即关闭文档画中画窗口，可以直接对画中画窗口使用 close()。

6.10. CSS display-mode

CSS 的 display-mode 媒体查询功能 picture-in-picture 允许网页开发者编写仅在（部分）Web 应用处于画中画模式时才生效的 CSS 规则。

6.11. 用户激活传递

由于文档画中画窗口的特性，窗口内按钮事件处理器通常最终会在打开方的上下文中运行。这会导致网站调用消耗激活 API时体验不友好，因为有时文档画中画窗口拥有临时激活，打开方却没有。

为简化此流程，我们将更新 激活通知步骤，使文档画中画窗口触发用户激活时也会在打开方触发用户激活。此外当打开方获得用户激活时，也会激活画中画窗口内的同源 frame，类似于同源后代 frame 的激活。

等获得足够共识后并入 激活通知步骤。

在 激活通知第 4 步后增加三步："用 扩展 windows 列表，内容为 document 的每个 后代 navigables 的 活动窗口，仅包含navigables，其 活动文档 的 来源与 document 的 来源同源："：

  1. 如果 document 的 node navigable 的 顶层可遍历对象 的 是否为文档画中画布尔值为 true，则用 扩展 windows，内容为 document 的 node navigable的 顶层可遍历对象的 活动浏览上下文 的 打开方浏览上下文的 活动窗口。

  2. 令 document picture-in-picture window 为 document 的 node navigable 的 顶层可遍历对象的 活动窗口的 documentPictureInPicture API 的 最后打开的窗口。

  3. 如果 document picture-in-picture window 不为 null，则用 扩展 windows，内容为 document picture-in-picture window 的每个 关联文档 的 后代 navigables 的 活动窗口，仅包含 navigables，其 活动文档 的 来源与 document picture-in-picture window 的 关联文档 的 来源同源。

此外，还需确保此激活被正确消耗，避免在打开方与画中画窗口都被重复使用。我们通过在 消耗用户激活 步骤中，消耗打开方的用户激活（当消耗画中画窗口的激活时），以及消耗关联的画中画窗口激活（当消耗打开方激活时）。

等获得足够共识后并入 消耗用户激活 步骤。

在 消耗用户激活 步骤 3 后增加三步："令 navigables为 top 的 包含式后代 navigables列表，内容为 top 的 活动文档："：

  1. 如果 top 的 是否为文档画中画布尔值为 true，则用 扩展 navigables，内容为 top 的 活动浏览上下文 的 打开方浏览上下文的 活动文档 的包含式后代 navigables。

  2. 令 document picture-in-picture window 为 top 的 活动窗口的 documentPictureInPicture API 的 最后打开的窗口。

  3. 如果 document picture-in-picture window 不为 null，则用 扩展 navigables，内容为 document picture-in-picture window 的 关联文档的包含式后代 navigables。

7.1. 将视频播放器提取到画中画

7.1.1. HTML

<body>
  <div id="player-container">
    <div id="player">
      <video id="video" src="foo.webm"></video>
      <!-- 更多播放器元素。 -->
    </div>
  </div>
  <input type="button" onclick="enterPiP();" value="进入画中画" />
</body>

7.1.2. JavaScript

// 画中画窗口的句柄。
let pipWindow = null;

function enterPiP() {
  const player = document.querySelector('#player');

  // 设置宽高，使窗口大小适配视频。
  const pipOptions = {
    width: player.clientWidth,
    height: player.clientHeight,
  };

  documentPictureInPicture.requestWindow(pipOptions).then((pipWin) => {
    pipWindow = pipWin;

    // 剩余容器增加样式，表明播放器已在画中画内。
    playerContainer.classList.add('pip-mode');

    // 把播放器加入画中画窗口。
    pipWindow.document.body.append(player);

    // 监听关闭画中画事件，把视频放回主窗口。
    pipWindow.addEventListener('pagehide', onLeavePiP.bind(pipWindow), { once: true });
  });
}

// 画中画窗口关闭时调用。
function onLeavePiP() {
  if (this !== pipWindow) {
    return;
  }

  // 移除容器上的画中画样式。
  const playerContainer = document.querySelector('#player-container');
  playerContainer.classList.remove('pip-mode');

  // 把播放器放回主窗口。
  const player = pipWindow.document.querySelector('#player');
  playerContainer.append(player);

  pipWindow = null;
}

7.2. 访问画中画窗口的元素

const video = pipWindow.document.querySelector('#video');
video.loop = true;

7.3. 监听画中画窗口事件

为优化画中画体验，网站常需定制按钮和控件以响应如点击等用户输入事件。

const pipDocument = pipWindow.document;
const video = pipDocument.querySelector('#video');
const muteButton = pipDocument.document.createElement('button');
muteButton.textContent = '切换静音';
muteButton.addEventListener('click', () => {
  video.muted = !video.muted;
});
pipDocument.body.append(muteButton);

7.4. 退出画中画

网站可能希望无需用户点击窗口关闭按钮，即关闭 DocumentPictureInPicture 的 Window。可通过 close() 方法实现： 

// 这会关闭画中画窗口并触发已有的 onLeavePiP() 监听。
// 监听器。
pipWindow.close();

7.5. 画中画窗口关闭时取出元素

无论是网站主动关闭还是用户关闭画中画窗口时，网站常需把元素从窗口中取出。可在 pagehide 事件的处理器内实现（见上面视频播放器示例里的 onLeavePiP()，如下所示）：

// 画中画窗口关闭时调用。
function onLeavePiP() {
  if (this !== pipWindow) {
    return;
  }

  // 去除容器上的画中画样式。
  const playerContainer = document.querySelector('#player-container');
  playerContainer.classList.remove('pip-mode');

  // 把播放器放回主窗口。
  const player = pipWindow.document.querySelector('#player');
  playerContainer.append(player);

  pipWindow = null;
}

7.6. 以编程方式调整画中画窗口大小

文档画中画窗口支持 resizeTo() 和 resizeBy() API，但仅可在画中画窗口用户手势下使用：

const expandButton = pipWindow.document.createElement('button');
expandButton.textContent = '放大画中画窗口';
expandButton.addEventListener('click', () => {
  // 将画中画窗口宽度增加20像素，高度增加30像素。
  pipWindow.resizeBy(20, 30);
});
pipWindow.document.body.append(expandButton);

7.7. 返回到打开的标签页

可用 focus() API 从画中画窗口聚焦打开标签页（需用户手势）：

const returnToTabButton = pipWindow.document.createElement('button');
returnToTabButton.textContent = '返回原标签页';
returnToTabButton.addEventListener('click', () => {
  window.focus();
});
pipWindow.document.body.append(returnToTabButton);

7.8. CSS 画中画显示模式使用示例

下例展示如何在画中画窗口去除 body 元素边距，同时减小标题字体以更好适配画中画内内容：

@media all and (display-mode: picture-in-picture) {
  body {
    margin: 0;
  }
  h1 {
    font-size: 0.8em;
  }
}

7.9. 隐藏返回打开方按钮

虽然用户代理常在视频和文档画中画窗口显示返回打开方并关闭窗口的按钮，但对于部分网站的文档画中画体验该按钮不总是适用。可用 disallowReturnToOpener 选项隐藏按钮。

await documentPictureInPicture.requestWindow({
  disallowReturnToOpener: true
});

7.10. 优先初始窗口定位

文档画中画窗口打开时，用户可能手动调整大小或位置。若窗口关闭后再打开，用户代理可能用上次的位置与大小作为新窗口参考，而不是原始默认位置。

如果站点认为不应复用先前位置和大小（如为与前次无关的活动请求新画中画窗口时），可设置 preferInitialWindowPlacement 为 true 作为提示。用户代理可选择用默认位置、默认尺寸或站点提供的尺寸提示。

await documentPictureInPicture.requestWindow({
  preferInitialWindowPlacement: true
});
