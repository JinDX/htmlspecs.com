摘要

CBOR 是一种紧凑的二进制数据序列化和消息传递格式。本 规范定义了 CBOR-LD 1.0，一种基于 CBOR 的格式，用于序列化链接数据。 该编码旨在利用现有的 JSON-LD 生态系统，该生态系统如今已 部署在数亿个系统上，为寻求高效编码方案的用户提供一种紧凑的 序列化格式。通过利用语义压缩方案，可实现比 通用压缩方案高出 60% 以上的压缩率。该格式 主要旨在作为一种在存储和带宽受限的编程环境中使用链接数据的方式， 用于构建可互操作的语义线级协议， 并在基于 CBOR 的存储引擎中高效存储链接数据。

1.3 设计目标和理由

本节为非规范性内容。

CBOR-LD 满足以下设计目标：

简单性
在已有 JSON-LD 实现的情况下，实现应当易于实现。
高效存储
编码过程应生成极其紧凑的链接数据二进制 格式。
通用算法
编码算法必须是通用的。
语义压缩
编码格式应最大化压缩链接数据 URL（术语和 值）。聚焦于此可确保算法实现优于通用压缩算法的 压缩率。
原始二进制
在不牺牲通用性的前提下，基于基数编码的二进制值以及其他可压缩数据类型， 应在可能时从基于基数编码的格式 转换为其原始二进制形式。

同样，以下内容不是目标。

  • 理论上的纯粹性。
  • 专门为 CBOR-LD 创建新的压缩/扩展算法。
  • 为地球上的每一个术语建立大型集中式注册表。
  • 需要大量规范变更或复杂实现，才能 从 JSON-LD 到 CBOR-LD 再返回 JSON-LD 进行往返转换的数据格式。

在制定本规范的过程中，已经识别出以下雷区：

  • 虽然人们普遍认为哈希很快，但在小规模字符串集合上，它比字符串 比较更慢。这是因为为了进行哈希， 必须消费整个字符串。如果执行字符串比较， 可以在逐字节比较失败的瞬间中止。也就是说，在 处理两个值时，字符串比较可以中止处理，而哈希 不能。虽然大多数全局抗碰撞哈希都是 256 位，但 32 字节 值很大，并且在许多情况下我们可以做得更好，而不牺牲 抗碰撞性（像 blake2b-8 这样的哈希算法无法做到这一点）。
  • 任何要求预处理 JSON-LD 数据的做法都不可行—— 它成本高昂，而且这意味着你在数据中编码语义。 虽然这在某些情况下可以带来更好的压缩，但它会增加 处理器的复杂性，却无法提供可接受的 投资回报。
  • 作用域上下文是更通用解决方案中最具挑战性的方面。 算法越需要深入理解作用域， 问题就越具有组合爆炸性，因此可能的压缩值也越多，这会 损害压缩大小。诀窍是利用现有 JSON-LD 处理算法中的确定性，并且只有在不会影响 JSON-LD 处理时才走捷径。

2. 基本概念

本节为非规范性内容。

在高层次上，CBOR-LD 是 JSON-LD 的一种紧凑二进制序列化格式，允许 使用以下机制进行额外压缩：

  • 语义压缩：通过解析 JSON-LD 文档使用的 JSON-LD 上下文， 动态创建从 JSON-LD 术语到整数的映射。这些映射以 可逆方式创建。
  • 类型值压缩（编解码器）：基于对数据类型的先验理解， 以可逆方式动态压缩类型值（例如，移除 URL 中的 scheme 字符串）。
  • 类型值压缩（注册表字典）：创建静态字典，列在 CBOR-LD 注册表条目中，用于特定用例的类型值压缩。

编解码器是以通用方式压缩类型值的基本原语。语义压缩 用于压缩 JSON-LD 术语。注册表字典用于以 特定用例方式压缩类型值。其中每一种都是可选的——CBOR-LD 可与这些 压缩策略中的任意一种、全部或完全不使用一起使用。总体而言，为 特定用例使用的这些策略集合称为处理模型。

3. 语义压缩

本节为非规范性内容。

语义压缩是创建紧凑 CBOR-LD 载荷的强大工具。其核心思想是 使用外部 JSON-LD 上下文对象的信息内容来压缩 JSON-LD 术语。 这些外部上下文对象对载荷的创建者和消费者都可用， 因此这以可逆方式完成。

一般的语义压缩过程是取一个 JSON-LD 文档并执行 以下操作：

  1. 确定 JSON-LD 是否可以被压缩（在 JSON-LD 文档中查找通过 URI 引用上下文的 @context 值；嵌入式上下文值不能被压缩）。
  2. 处理 JSON-LD 上下文，构建 CBOR-LD 术语编解码器映射——一个 包含所有可压缩术语的列表。对列表排序（按 Unicode 码点顺序）。 将字节值与每个术语关联，从 0 开始递增。
  3. 将 JSON-LD 文档编码为 CBOR-LD。这包括将每个键替换为 与术语编解码器映射中的术语关联的字节值。
  4. 如果并且当需要原始 JSON-LD 时，从 CBOR-LD 解码到 JSON-LD 包括 将每个 CBOR 字节值替换为术语编解码器映射中关联的键。

4. 用于 CBOR-LD 的 CBOR 标签

CBOR-LD 载荷需要能在二进制层面被识别为此类载荷。CBOR 通过其“标签” 功能原生支持这一点——即二进制数据中的一个头部值， 通过全局注册表描述其余载荷。

CBOR 标签 0xCB1D（标签值 51997）已在 IANA CBOR 标签注册表中注册，用于 CBOR-LD。紧随该标签值之后的 数据标识用于创建压缩载荷的注册表条目。

CBOR-LD 载荷MUST被结构化，使得带有标签 0xCB1D 的项 是一个双元素数组，并且第一个元素MUST是一个 主类型 0 整数。 这个整数是 CBOR-LD 注册表条目 ID。其二进制结构为 tag([registryEntryId, payload])。

CBOR-LD 注册表条目 ID 的值随后用于在 CBOR-LD 注册表 中查找一个 CBOR-LD 注册表条目。

4.1 CBOR-LD 注册表

为了支持可能用例的无限扩展，这些用例需要不同的 压缩表材料以供消费，同时使用单个 CBOR 标签值，我们定义如下内容。

CBOR-LD 注册表是一个全局列表，为 CBOR-LD 载荷的消费者提供重建解压所需 术语编解码器映射所需要的信息。CBOR-LD 注册表条目包含以下内容：

  • Registry Entry ID：正整数。
  • Use Case：此条目要用于的 CBOR-LD 载荷类型。
  • typeTables：包含要与此 注册表条目一起使用的注册表字典的数组。
  • processingModel：此注册表条目使用什么处理模型。处理 模型 指定以下内容：
    • 是否使用语义压缩进行动态 JSON-LD 术语压缩，
    • 用于动态类型值压缩的编解码器。
    default 处理模型使用语义压缩，并包含 本文档中指定的编解码器。
  • provisional：一个布尔值，表示该条目是否为临时条目。临时条目 可能会更改或移除。

字符串 "callerProvidedTable" 可出现在 typeTables 中，表示对于此用 例， 需要一个未全局定义的 Type Table。

5. 算法

在本节中，我们规定将 JSON-LD 转换为 CBOR-LD 以及反向转换所需的算法。本节中的大多数算法都与语义 压缩功能有关，并且在 CBOR-LD 与 JSON-LD 之间进行转换时，如果 CBOR-LD 载荷未使用语义压缩，则不会使用这些算法。

5.1 编码与解码

5.1.1 JSON-LD 到 CBOR-LD 的编码算法

此算法以映射 typeTable、整数 registryEntryId 和 JSON-LD 文档 jsonldDocument 作为输入，并返回十六进制字符串 cborldBytes。

  1. 将 prefix 设为把 registryEntryId 传递给 5.6.1 获取 CBOR 标签结构算法所得的结果。
  2. 将 state 设为空映射。
  3. 如果与 registryEntryId 关联的注册表条目需要语义 压缩：
    1. 将 state.strategy 设为 "compression"。
    2. 将 state.typeTable 设为 typeTable。
    3. 将 state.registryEntryId 设为 registryEntryId。
    4. 将 state 设为把 state 传递给 5.2.1 初始化转换算法所得的结果。
    5. 将 output 设为把 state 和 jsonldDocument 作为 inputDocument 传递给 5.2.2 转换文档算法所得的结果。
    6. 将 suffix 设为 output 的 CBOR 编码。
  4. 否则，将 suffix 设为 jsonldDocument 的 CBOR 编码。
  5. 将 cborldBytes 设为在 suffix 前添加 prefix 后得到的 十六进制编码。
  6. 返回 cborldBytes。

5.1.2 CBOR-LD 到 JSON-LD 的解码算法

此算法以 CBOR-LD 载荷 cborldBytes 作为输入，并返回 JSON-LD 文档 jsonldDocument。

  1. 将 state 设为空映射。
  2. 将 result 设为把 cborldBytes 传给 5.6.2 获取 注册表条目 ID 算法的结果。
  3. 将 state.registryEntryId 设为 result.registryEntryId，并将 suffix 设为 result.suffix。
  4. 如果与 state.registryEntryId 关联的注册表条目使用 语义压缩：
    1. 将 state.strategy 设为 "decompression"。
    2. 对于与 registryEntryId 关联的 CBOR-LD Varint 注册表条目中 typeTables 数组里的每个条目 type: map， 将该条目添加到 state.typeTable，并将 state.reverseTypeTable 中 type 的值设为 inverseMap，其中 inverseMap 是将 map 的映射反转后的结果。
    3. 将 state 设为把 state 传给 5.2.1 初始化转换算法的结果。
    4. 将 input 设为把 suffix 从字节解码为 映射的结果。
    5. 将 jsonldDocument 设为把 state 和 input 作为 inputDocuments 传给 5.2.2 转换文档算法的结果。
  5. 否则，将 jsonldDocument 设为解析 cborldBytes 的结果。
  6. 返回 jsonldDocument。

5.2 转换算法

本节中的算法描述了“转换器”的行为，用于在数据形式之间进行抽象 转换。当与“策略”结合使用时， 例如本节后面定义的 "compression" 和 "decompression" 策略， 这些算法可以被实例化以在具体数据形式之间进行转换。"compression" 策略从 JSON-LD 转换为 CBOR-LD，而 "decompression" 策略从 CBOR-LD 转换为 JSON-LD。

5.2.1 初始化转换 算法

此算法接收并返回映射 state。

  1. 将 state 设为把 state 传递给 5.4.1 初始化上下文加载器算法所得的结果。
  2. 将 state.initialActiveContext 设为把空映射 termMap 和 previousActiveContext 传递给 5.3.1 初始化活动上下文算法所得的结果。
  3. 将 state.typesEncodedAsBytes 设为空集合。
  4. 将 "none"、"http://www.w3.org/2001/XMLSchema#date"、 "http://www.w3.org/2001/XMLSchema#dateTime" 和 "url" 添加到 state.typesEncodedAsBytes。
  5. 返回 state。

5.2.2 转换文档算法

此算法以映射 state 和映射或 映射数组 inputDocuments 作为输入，并返回一个包含映射 state 和映射或 映射数组 outputMaps 的映射。

  1. 如果 inputDocuments 是数组，则将 inputs 设为 inputDocuments。否则，将 inputs 设为 [inputDocuments]。
  2. 将 outputMaps 设为空数组。
  3. 对于 inputs 中的 input：
    1. 将 output 设为空映射。
    2. 将 result 设为把 state、input、 output 和 state.initialActiveContext 作为 activeContext 传递给 5.2.3 通用转换算法所得的结果。
    3. 将 result.output 添加到 outputMaps。
    4. 将 state 设为 result.state。
  4. 如果 inputDocuments 是数组，则返回 outputMaps。否则，返回 outputMaps 的 第一个元素。

5.2.3 通用转换 算法

此算法以映射 input、output、state 和 activeContext 作为输入，并返回一个包含映射 state 和 output 的映射。

  1. 如果 state.strategy 被设为 "compression"：
    1. 将 contextConversionResult 设为 5.2.5.1 为压缩转换上下文算法的结果， 并传入 state、activeContext、input 和 output。
    2. 将 activeContext 设为 contextConversionResult.activeContext， output 设为 contextConversionResult.output，并将 state 设为 contextConversionResult.state。
  2. 否则，将 activeContext 设为 result.activeContext，并将 state 设为 result 的 result.state，其中 result 来自 5.2.6.1 为解压缩转换上下文算法，并传入 state、 activeContext、input 和 output。
  3. 如果 state.strategy 被设为 "compression"，则将 state 设为 result.state，并将 objectTypes 设为 result.objectTypes，其中 result 来自 5.2.5.4 获取用于压缩的对象类型算法，并传入 state、 activeContext、input 和 output。
  4. 否则，将 state 设为 result.state，并将 objectTypes 设为 result.objectTypes， 其中 result 来自 5.2.6.4 获取用于解压缩的对象类型 算法， 并传入 state、activeContext、input 和 output。
  5. 将 activeContext 设为把 activeContext 和 objectTypes 传递给 5.3.4 应用类型作用域上下文算法所得的结果。
  6. 如果 state.strategy 被设为 "compression"，则将 state 设为 result.state，并将 termEntries 设为 result.termEntries，其中 result 来自 5.2.5.3 获取用于压缩的输入项算法，并传入 state、 input 和 activeContext。
  7. 否则，将 state 设为 result.state，将 output 设为 result.output， 并将 termEntries 设为 result.termEntries，其中 result 来自 5.2.6.3 获取用于解压缩的输入项算法， 并传入 state、input、output 和 activeContext。
  8. 对于 termEntries 中的 [termInfo, value]：
    1. 将 term 设为 termInfo.term。
    2. 将 valueActiveContext 设为把 activeContext 和 term 传递给 5.3.3 应用属性作用域上下文算法所得的结果。
    3. 将 plural 设为 termInfo.plural 的值，并将 termType 设为 termInfo.def 中 @type 的值。
    4. 如果 plural 被设为 true，则将 values 设为 value 的值。否则，将 values 设为 一个包含 value 的值作为单个元素的数组。
    5. 将 outputs 设为空数组。
    6. 对于 values 中的 unconvertedValue：
      1. 将 result 设为 5.2.4 转换值算法的结果， 并传入 state、termType、作为 value 的 unconvertedValue 以及作为 activeContext 的 valueActiveContext。
      2. 将 state 设为 result.state，并将 result.output 添加到 outputs。
    7. 如果 plural 被设为 true，则将 outputValues 设为 outputs。否则，将 outputValues 设为 outputs 的第一个元素。
    8. 如果 state.strategy 被设为 "compression"，则在 output 中将 termInfo.termId 的值映射到 outputValues。否则，在 output 中将 termInfo.term 的值映射到 outputValues。
  9. 将 result 设为空映射。
  10. 将 result.state 设为 state，并将 result.output 设为 output。
  11. 返回 result。

5.2.4 转换值算法

此算法以映射 state、activeContext、termInfo 以及 值 value 和 termType 作为输入。 它返回一个包含映射 state 和 output 的 result 对象。

  1. 如果 value 是 null，则返回 null。
  2. 如果 state.strategy 被设为 "compression"，则将 output 设为 把 state、termType、 termInfo 和 value 传递给 5.2.5.2 为压缩转换值算法所得的结果。
  3. 否则，将 output 设为把 state、 termType、 termInfo 和 value 传递给 5.2.6.2 为解压缩转换值算法所得的结果。
  4. 如果 output 已定义，则返回 result，即一个包含 state 和 output 的映射。
  5. 如果 value 是数组：
    1. 将 outputs 设为空数组。
    2. 对于 value 中的 element：
      1. 令 result 为 5.2.4 转换值算法的结果， 并传入 activeContext、state、 termInfo、termType 以及作为 value 的 element。将 state 设为 result.state，并将 result.output 添加到 outputs。
    3. 将 result 设为空映射。将 result.state 设为 state，并将 result.output 设为 outputs。
    4. 返回 result。
  6. 将 output 设为空映射。
  7. 将 result 设为 5.2.3 通用转换算法的结果，并传入 state、 activeContext、作为 input 的 value，以及 output。
  8. 返回 result。

5.2.5 压缩策略 算法

本节中的算法定义了 "compression" 策略， 该策略将与前面定义的“转换”算法一起使用， 以将 JSON-LD 转换为 CBOR-LD。

5.2.5.1 为压缩转换上下文算法

此算法以映射 state、activeContext、input 和 output 作为输入，并返回一个包含映射 output、state 和 activeContext 的映射 result。

  1. 将 applyEmbeddedResult 设为 5.3.2 应用嵌入式上下文算法的结果， 并传入 state、activeContext 和 input。
  2. 将 activeContext 设为 applyEmbeddedResult.activeContext，并将 state 设为 applyEmbeddedResult.state。
  3. 如果 "@context" 在 input 中没有条目：
    1. 将 result 设为空映射。
    2. 将 result.state 设为 state，并将 result.activeContext 设为 activeContext。
    3. 返回 result。
  4. 将 context 设为 input 中 "@context" 的值。
  5. 将 encodedContexts 设为空数组。
  6. 如果 context 是数组，则将 isArray 设为 true， 并将 contexts 设为 context。 否则，将 isArray 设为 false，并将 contexts 设为 [context]。
  7. 对于 contexts 中的 contextValue：
    1. 将 encoderData 设为 5.5.1.1 创建上下文编码器的结果， 并传入 state.typeTable 和 contextValue。
    2. 如果 encoderData 是空映射，则将 contextValue 添加到 encodedContexts。
    3. 否则，将 encoderData 的值添加到 encodedContexts。
  8. 如果 isArray 为 true，则将 id 设为 state.keywordsMap 中 "@context" 的值加 1， 并在 output 中将 id 的值设为 encodedContexts。
  9. 否则，将 id 设为 state.keywordsMap 中 "@context" 的值， 并在 output 中将 id 的值设为 encodedContexts 的第一个元素。
  10. 将 result.output 设为 output，将 result.state 设为 state，并将 result.activeContext 设为 activeContext。
  11. 返回 result。
5.2.5.2 为压缩转换值算法

此算法以映射 state 和 termInfo，以及值 valueToEncode 和 termType 作为输入，并返回映射 encoderData。

  1. 如果 valueToEncode 是对象，则返回。
  2. 否则，将 result 设为 5.5.2.1 创建值编码器的结果，并传入 state、termInfo、valueToToEncode 和 termType。
  3. 返回 result。
5.2.5.3 获取用于压缩的输入项算法

此算法以映射 state、activeContext 和 input 作为输入，并返回一个映射 state 和一个数组 entries。

  1. 将 entries 初始化为空数组。
  2. 将数组 keys 设为 input 的键，并按字典序排序。
  3. 对于 keys 中的 key：
    1. 如果 key 是 "@context"，则继续。
    2. 将 value 设为 input 中 key 的值。
    3. 如果 value 是数组，则将 plural 设为 true。 否则，将 plural 设为 false。
    4. 如果 key 在 state.termToId 中没有条目，则将 termId 设为 key。
    5. 否则，如果 plural 为 true，则将 termId 设为 state.termToId 中 key 的值加 1。
    6. 否则，将 termId 设为 state.termToId 中 key 的值。
    7. 如果 activeContext.termMap 有 key 的条目，则将 definition 设为 activeContext.termMap 中 key 的值。否则，将 definition 设为空映射。
    8. 将 entryTerm 设为一个新映射。
    9. 将 entryTerm 中 "term" 的值设为 key 的值。将 termId、plural 和 definition 添加到 entryTerm。
    10. 创建一个包含两个元素的数组 entry，即 entryTerm 和 value。
    11. 将 entry 添加到 entries。
  4. 返回包含 entries 和 state 的映射 result。
5.2.5.4 获取用于压缩的对象类型算法

此算法以映射 activeContext 和 input 作为输入，并返回集合 objectTypes。

  1. 将 objectTypes 设为空集合。
  2. 对于 activeContext.typeTerms 中的 term：
    1. 如果 term 在 input 中有条目：
      1. 将 types 设为 input 中 term 的值。
      2. 将 types 中的每个值添加到 objectTypes。
  3. 返回 objectTypes。

5.2.6 解压缩 策略算法

本节中的算法定义了 "decompression" 策略， 该策略将与前面定义的“转换”算法一起使用， 以将 CBOR-LD 转换为 JSON-LD。

5.2.6.1 为解压缩转换上下文算法

此算法以映射 state、activeContext、input 和 output 作为输入，并返回一个 包含映射 output、state 和 activeContext 的映射 result。

  1. 将 decoderData 设为 5.5.1.3 创建上下文解码器的结果，并传入 state.reverseTypeTable。
  2. 将 contextTermId 设为 state.keywordsMap 中 "@context" 的值。
  3. 如果 contextTermId 在 input 中有条目，则将 output 中 "@context" 的值设为 5.5.1.4 解码上下文的结果，并传入 decoderData 以及 input 中 contextTermId 的值作为 value。
  4. 将 contextTermIdPlural 设为 contextTermId 的值加 1。
  5. 如果 contextTermIdPlural 在 input 中有条目：
    1. 如果 contextTermId 在前一次检查期间也已经在 input 中有条目，则抛出 ERR_INVALID_ENCODED_CONTEXT 错误。
    2. 将 encodedContexts 设为 input 中 contextTermIdPlural 的值。如果 encodedContexts 不是数组，则抛出 ERR_INVALID_ENCODED_CONTEXT 错误。
    3. 将 contexts 设为空数组。
    4. 对于 encodedContexts 中的每个 valueToDecode， 将把 decoderData 和作为 value 的 valueToDecode 传递给 5.5.1.4 解码上下文所得的结果添加到 contexts。
    5. 将 output 中 "@context" 的值设为 contexts。
  6. 将 embeddedContextResult 设为 5.3.2 应用嵌入式上下文算法的结果，并传入 activeContext、作为 input 的 output，以及 state。
  7. 将 result 设为空映射。
  8. 将 result.state 设为 embeddedContextResult.state，并将 result.activeContext 设为 embeddedContextResult.activeContext。
  9. 返回 result。
5.2.6.2 为解压缩转换值算法

此算法以映射 state 和 termInfo，以及值 termType 和 valueToDecode 作为输入，并返回值 decodedValue。

  1. 如果 value 是映射，则返回。
  2. 将 decoderData 设为 5.5.2.3 创建值解码器的结果，并传入 valueToDecode、 state、termInfo 和 termType。
  3. 将 decodedValue 设为 5.5.2.4 解码值的结果，并传入 decoderData。
  4. 返回 decodedValue。
5.2.6.3 获取用于解压缩的输入项算法

此算法以映射 state、activeContext 和 input 作为输入，并返回一个映射 state 和一个数组 entries。

  1. 将 entries 初始化为空数组。
  2. 对于 input 中的键值对 key 和 value：
    1. 如果 key 是 state.keywordsMap 中 "@context" 的值，或该值加 1，则继续。
    2. 否则，如果 key 是字符串，则将 plural 设为 false，并将 term 设为 key。
    3. 否则：
      1. 如果 key 是奇数，则将 plural 设为 true。否则，将 plural 设为 false。
      2. 如果 plural 为 true，则将 term 设为 state.idToTerm 中 id 减 1 的值。 如果该值没有条目，则抛出错误 ERR_UNKNOWN_CBORLD_TERM_ID。
      3. 否则，将 term 设为 state.idToTerm 中 id 的值。如果该值没有 条目，则抛出错误 ERR_UNKNOWN_CBORLD_TERM_ID。
    4. 将 definition 设为 activeContext.termMap 中 term 的值。
    5. 将 entryTerm 设为一个新映射。
    6. 将 entryTerm 中 "termId" 的值设为 key 的值。将 term、plural 和 definition 添加到 entryTerm。
    7. 创建一个包含两个元素的数组 entry，即 entryTerm 和 value。
    8. 将 entry 添加到 entries。
  3. 按 entries 中每个元素里的 term 值对 entries 排序。
  4. 返回包含 entries 和 state 的映射 result。
5.2.6.4 获取用于解压缩的对象类型算法

此算法以映射 state、activeContext、input 作为输入，并 返回一个映射 state 和一个集合 objectTypes。

  1. 将 objectTypes 设为空集合。
  2. 对于 activeContext.typeTerms 中的 term：
    1. 如果 term 在 state.termToId 中没有条目，则将 termId 设为 term。
    2. 否则，将 termId 设为 state.termToId 中 term 的值。
    3. 如果 termId 和 termId 加 1 都不存在于 input 中，则继续。
    4. 否则，如果 termId 存在于 input 中，则将 value 设为 input 中 termId 的值。
    5. 否则，将 value 设为 input 中 termId 加 1 的值。
    6. 如果 key 是字符串，则将 plural 设为 false，并将 term 设为 key。
    7. 否则：
      1. 如果 key 是奇数，则将 plural 设为 true。否则，将 plural 设为 false。
      2. 如果 plural 为 true，则将 term 设为 state.idToTerm 中 id 减 1 的值。 如果该值没有条目，则抛出错误 ERR_UNKNOWN_CBORLD_TERM_ID。
      3. 否则，将 term 设为 state.idToTerm 中 id 的值。如果该值没有 条目，则抛出错误 ERR_UNKNOWN_CBORLD_TERM_ID。
    8. 将 definition 设为 activeContext.termMap 中 term 的值。
    9. 将 termInfo 设为一个新映射。
    10. 将 term、termId、plural 和 definition 添加到 termInfo。
    11. 如果 value 不是数组，则将 values 设为一个 包含 value 作为单个元素的数组。 否则，将 values 设为 value 的值。
    12. 对于 values 中的每个 value：
      1. 将 decoderData 设为 5.5.2.3 创建值解码器的结果，并传入 value、 termInfo、state 以及作为 termType 的 "@vocab"。
      2. 如果 decoderData 存在，则将把 5.5.2.4 解码值的结果添加到 `objectTypes，其中传入 decoderData。
      3. 否则，将 value 添加到 objectTypes。
  3. 返回 objectTypes。

5.3 活动上下文处理

本节中的算法描述如何确定 与 JSON-LD 文档关联的上下文文档中的哪些组件 在压缩或解压缩期间的任意时刻正在被使用。 这些算法包括如何在 CBOR-LD 中应用嵌入式、类型作用域和 属性作用域上下文。这与本规范稍后定义的上下文 加载算法形成对比，后者描述如何 构造从术语到整数的映射，这些映射是 CBOR-LD 压缩技术的核心。活动上下文处理和上下文 加载算法共同规定了在与 CBOR-LD 之间转换时应如何处理 JSON-LD 上下文文档。

5.3.1 初始化活动 上下文算法

此算法以映射 previousActiveContext 和 termMap 作为输入，并返回映射 activeContext。它会更新正在使用的 活动上下文，并查找 '@type' 的所有别名。

  1. 将 activeContext 设为一个新映射。
  2. 将 activeContext.previousActiveContext 设为 previousActiveContext。
  3. 将 activeContext.termMap 设为 termMap。
  4. 将 activeContext.typeTerms 设为数组 ['@type']。
  5. 对于 termMap 中的 [term, def]：
    1. 如果 def 中 "@id" 的值是 "@type"，则将 term 添加到 activeContext.typeTerms。
  6. 返回 activeContext。

5.3.2 应用嵌入式 上下文算法

此算法以映射 state、activeContext 和 input 作为输入，并 返回一个包含映射 state 和 activeContext 的映射 result。

  1. 将 termMapUpdateResult 设为把 state、 作为 activeTermMap 的 activeContext.termMap，以及 input 中 '@context' 的值作为 contexts 传递给 5.3.5 更新术语 映射算法所得的结果。
  2. 将 state 设为 termMapUpdateResult.state。
  3. 将 termMap 设为 termMapUpdateResult.activeTermMap。
  4. 将 newActiveContext 设为 5.3.1 初始化活动上下文算法的结果， 并传入 termMap 以及作为 previousActiveContext 的 activeContext。
  5. 将 result 设为一个新映射，并将 result.activeContext 设为 newActiveContext，并将 result.state 设为 state。
  6. 返回 result。

5.3.3 应用 属性作用域上下文算法

此算法以映射 state、activeContext 和字符串 term 作为输入，并 返回一个包含映射 state 和 activeContext 的映射 result。

  1. 将 revertedTermMap 设为 5.3.6 还原术语映射算法的结果， 并传入 activeContext。
  2. 将 termDef 设为 activeContext.termMap 中 term 的值。将 contexts 设为 termDef 中 "@context" 的值。
  3. 将 termMapUpdateResult 设为把 state、 作为 activeTermMap 的 revertedTermMap，作为 propertyScope 的 true，以及 contexts 传递给 5.3.5 更新术语 映射算法所得的结果。
  4. 将 state 设为 termMapUpdateResult.state。
  5. 将 termMap 设为 termMapUpdateResult.activeTermMap。
  6. 将 newActiveContext 设为 5.3.1 初始化活动上下文算法的结果， 并传入 termMap 以及作为 previousActiveContext 的 activeContext。
  7. 将 result 设为一个新映射，并将 result.activeContext 设为 newActiveContext，并将 result.state 设为 state。
  8. 返回 result。

5.3.4 应用类型 作用域上下文算法

此算法以映射 state、activeContext 和集合 objectTypes 作为输入，并 返回一个包含映射 state 和 activeContext 的映射 result。

  1. 将 objectTypesSorted 设为空数组。
  2. 按字典序排序 objectTypes 的元素，并按顺序将这些元素添加到 objectTypesSorted。
  3. 将 newTermMap 设为 activeContext.termMap。
  4. 对于 objectTypesSorted 中的 type：
    1. 将 typeDef 设为 newTermMap 中 type 的值。 将 contexts 设为 typeDef 中 "@context" 的值。
    2. 将 termMapUpdateResult 设为把 state、 作为 activeTermMap 的 newTermMap、contexts 以及作为 typeScope 的 true 传递给 5.3.5 更新术语映射算法所得的结果。
    3. 将 state 设为 termMapUpdateResult.state，并将 newTermMap 设为 termMapUpdateResult.activeTermMap。
  5. 将 newActiveContext 设为 5.3.1 初始化活动上下文算法的结果， 并传入作为 termMap 的 newTermMap 以及作为 previousActiveContext 的 activeContext。
  6. 将 result 设为一个新映射，并将 result.activeContext 设为 newActiveContext，并将 result.state 设为 state。
  7. 返回 result。

5.3.5 更新术语映射算法

此算法以映射 state、activeTermMap 和映射或数组 contexts，以及布尔值 typeScope 和 propertyScope 作为输入；若未提供，这两个布尔值都默认为 false。它返回映射 state 和 activeTermMap。

  1. 如果 contexts 不是数组，则将 contexts 设为一个数组，其唯一元素为 contexts 之前的值。
  2. 将 allowProtectedOverride 设为 propertyScope 的值。
  3. 将 propagateDefault 设为 typeScope 的值的否定。
  4. 对于 contexts 中的 contextIdentifier：
    1. 将 loadResult 设为 5.4.2 加载上下文算法的结果，并传入 state 和 contextIdentifier。
    2. 将 entry 设为 loadResult.entry， 将 context 设为 entry.context，并将 state 设为 loadResult.state。
    3. 如果 @propagate 出现在 context 中，则将 propagate 设为 context 中 @propagate 的值。 否则，将 propagate 设为 propagateDefault 的值。
    4. 将 newTermMap 设为空映射。对于 entry.termMap 中的 [key, value]：
      1. 将 value 的内容浅拷贝到一个新映射 newValue 中，并将 propagate 添加到 newValue。
      2. 将 newTermMap 中 key 的值设为 newValue。
    5. 对于 activeTermMap 中的 [term, activeDef]：
      1. 令 def 为 newTermMap 中 term 的值。
      2. 如果 def 已定义：
        1. 如果 activeDef 中 protected 的值为 true：
          1. 如果 allowProtectedOverride 被设为 false，并且 def 与 activeDef 不完全相同， 则抛出错误 ERR_PROTECTED_TERM_REDEFINITION。
          2. 否则，将 newTermMap 中 term 的值设为一个映射，该映射包含来自 activeDef 的值，并将 propagate 设为 def.propagate 的值。
      3. 否则，如果 term 出现在 context 中，则将 newTermMap 中 term 的值设为 一个包含来自 activeDef 的所有值的映射。
    6. 将 activeTermMap 的值设为 newTermMap 的值。
  5. 将 result 设为空映射。
  6. 将 result.state 设为 state，并将 result.activeTermMap 设为 activeTermMap。
  7. 返回 result。

5.3.6 还原术语映射算法

此算法以映射 activeContext 作为输入，并返回映射 newTermMap。

  1. 将 newTermMap 设为空映射。
  2. 将 nonPropagatingTerms 设为空数组。
  3. 对于 activeContext 中的 [term, def]：
    1. 如果 def.propagate 被设为 false，则将 term 添加到 nonPropagatingTerms， 并进入此循环的下一次迭代。
    2. 否则，将 newTermMap 中 term 的值设为 def。
  4. 对于 nonPropagatingTerms 中的 term：
    1. 将 currentContext 设为 activeContext.previousActiveContext。
    2. 将 def 设为 currentContext.termMap 中 term 的值。
    3. 当 def 未定义且 def.propagate 被设为 false 时：
      1. 将 currentContext 设为 activeContext.previousActiveContext。
      2. 将 def 设为 currentContext.termMap 中 term 的值。
    4. 如果 def 未定义，则将 newTermMap 中 term 的值设为 def。
  5. 返回 newTermMap。

5.4 上下文加载

本节中的算法定义如何构造 术语与整数之间的映射，这些映射被用作 CBOR-LD 压缩技术的核心。

5.4.1 初始化 上下文加载器算法

此算法接收并返回映射 state。

  1. 将 state.contextMap 设为一个新映射。
  2. 将 state.nextTermId 设为 100。
  3. 将 state.keywordsMap 设为以下 JSON-LD 关键词到其关联 整数值的映射： 
    {
  '@context' => 0,
  '@type' => 2,
  '@id' => 4,
  '@value' => 6,
  '@direction' => 8,
  '@graph' => 10,
  '@included' => 12,
  '@index' => 14,
  '@json' => 16,
  '@language' => 18,
  '@list' => 20,
  '@nest' => 22,
  '@reverse' => 24,
  '@base' => 26,
  '@container' => 28,
  '@default' => 30,
  '@embed' => 32,
  '@explicit' => 34,
  '@none' => 36,
  '@omitDefault' => 38,
  '@prefix' => 40,
  '@preserve' => 42,
  '@protected' => 44,
  '@requireAll' => 46,
  '@set' => 48,
  '@version' => 50,
  '@vocab' => 52,
  '@propagate' => 54
}
  4. 将 state.keywordsMap 中的每个条目添加到 state.termToId。
  5. 如果 state.strategy 被设为 "decompression"，则将 state.idToTerm 设为 state.termToId 的反向映射 （即从整数到 JSON-LD 关键词的映射）。
  6. 返回 state。

5.4.2 加载上下文算法

此算法以映射 state 和上下文映射或 URL contextIdentifier 作为输入，并返回 result，即一个包含映射 state 和 entry 的映射。

  1. 如果 state.contextMap 有 contextIdentifier 的条目：
    1. 将 result 初始化为空映射。
    2. 将 result.state 设为 state。
    3. 将 result.entry 设为 state.contextMap 中 contextIdentifier 的值。
    4. 返回 result。
  2. 如果 context 是字符串：
    1. 获取关联的上下文对象，并将 context 设为 该对象中 "@context" 的值。
    2. 将 contextUrl 设为 contextIdentifier 的值。
  3. 否则，将 context 设为 contextIdentifier。
  4. 将 result 设为 5.4.3 添加上下文算法的结果，并传入 state、context，以及如果已设置则传入 contextUrl。
  5. 返回 result。

5.4.3 添加上下文算法

此算法以映射 state、上下文对象 context 和上下文 URL contextUrl 作为输入，并返回 result，即一个包含映射 state 和 entry 的映射。

  1. 如果 context 有条目 "@import"：
    1. 将 importUrl 设为 `context 中 "@import" 的值。
    2. 如果 state.contextMap 没有 importUrl 的条目：
      1. 获取与 importUrl 关联的上下文对象，并将 importContext 设为该对象中 "@context" 的值。
      2. 将 importedContextAdditionResult 设为 5.4.3 添加上下文算法的结果， 并传入 state、作为 context 的 importContext， 以及作为 contextUrl 的 importUrl。
      3. 将 state 设为 importedContextAdditionResult.state，并将 importEntry 设为 importedContextAdditionResult.entry。
    3. 否则，将 importEntry 设为 state.contextMap 中 importUrl 的值。
    4. 将 context 设为一个包含来自 context 的所有条目以及 importEntry.context 的映射。
  2. 将 termMap 设为空映射。
  3. 将 entry 设为一个包含 context 和 termMap 的对象。
  4. 将 sortedTerms 设为按字典序排序 context 中键所得的结果。
  5. 如果 "@protected" 在 context 中有条目，则将 isProtected 设为 true，否则设为 false。
  6. 对于 sortedTerms 中的 term：
    1. 如果 term 在 state.keywordsMap 中有条目，则进入此循环的下一次 迭代。
    2. 将 definition 设为 context 中 term 的值。
    3. 如果 definition 是 null，则进入此循环的下一次 迭代。
    4. 如果 definition 是字符串：
      1. 将 newDefinition 设为空映射。
      2. 将 newDefinition 中 "@id" 的值设为 definition。
      3. 将 definition 的值设为 newDefinition。
    5. 将 definition 中 protected 的值设为 isProtected。
    6. 将 termMap 中 term 的值设为 definition。
    7. 如果 term 在 state.termToId 中没有条目：
      1. 将 termId 设为 state.nextTermId。
      2. 将 state.nextTermId 增加 2。
      3. 将 state.termToId 中 term 的值设为 termId。
      4. 将 state.idToTerm 中 termId 的值设为 term。
  7. 如果 contextUrl 已定义，则将 state.contextMap 中 contextUrl 的值设为 entry。
  8. 否则，将 state.contextMap 中 context 的值 设为 entry。
  9. 将 result 设为空映射。
  10. 将 result.state 设为 state，并将 result.entry 设为 entry。
  11. 返回 result。

5.5 编解码器

本节中的编解码器明确规定了 JSON-LD 中各个值应如何转换为 CBOR， 以及如何反向转换。它们被前一节中的算法使用，并使 CBOR-LD 能够 高效地 将原始类型和非原始类型编码为 CBOR。

5.5.1 上下文编解码器

5.5.1.1 创建上下文编码器

此算法以映射 typeTable 和值 contextValue 作为输入，并 返回映射 encoderData。

  1. 将 encoderData 初始化为空映射。
  2. 如果 contextValue 不是字符串，则返回。
  3. 否则，将 contextTable 设为 typeTable 中 "context" 的值。
  4. 将 encoderData.context 设为 contextValue，并将 encoderData.contextTable 设为 contextTable。
  5. 返回 encoderData。
5.5.1.2 编码上下文

此算法以映射 encoderData 作为输入，并返回 CBOR 二进制数据。

  1. 如果 encoderData.context 在 encoderData.contextTable 中有条目，则返回 encoderData 中 encoderData.context 的值作为 主类型 0（无符号整数）对象的 CBOR 编码。
  2. 否则，返回 encoderData.context 的值作为主类型 3 （文本字符串）对象的 CBOR 编码。
5.5.1.3 创建上下文解码器

此算法以映射 reverseTypeTable 作为输入，并 返回映射 encoderData。

  1. 将 reverseContextTable 设为 reverseTypeTable 中 "context" 的值。
  2. 将 decoderData 初始化为空映射。
  3. 将 decoderData.reverseContextTable 设为 reverseContextTable 的值，并 返回 decoderData。
5.5.1.4 解码上下文

此算法以映射 decoderData 和值 value 作为输入，并返回 一个值。

  1. 如果 value 不是数字，则返回 value。
  2. 否则，如果 decoderData.reverseContextTable 有 value 的条目，则返回 该条目的值。
  3. 否则，抛出错误 ERR_UNDEFINED_COMPRESSED_CONTEXT。

5.5.2 值编解码器

5.5.2.1 创建值编码器

此算法以映射 state 和 termInfo，以及值 termType 和 valueToEncode 作为输入，并返回映射 encoderData 或 valueToEncode。

  1. 将 isUrl 设为 false。
  2. 如果 termInfo.term 是 "@id" 或 "@type"，则将 isUrl 设为 true。
  3. 如果 termInfo.def 中 "@id" 的值是 "@id" 或 "@type"，则将 isUrl 设为 true。
  4. 如果 termType 是 "@id" 或 "@vocab"，则将 isUrl 设为 true。
  5. 如果 isUrl 为 true，则将 tableType 设为 "url"。
  6. 否则，如果 termType 已定义，则将 tableType 设为 termType。
  7. 否则，将 tableType 设为 "none"。
  8. 如果 state.typeTable 有 tableType 的条目：
    1. 将 subTable 设为 state.typeTable 中 tableType 的值。
    2. 如果 subTable 有 valueToEncode 的条目：
      1. 将 intValue 设为 subTable 中 valueToEncode 的值。 将 includeSign 设为 false。
      2. 如果 state.typesEncodedAsBytes 有 tableType 的条目，则将 convertToBytes 设为 true。 否则，将 convertToBytes 设为 false。
    3. 否则，如果 tableType 不是 "none" 且 valueToEncode 是 整数：
      1. 将 intValue 设为 valueToEncode 的值。
      2. 将 convertToBytes 和 includeSign 设为 true。
    4. 如果 intValue 已定义：
      1. 将 encoderData 初始化为空映射。
      2. 将 encoderData.intValue 设为 intValue 的值，将 encoderData.convertToBytes 设为 convertToBytes 的值，并将 encoderData.includeSign 设为 includeSign 的值。
      3. 返回 encoderData。
  9. 如果 tableType 在 state.processingModeTypeEncoders 中有条目，则将 encoderData 设为 调用与该条目的编解码器关联的 Create Encoder 算法所得的结果。
  10. 如果 encoderData 已定义，则返回 encoderData。
  11. 返回 valueToEncode。
5.5.2.2 编码值

此算法以映射 encoderData 作为输入，并返回 CBOR 二进制数据。

  1. 如果 encoderData.convertToBytes 为 true：
    1. 将 bytes 设为把 intValue 转换为字节所得的结果， 使用 includeSign 的值来 确定该整数的二进制表示应为有符号还是 无符号。
    2. 返回 bytes 作为主类型 2（字节字符串）对象的 CBOR 编码。
  2. 否则，返回 intValue 作为主类型 0（无符号 整数）对象的 CBOR 编码。
5.5.2.3 创建值解码器

此算法以映射 state 和 termInfo，以及值 termType 和 valueToDecode 作为输入，并返回映射 decoderData。

  1. 将 isUrl 设为 false。
  2. 如果 termInfo.term 是 "@id" 或 "@type"，则将 isUrl 设为 true。
  3. 如果 termInfo.def 中 "@id" 的值是 "@id" 或 "@type"，则将 isUrl 设为 true。
  4. 如果 termType 是 "@id" 或 "@vocab"，则将 isUrl 设为 true。
  5. 如果 isUrl 为 true，则将 tableType 设为 "url"。
  6. 否则，如果 termType 已定义，则将 tableType 设为 termType。
  7. 否则，将 tableType 设为 "none"。
  8. 如果 state.reverseTypeTable 有 tableType 的条目：
    1. 将 subTable 设为 state.reverseTypeTable 中 tableType 的值。
    2. 将 useTable 设为 false。
    3. 如果 valueToDecode 是字节数组，并且 state.typesEncodedAsBytes 有 tableType 的条目：
      1. 将 useTable 设为 true。
      2. 将 intValue 设为 valueToDecode 字节的无符号整数转换结果。
    4. 否则，如果 valueToDecode 是整数，并且 state.typesEncodedAsBytes 没有 tableType 的条目：
      1. 将 useTable 设为 true。
      2. 将 intValue 设为 valueToDecode。
    5. 如果 useTable 为 true：
      1. 如果 intValue 不在 subTable 中，则抛出错误 ERR_UNKNOWN_COMPRESSED_VALUE。
      2. 否则，将 decoded 设为 subTable 中 intValue 的值。
    6. 否则，如果 valueToDecode 是字节数组且 tableType 不是 "none"，则将 decoded 设为 valueToDecode 的整数转换结果。
    7. 如果 decoded 已定义，则将 decoderData 初始化为空 映射，将 decoderData.decoded 设为 decoded 的值，并返回 decoderData。
  9. 如果 tableType 在 state.processingModeTypeDecoders 中有条目，则将 DecoderData 设为 调用与该条目的编解码器关联的 Create Decoder 算法所得的结果。
  10. 如果 decoderData 已定义，则返回 decoderData。
  11. 否则，如果 valueToDecode 不是数组，则初始化 decoderData 为空映射，将 decoderData.decoded 设为 valueToDecode，并返回 decoderData。
5.5.2.4 解码值

此算法以映射 decoderData 作为输入，并返回一个值。

  1. 返回 decoderData.decoded。

5.6 CBOR 标签处理

5.6.1 获取 CBOR 标签 结构算法

此算法以整数 registryEntryId 作为输入，并返回字节字符串 prefix。
  1. 将 registryEntryBytes 设为 registryEntryID 的 CBOR 编码
  2. 将 prefix 设为把 registryEntryBytes 追加到 字节字符串 0xD9CB1D82 末尾所得的结果。
  3. 返回 prefix。

5.6.2 获取注册表条目 ID 算法

此算法以已编码的 CBOR-LD 载荷 cborldBytes 作为输入，并返回 suffix（要解码的主要数据）以及 应用于解压缩 suffix 的 registryEntryId 值。

  1. 如果 cborldbytes 上的 CBOR 标签不是 0xCB1D（标签值 51997）， 则抛出 ERR_NON_CBOR_LD_TAG 错误。
  2. 如果被标记的项不是一个包含两个元素且第一个元素是 CBOR 整数的数组， 则抛出 ERR_INVALID_PAYLOAD_STRUCTURE 错误。
  3. 将 registryEntryId 的值设为被标记 数组索引 0 处整数的值。
  4. 将 suffix 的值设为该数组第二个元素的值。
  5. 将 result 设为空映射。
  6. 将 result.suffix 设为 suffix 的值，并将 result.registryEntryId 设为 registryEntryId 的值。
  7. 返回 result。
