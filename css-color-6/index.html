1. 简介

本节不是规范性的。

本模块添加了两个新的 <color> 函数： contrast-color() 和 color-layers()。

2. 计算对比颜色：contrast-color() 函数

contrast-color() 函数式记法 识别一种相对于指定背景色或前景色具有足够对比度的颜色， 而无需手动计算。

其语法为：

<contrast-color()> = contrast-color(
  [ [ <color> && [ tbd-fg | tbd-bg ] && <target-contrast>? ] |
    [ <color> && [ tbd-fg | tbd-bg ] && <target-contrast>, <color># ] ] )
<target-contrast> = <wcag2>
测试
  • contrast-color-001.html （实时测试） （源码）
  • color-computed-contrast-color-function.html （实时 测试） （源码）
  • color-invalid-contrast-color-function.html （实时 测试） （源码）
  • color-valid-contrast-color-function.html （实时 测试） （源码）

唯一必需的参数是用于计算对比度的基准颜色。 这通常（但不一定）是背景色。

可选的 <color> 值列表表示 候选颜色。 如果未提供候选颜色，则使用默认候选颜色：white, black。

tbd-fg 和 tbd-bg 关键字 表示在计算对比度时，基准颜色所承担的角色： 作为文本，与一组候选背景色进行对比（text）， 或作为背景，与一组候选文本 颜色进行对比。 （一对颜色的有效对比度取决于它们的用途。）

用于指定 基准 颜色是前景还是背景的关键字仍为 TBD。[议题 #7359]

<target-contrast> 参数指定要使用的对比度 算法。 如果尚未提供任何候选颜色， 则可以省略 <target-contrast>， 在这种情况下会使用 UA 选择的算法。

<target-contrast> 函数式记法的参数 表示目标 对比度级别。 可以指定多个具有目标对比度级别的对比算法， 在这种情况下，它们的所有要求会同时应用。 指定多个算法的 contrast-color() 函数 必须为每个算法指定一个目标对比度级别， 否则即为无效。

如果省略目标 对比度级别， 则返回对比度最大的候选颜色。 否则， 返回的颜色是第一个达到或超过该级别的候选颜色； 如果没有候选颜色符合条件，则默认返回 white 或 black。

2.1. 寻找胜出颜色

2.1.1. 如果存在目标对比度

候选颜色会按顺序测试， 从列表中的第一个颜色开始， 并以自动附加的 white, black 结束。 第一个通过与基准 颜色之间指定对比度级别的颜色胜出。

如果没有候选颜色通过，则在 white 和 black 中， 按照第一个指定算法具有最高对比度的那个胜出。 如果二者具有相同的对比度，则 white 胜出。 

contrast-color(wheat tbd-bg wcag2(aa), bisque, darkgoldenrod, olive, sienna, darkgreen, maroon)

计算如下：

  • wheat (#f5deb3)，背景色，相对 亮度为 0.749

  • bisque (#ffe4c4) 的相对亮度为 0.807，对比度为 1.073

  • darkgoldenrod (#b8860b) 的相对 亮度为 0.273，对比度为 2.477

  • olive (#808000) 的相对亮度为 0.200， 对比度为 3.193

  • sienna (#a0522d) 的相对亮度为 0.137，对比度为 4.274

  • darkgreen (#006400 ) 的相对 亮度为 0.091，对比度为 5.662

  • maroon (#800000) 的相对亮度为 0.046，对比度为 8.333

列表中第一个满足所需对比度 4.5 的颜色是 darkgreen。

contrast-color(wheat tbd-bg wcag2(5.8), bisque, darkgoldenrod, olive, sienna, darkgreen, maroon)

计算如下：

  • 相对亮度和对比度与前一个示例相同。

列表中第一个满足所需对比度 5.8 的颜色是 maroon。

contrast-color(wheat tbd-bg wcag2(AA), bisque, darkgoldenrod, olive)

计算如下：

  • 相对亮度和对比度与前一个示例相同。

列表中没有颜色满足所需对比度 4.5。 white 与 wheat 这个基准颜色之间的对比度 是 1.314，小于 4.5（AA）， 因而未通过目标对比度。 因此，返回 black，其对比度为 15.982 > 4.5。

2.1.2. 如果未指定目标对比度

候选颜色会按顺序测试， 从列表中的第一个颜色开始。 如果一个颜色在目前已测试的所有颜色中， 与基准颜色的对比度最高， 则它是临时胜出者。

到达列表末尾后，当前临时胜出者就是最终胜出者。 因此，如果列表中两个颜色碰巧具有相同的对比度， 较早出现的那个胜出。

--myAccent: #b22222;
color: contrast-color(wheat tbd-bg wcag2, tan, sienna, var(--myAccent), #d2691e)

计算如下：

  • wheat (#f5deb3)，背景色，相对 亮度为 0.749

  • tan (#d2b48c) 的相对亮度为 0.482， 对比度为 1.501

  • sienna (#a0522d) 的相对亮度为 0.137，对比度为 4.273

  • --myAccent (#b22222) 的相对亮度为 0.107，对比度为 5.081

  • #d2691e 的相对亮度为 0.305， 对比度为 2.249

最高对比度为 5.081，因此 var(--myAccent) 胜出

foo {
  --bg: hsl(200 50% 80%);
  --purple-in-hsl: hsl(300 100% 25%);
  color: contrast-color(var(--bg) tbd-fg wcag2, hsl(200 83% 23%), purple, var(--purple-in-hsl));
}

计算如下：

  • --bg 是 rgb(179 213 230)，其 相对亮度为 0.628835

  • hsl(200 83% 23%) 是 rgb(10 75 107)， 其相对亮度为 0.061575，对比度为 6.08409

  • purple 是 rgb(128 0 128)，其相对 亮度为 0.061487，对比度为 6.08889

  • --purple-in-hsl 也是 rgb(128 0 128)，其相对亮度为 0.061487，对比度为 6.08889。这并 不大于 purple 的对比度，因此 purple 胜出。

这里的计算值显示为六位有效数字，以说明过早舍入到较低精度 会给出错误结果（0.061575 非常接近 0.061487； 6.08409 非常接近 6.08889）。

2.2. 对比算法

目前仅支持 WCAG 2.1， 但是已知该算法存在问题， 尤其是在深色背景上。 本模块的未来修订可能会引入更多对比算法。

2.2.1. WCAG 2.1：wcag2 关键字和 wcag2() 函数

wcag2 关键字和 wcag2() 函数式记法 表示使用 [WCAG21] 亮度对比度算法。 其语法为：

<wcag2> =  wcag2 | wcag2([<number> | [ aa | aaa ] && large? ])

其目标对比度 级别关键字映射如下：

  • aa 计算为 4.5

  • aa large（或 large aa）计算为 3

  • aaa 计算为 7

  • aaa large（或 large aaa）计算为 4.5

要查找一对颜色的对比度， 首先相对于 D65 白点计算它们的 CIE 亮度（Y）。 然后计算 WCAG 2.1 对比度： contrast = (Yl + 0.05) / (Yd + 0.05) 其中 Yd 是这对颜色中较暗颜色的亮度， Yl 是较亮颜色的亮度。 因子 0.05 表示观察眩光的亮度贡献。

假设基准 颜色为 
color(display-p3 0.38 0.11 0.05)

而列表中的第一个候选颜色是

yellow

计算如下：

  • color(display-p3 0.38 0.11 0.05) 是 color(xyz 0.06191 0.03568 0.00463)，因此相对亮度为 0.03568

  • yellow 是 rgb(100% 100% 0%)，也就是 color(xyz 0.76998 0.92781 0.13853)，因此相对亮度 为 0.92781

  • 对比度为 (0.92781 + 0.05) / (0.03568 + 0.05) = 11.4123

2.2.2. 对半透明颜色进行对比

本节仅适用于要求输入为不透明颜色的对比算法。

计算一对颜色的对比度时：

  1. 首先将背景色在不透明画布颜色上混合， 使用简单 alpha 合成（参见 Compositing 1 § 5.1 简单 alpha 合成）， 以得到一个不透明背景色。

    #7358 为此计算寻找一个合适的不透明画布颜色。 候选项为：

    • alpha = 1 的前景色

    • canvas 系统颜色

    • 某种为所用算法最小化对比度而计算出的其他颜色

  2. 然后将前景色在这个不透明背景色上混合， 以得到一个不透明前景色。

3. 叠加多个颜色：color-layers() 函数

color-layers() 函数式记法 接受一个可选的 <blend-mode>， 后跟两个或更多 <color> 图层的列表。

其语法为：

<color-layers()> = color-layers([ <blend-mode>, ]? <color># )

4. 解析 <color> 值

4.1. 解析 contrast-color() 值

如果所有 <color> 参数都解析为其各自颜色空间中的对应颜色， 则计算值是胜出颜色， 按照 CSS Color 4 § 15. 解析 <color> 值进行解析。 否则（如果在函数中使用了 currentColor）， 计算值是 contrast-color() 函数， 其中每个组件都已计算， 从而保留向子元素的继承。

例如，给定值 
contrast-color(rgb(179 213 230) tbd-bg wcag2(AA), cadetblue, hsl(200 83% 23%))

与 cadetblue 的对比度为 1.97， 而与 hsl(200 83% 23%) 的对比度为 6.09， 超过了 AA 的值 4.5； 因此其计算值为已解析的 hsl 函数：

rgb(10 75 107)

例如，给定当前颜色值 rgb(179 213 230)， 值 

contrast-color(currentColor tbd-bg wcag2(AA), hsl(200 83% 23%), purple)

具有计算值

contrast-color(currentColor tbd-bg wcag2(4.5), rgb(10 75 107), rgb(128 0 128))

4.2. 解析 color-layers() 值

如果所有 <color> 参数都解析为其各自颜色空间中的对应颜色， 则计算值是合成后的颜色， 按照 CSS Color 4 § 15. 解析 <color> 值进行解析。 否则（如果在函数中使用了 currentColor）， 计算值是 color-layers() 函数， 带有一个显式的 <blend-mode>， 并且每个组件都已计算， 从而保留向子元素的继承。

9. 变更

9.1. 自 Colors 5 以来的变更

新的 contrast-color() 函数允许从颜色列表中选择一个颜色， 选择依据是它与指定颜色之间的 WCAG 2.1 对比度。 新的 color-layers() 函数返回 使用给定混合模式合成两个或更多颜色所得的结果。
