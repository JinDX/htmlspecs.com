摘要

本文档描述用户代理如何从 Web 内容语言中确定名称和 描述，这些名称和描述属于可访问 对象。 此信息又会通过 可访问性 API 公开，以便辅助技术能够识别这些对象并 向用户呈现 其名称或描述。记录用于确定名称和描述的算法，有助于促进不同 可访问性 API 之间对这些属性的可互操作公开，并有助于确保此 信息以符合作者意图的方式呈现。

可访问名称和描述计算规范定义了适用于多种 内容技术的支持。这包括由通用型 WAI-ARIA [WAI-ARIA] 角色、状态和属性提供的可访问名称和描述，以及 个别内容语言特有的特性。

本文档更新并最终将取代 可访问名称和描述计算 1.1 [ACCNAME-1.1] W3C 推荐标准中的可访问名称和描述指导。它是 WAI-ARIA 概述中所述 WAI-ARIA 套件的一部分。

1. 简介

本节是非规范性的。

用户 代理从 DOM [DOM] 获取信息，并 创建一个称为 可访问性树的并行结构，该结构由可访问对象组成。可访问 对象提供有关其角色、 状态和属性的信息。一个示例是角色为 menuitem、当前处于 enabled 状态，并带有 haspopup 属性的可访问对象，这表示它会通向一个子菜单。

本文档中描述的可访问对象的两个属性是其可访问名称和可访问描述。名称是一个简短标签， 它提供关于对象用途的信息。菜单项的一个可访问名称示例是 New，表示该菜单项用于创建新的 文档、窗口等。

描述是一个简短说明，用于进一步阐明可访问对象的性质。如果名称已经足够， 并不总是需要提供描述，但它可以 帮助用户更好地理解对象的用途。

可访问性 API 目前支持用于可访问名称和描述的扁平、非结构化 字符串。因此，名称/描述计算的结果 是一个扁平字符串。

使用“可访问名称”和“可访问描述”这些术语，是为了强调它们是 可访问对象的属性，这些对象由 可访问性 API公开。不过，在下文中它们通常被简称为 “名称”和“描述”。

2. 重要术语

本节是非规范性的。

虽然有些术语是在使用处定义的，但以下定义会在本文档通篇使用。

辅助技术

满足以下条件的硬件和/或软件：

  • 依赖用户代理提供的服务来检索和渲染 Web 内容
  • 通过使用 API 与用户代理或 Web 内容本身协作，以及
  • 提供超出用户代理所提供服务的服务，以帮助残障人士与 Web 内容进行交互

此定义可能与其他文档中使用的定义不同。

在本文档的语境中重要的辅助技术示例包括 以下各项：

  • 屏幕放大器，用于放大并改善所渲染 文本和图像的视觉可读性；
  • 屏幕阅读器，最常用于通过合成语音 或可刷新盲文显示器传达信息；
  • 文本转语音软件，用于将文本转换为合成语音；
  • 语音识别软件，用于允许语音控制和听写；
  • 替代输入技术（包括头控指针、屏幕键盘、单开关 和吸吹设备），用于模拟键盘；
  • 替代指点设备，用于模拟鼠标指向和点击。
可访问描述

可访问描述提供与界面元素相关的附加信息，用于 补充可访问名称。可访问描述可能 也可能不会 在视觉上可感知。

可访问名称

可访问名称是用户界面元素的名称。每个平台可访问性 API都会提供可访问名称 属性。可访问名称的值可以派生自用户界面元素的可见属性（例如，按钮上的可见文本） 或不可见属性（例如，描述图标的文本替代） 。另见相关的可访问描述。

“OK”按钮可以说明可访问名称属性的一个简单用法。文本 “OK”就是可访问名称。当该按钮获得焦点时，辅助 技术可以将平台的角色描述与可访问名称拼接起来。例如， 屏幕阅读器可能朗读“push-button OK”或“OK button”。拼接的 顺序以及角色描述的具体内容（例如，“button”、“push-button”、 “clickable button”）由平台 可访问性 API 或辅助技术决定。

工具提示 属性

任何会导致用户代理生成工具提示的宿主语言属性，例如在桌面用户代理中 响应鼠标悬停时生成工具提示。

4. 名称和描述

名称和描述计算的起点是一个 DOM 元素。输出是一个扁平、非结构化的 字符串，它可以简单到只是一个 单词，也可以是一个由空格分隔的标记组成的字符串。示例包括 Save 和 Reload from disk。

一个重要因素是该元素的角色，它决定哪些 内容会贡献到名称字符串中。角色具有一个 nameFrom RDF 属性，其可能值有三个：

author
名称由作者在显式标记特性中提供的值生成，例如 aria-label 和 aria-labelledby 属性，或宿主语言的标注 机制，例如 HTML 中的 alt 或 title 属性， 或 SVG 中的 desc 元素。
contents
名称由与该文本节点关联的元素生成。虽然在某些角色中， 除了“author”之外也可能允许这种方式，但只有在未提供更高优先级的“author”特性时，才使用 “content”。优先级由 文本等价计算算法定义。
prohibited
该元素没有名称。作者MUST NOT 使用 aria-label 或 aria-labelledby 属性来命名该元素。

可访问富互联网 应用（WAI-ARIA）1.2 [WAI-ARIA] 规范 提供了 支持由 author 提供名称的角色、支持由 content 提供名称的角色以及 不能被 命名的角色列表。

4.3 文本等价计算

文本等价计算同时用于可访问 名称和可访问描述。针对若干 不同类型的元素、节点以及标记组合，提供了不同规则。适当时， 文本替代会由包含在 元素中的所有相关内容逐步构建起来。这是 通过递归的步骤 2B 和 2F 完成的，它们使用完整的一组规则，从 自身的子节点或其引用的节点中检索文本。

计算的目的是为替代表现形式创建一个可感知的标签或描述， 其形式是由空格分隔的文本 标记组成的扁平字符串。

4.3.1 术语

根节点
正在为其寻求文本 替代的 DOM 节点或元素。
当前节点
为计算 root node 的文本等价而当前遍历到的 DOM 节点。最初，current node 是 root node，但在后续阶段，它要么是 root node 的某个后代，要么是另一个被引用的节点。
渲染后的子节点
在考虑影子 根和插槽时，被渲染为给定节点的子节点的节点。
扁平字符串
一个字符字符串，其中所有回车、换行、制表符和换页符都被替换为 一个空格，并且多个空格会被压缩为一个空格。该字符串仅包含 字符数据；不包含任何标记。
总累计文本
截至但不包括 current node 时已经计算出的文本等价。
累计文本
在下文描述的某个步骤或一系列步骤中累计的文本。它是 这些步骤的临时存储。
结果
在下文描述的某个步骤中计算出的文本等价。
将结果附加到 X，前面不加空格
  • 如果 X 为空，则将 result 复制到 X。
  • 如果 X 非空，则将 result 复制到 X 的末尾。
将结果附加到 X，前面加空格
  • 如果 X 为空，则将 result 复制到 X。
  • 如果 X 非空，则在 X 的末尾添加一个空格，然后将 result 复制到 该空格之后的 X 中。
将结果前置到 X，后面不加空格
  • 如果 X 为空，则将 result 复制到 X。
  • 如果 X 非空，则将 result 复制到 X 的开头。
将结果前置到 X，后面加空格
  • 如果 X 为空，则将 result 复制到 X。
  • 如果 X 非空，则将 result 复制到 X 的开头，并在 该副本之后添加一个空格。

4.3.2 计算步骤

给定元素的文本替代按如下方式计算：

  1. 初始化： 将 root node 设置为给定 元素，将 current node 设置为 root node，并将 total accumulated text 设置为空字符串 ("")。如果 root node 的角色禁止命名，则返回空字符串 ("")。
  2. 计算： 计算 current node 的文本替代：
    1. 隐藏且未被引用： 如果 current node 是 隐藏的 并且是：
      1. 不是 aria-labelledby 或 aria-describedby 遍历的一部分，其中由 该关系直接引用的节点是隐藏的。
      2. 也不是 原生宿主语言文本替代元素（例如 HTML 中的 label）或 属性 遍历的一部分，其中该遍历的根是隐藏的。
      返回空字符串。
      注

      重要的是要阐明用于可访问名称计算目的的隐藏的广义定义：

      1. 具有 CSS 属性 display:none、visibility:hidden、 visibility:collapse 或 content-visibility:hidden 的节点：它们 被认为是隐藏的，因为它们符合“不可感知”和 “显式隐藏”的准则。
      2. 具有 CSS 属性 opacity:0 或 filter:opacity(0%)，或类似 SVG 机制的节点：它们 不被认为是隐藏的。用这些方法隐藏的文本 仍然可以被选中或复制，并且用户代理仍会在 其可访问性树中公开它。
      3. 具有 aria-hidden="true" 属性的节点：它被 认为是隐藏的，符合“显式隐藏”准则。
      4. 在屏幕外或位于另一个对象后面而隐藏的节点：它们不 被认为是隐藏的。它们会在可访问性树中公开，并且 甚至可以为屏幕上的对象命名。
      注

      默认情况下，辅助技术不会 传达隐藏信息，但作者可以显式覆盖这一点，并通过 使用 aria-labelledby 或 aria-describedby 将隐藏文本作为 可访问 名称或可访问描述的一部分 包含进来。

      示例 1

      以下示例展示了子句“不是 aria-labelledby 或 aria-describedby 遍历的一部分，其中 由该关系直接引用的 节点是隐藏的”的含义。

      首先，element1 的可访问名称是“hello”，因为 尽管 element3 是隐藏的，但它是 从 element2 开始的 aria-labelledby 遍历的一部分，而 element2 本身也是隐藏的。

      示例 
      <element1 id="el1" role="button" aria-labelledby="el2" />
  <element2 id="el2" class="hidden">
    <element3 id="el3" class="hidden">hello</element3>
  </element2>
      示例 2

      相反，如果 element3 是隐藏的，并且它是 从 element2 开始的 aria-labelledby 遍历的一部分， 但 element2 不是隐藏的，则 element1 没有可访问名称。 

      <element1 id="el1" role="button" aria-labelledby="el2" />
<element2 id="el2">
  <element3 id="el3" class="hidden">hello</element3>
</element2>
    2. LabelledBy： 否则，如果 current node 具有一个 aria-labelledby 属性，其中包含至少 一个有效 IDREF，并且 current node 尚未作为正在进行的 aria-labelledby 或 aria-describedby 遍历的一部分，则按 IDREF 出现的顺序处理它们：
      1. 将 accumulated text 设置为空 字符串。
      2. 对于每个 IDREF：
        1. 将 current node 设置为 该 IDREF 所引用的节点。
        2. LabelledBy 遍历： 从总体计算步骤开始，计算 current node 的文本替代。将 result 设置为该文本替代。
        3. 将一个空格字符和 result 附加到 accumulated text。
      3. 如果 accumulated text 不是 空字符串 ("")，则返回它。

      LabelledBy 遍历与 隐藏且未被引用结合后的结果 意味着，当由 aria-labelledby 或 aria-describedby 引用的节点是隐藏的时， 用户代理MUST 将子树中的所有节点都作为可访问名称 或可访问 描述的一部分包含进来。

      示例 3

      以下示例展示了子句“……并且 current node 尚未作为 aria-labelledby 遍历的一部分……”的含义。

      1. element1 的可访问名称是“hello”， 因为这是对其 aria-labelledby 的第一次遍历， 并通向 element3。
      2. element2 没有可访问名称。该 计算涉及对其 aria-labelledby 的第一次遍历，并通向 element1，但随后不会继续跟随 element1 的 aria-labelledby。 
      <element1 id="el1" aria-labelledby="el3" />
<element2 id="el2" aria-labelledby="el1" />
<element3 id="el3"> hello </element3>
    3. 嵌入控件： 否则，如果 current node 是另一个部件的标签 （例如由 aria-labelledby 直接引用的任何元素）中嵌入的控件， 且用户可以调整该嵌入控件的值，则按如下方式将 该嵌入控件作为文本替代的一部分返回：
      1. 文本框： 如果嵌入控件的角色为 textbox，则返回其 值。
      2. 组合框/列表框： 如果嵌入控件的角色为 combobox 或 listbox， 则返回所选选项的文本替代。
      3. 范围： 如果嵌入控件的角色为 range（例如 spinbutton 或 slider）：
        1. 如果存在 aria-valuetext 属性，则返回其值，
        2. 否则，如果存在 aria-valuenow 属性，则返回其值，
        3. 否则，使用由宿主语言属性指定的 值。
      示例 4

      考虑一个包含文本输入字段的复选框标签：“Flash the screen [input] times”。 如果输入字段的值为 “5”，则完整标签是“Flash the screen 5 times”，例如： 

      <label for="flash">
  <input type="checkbox" id="flash">
  Flash the screen <span tabindex="0" role="textbox" aria-label="number of times" contenteditable>5</span> times.
</label>
    4. AriaLabel： 否则，如果 current node 不是 slot 元素，并且具有一个 aria-label 属性，其值 不是 undefined，不是空字符串，也不是在去除空白后为 空字符串：
      1. 如果对 current node 的遍历是由于Name From Content 后代 节点递归，并且 current node 是嵌入控件，则忽略 aria-label 并跳至规则 嵌入控件。
      2. 否则，返回 aria-label 的值。
      示例 5

      以下示例展示了当一个节点具有引用 自身的 aria-labelledby 时，aria-labelledby 与 aria-label 的交互。<span role="button"> 元素的可访问 名称分别是“Delete Documentation.pdf”和“Delete HolidayLetter.pdf”。

      示例 
      <h1>Files</h1>
<ul>
  <li>
    <a id="file_row1" href="./files/Documentation.pdf">Documentation.pdf</a>
    <span role="button" tabindex="0" id="del_row1" aria-label="Delete" aria-labelledby="del_row1 file_row1"></span>
  </li>
  <li>
    <a id="file_row2" href="./files/HolidayLetter.pdf">HolidayLetter.pdf</a>
    <span role="button" tabindex="0" id="del_row2" aria-label="Delete" aria-labelledby="del_row2 file_row2"></span>
  </li>
</ul>
    5. 宿主语言标签： 否则，如果 current node 的原生标记提供了一个定义文本 替代的属性（例如 alt）或元素（例如 HTML label 或 SVG title），则以 宿主语言定义的 flat string 形式返回该替代（例如 HTML-AAM），除非 current node 被公开为呈现性（role="presentation" 或 role="none"、alt=""）。
      注
      重要！原生 HTML 元素的可访问名称可能会根据宿主语言 以不同方式计算。请务必查阅 HTML-AAM 以获取更多 信息。
      注

      例如，在 HTML 中， img 元素的 alt 属性定义文本 替代字符串，而 label 元素为被引用的表单元素提供文本。在 SVG2 中，desc 和 title 元素提供其父元素的 描述。

    6. 由内容提供名称： 否则，如果 current node 的角色允许 由 内容提供名称，或者如果 current node 由 aria-labelledby、aria-describedby 引用，或者 是原生宿主语言文本替代元素（例如 HTML 中的 label），或者是 原生宿主语言文本替代元素的后代：
      1. 由内容提供名称重置： 将 accumulated text 设置为空字符串。
      2. 由生成内容提供名称： 检查与 current node 关联的 CSS 生成文本内容，并将其包含在 accumulated text 中。 CSS ::before 和 ::after 伪元素 [CSS2] 可以为具有内容模型的元素提供文本内容，此外还有 ::marker 伪元素。
        1. 对于 ::marker 伪元素，用户代理MUST 将 CSS 文本内容前置到 current node 的文本内容之前，不加 空格，前提是 current node 支持 ::marker。
        2. 对于 ::before 伪元素，用户代理MUST 将 CSS 文本内容前置到 current node 的文本内容之前，不加 空格。
        3. 对于 ::after 伪元素，用户代理MUST 将 CSS 文本内容附加到 current node 的文本内容之后，不加 空格。
      3. 确定子节点： 确定 current node 的 rendered child nodes：
        1. 如果 current node 具有一个已附加的影子 根，则将 rendered child nodes 设置为该影子 根的子 节点。
        2. 否则，如果 current node 是具有已分配 节点的slot，则将 rendered child nodes 设置为 current node 的已分配 节点。
        3. 否则，将 rendered child nodes 设置为 current node 的子节点。
      4. 从每个子节点取得名称： 对 current node 的每个 rendered child node：
        1. 将 current node 设置为 rendered child node。
        2. 从总体 计算步骤开始，计算 current node 的文本替代。将 result 设置为该文本替代。
        3. 将 result 附加到 accumulated text。
      5. 如果 accumulated text 不是 空字符串 ("")，则返回它。

      重要：子树中的每个节点只会 被查询一次。如果已从某个后代收集了文本，但该后代又被某个 后代节点中的另一个 IDREF 引用，则不会跟随第二个或后续的引用。这是 为了避免无限循环。

      注

      此步骤可以应用于子节点本身，这意味着 计算是递归的，并会从 current node 的 子树中的所有元素收集文本，无论其深度如何。不过，任何给定后代节点的文本 替代都可能来自上文步骤 B 到 D 中描述的更高优先级标记， 其中“Namefrom: author”属性为 整个子树提供文本替代。

      注

      2024 年 1 月 18 日：ARIA 工作组正在考虑 根据 current node 及其相邻节点和伪元素的 CSS display 值， 以带空格或不带空格的方式连接文本字符串的可行性。 正在进行的讨论见 AccName #225。

    7. 文本节点： 否则，如果 current node 是一个文本节点，则返回 其文本内容。
    8. 递归地由内容提供名称： 否则，如果 current node 是某个正在计算其可访问名称或可访问 描述的元素的后代，并且包含后代，则继续执行 由内容提供名称重置。
    9. 工具提示： 否则，如果 current node 具有一个工具提示属性， 则返回其值。
      注

      只有在没有任何其他内容（包括子树内容）提供结果时，才使用工具提示属性。

    10. 将一个空格字符以及上面每个步骤的 result 附加到 total accumulated text。
  3. 所有步骤完成后，total accumulated text 将用作发起计算的 元素的可访问名称或可访问描述。

5. 可访问名称和 描述映射

关于名称和描述的可访问性 API 映射信息，包括 labelled-by/label-for 和 described-by/description-for 等关系，记录在 核心可访问性 API 映射规范 [CORE-AAM-1.2] 中。请参见 aria-label、aria-labelledby 和 aria-describedby 的映射表条目。
