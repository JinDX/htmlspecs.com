摘要

屏幕方向规范标准化了设备屏幕方向的类型和角度，并提供了锁定和解锁屏幕方向的方式。该规范定义的API暴露了设备屏幕方向的当前类型和角度，并在变化时触发事件。这使得Web应用程序能够以编程方式适应多种屏幕方向的用户体验，与CSS协同工作。该API还允许在某些前提条件下锁定屏幕方向。这对于用户物理旋转设备但屏幕方向不应改变的应用程序（例如电脑游戏）特别有用。

1. 使用示例

本节为非规范性内容。

在本例中，点击“锁定”按钮会请求进入全屏模式，并将屏幕锁定为相反方向。点击“解锁”按钮则会解除屏幕锁定。

示例 1：锁定到指定方向并解锁 
<script>
function updateLockButton() {
  const lockButton = document.getElementById("button");
  const newOrientation = getOppositeOrientation();
  lockButton.textContent = `Lock to ${newOrientation}`;
}

function getOppositeOrientation() {
  return screen
    .orientation
    .type
    .startsWith("portrait") ? "landscape" : "portrait";
}

async function rotate(lockButton) {
  if (!document.fullscreenElement) {
    await document.documentElement.requestFullscreen();
  }
  const newOrientation = getOppositeOrientation();
  await screen.orientation.lock(newOrientation);
  updateLockButton(lockButton);
}

screen.orientation.addEventListener("change", updateLockButton);

window.addEventListener("load", updateLockButton);
</script>

<button onclick="rotate(this)" id="button">
  锁定到...
</button>
<button onclick="screen.orientation.unlock()">
  解锁
</button>

2. 概念

锁定屏幕方向到某个 OrientationLockType orientation 意味着用户只能将屏幕旋转到特定的 屏幕方向 —— 可能会排除其他方向。屏幕可旋转到的方向由用户代理、用户偏好、操作系统的惯例或屏幕本身决定。例如， 将方向锁定为 landscape 意味着用户可以将屏幕旋转到 landscape-primary，如果系统允许，也许还能旋转到 landscape-secondary，但不会变为 portrait-secondary 方向。

解锁屏幕方向后，最终用户可以不受限制地将屏幕旋转到系统允许的任何 屏幕方向。

2.1 屏幕方向类型

屏幕可以处于，或锁定为下列之一的 屏幕方向：

任意
用户可以将屏幕旋转到设备操作系统或最终用户允许的任意方向。
默认（未锁定）
当屏幕未锁定时，设备的默认行为 （即活动方向锁定为 null）。该方向由设备的操作系统、用户代理、最终用户控制，或可能由 已安装的 Web 应用设置。例如， 当屏幕方向未锁定且用户旋转设备时，某些设备会将方向变化限制为 portrait-primary、 landscape-primary 和 landscape-secondary， 但不会变为 portrait-secondary。
横屏
屏幕的宽度大于高度。
自然
设备显示屏最自然的方向，由用户代理、用户、操作系统或屏幕本身决定。例如，设备面向用户时被直立持握。电脑显示器通常的自然方向为 landscape-primary，而手机通常的自然方向为 portrait-primary。
竖屏
屏幕的高度大于宽度。
主方向
设备屏幕的自然方向，可以是 竖屏 或 横屏。
次方向
设备屏幕主方向的相反方向，可以是 竖屏或横屏。

2.2 当前屏幕方向类型和角度

输出设备的屏幕关联以下概念：

当前激活的方向锁定
屏幕锁定到的 屏幕方向，用 OrientationLockType 表示；如果未锁定则为 null。
当前方向角度
屏幕相对于其自然方向逆时针旋转的角度（度数），由 屏幕方向取值列表推导得出。
当前方向类型
屏幕的屏幕方向，用 OrientationType 表示。

下方的屏幕方向取值列表标准化了不同自然方向的屏幕，每种方向类型对应的角度：

对于自然 竖屏方向的屏幕：
  • portrait-primary：0°
  • landscape-primary：90°
  • portrait-secondary：180°
  • landscape-secondary：270°
对于自然 横屏方向的屏幕：
  • landscape-primary：0°
  • portrait-primary：90°
  • landscape-secondary：180°
  • portrait-secondary：270°

3. Document 接口的扩展

3.1 内部槽

Document 接口扩展了以下内部插槽：

内部插槽 描述
[[orientationPendingPromise]] 可能为 null，或为一个 Promise。当被赋值为 Promise 时， 该 promise 表示对屏幕方向锁定的请求。

4. Screen 接口的扩展 

WebIDLpartial interface Screen {
  [SameObject] readonly attribute ScreenOrientation orientation;
};

Window 对象有一个 关联的 ScreenOrientation， 它是 Screen 的 orientation 对象（即 ScreenOrientation 实例，位于 window.screen.orientation）。

5. ScreenOrientation 接口 

WebIDL[Exposed=Window]
interface ScreenOrientation : EventTarget {
  Promise<undefined> lock(OrientationLockType orientation);
  undefined unlock();
  readonly attribute OrientationType type;
  readonly attribute unsigned short angle;
  attribute EventHandler onchange;
};

5.1 内部槽

内部插槽 描述
[[angle]] 表示屏幕上次已知的当前方向角度（单位为度），类型为 unsigned short，值来源于 屏幕方向取值列表。
[[initialType]] 表示在浏览上下文创建时，屏幕的当前方向类型。
[[type]] 表示屏幕上次已知的当前方向类型，类型为OrientationType枚举值。

5.2 lock() 方法

当调用 lock() 方法并传入 OrientationLockType orientation 时，用户代理 必须执行以下步骤。

用户代理 可以要求 文档及其关联的 浏览上下文满足一个或多个 锁定前置条件，以便锁定屏幕方向。参见 10. 与 Web 应用清单的交互和9. 与全屏 API 的交互。

  1. 令 document 为 this 的 相关全局对象的 关联 Document。
  2. 使用 document 运行通用安全检查。如果 异常被抛出，则返回一个以该异常拒绝的 promise并中止这些步骤。
  3. 如果用户代理不支持将屏幕方向锁定为 orientation，则返回一个以 "NotSupportedError" DOMException拒绝的 promise 并中止这些步骤。
  4. 如果 document 的 [[orientationPendingPromise]] 不为 null，则对 document 的当前锁定 promise 进行拒绝并置为 null，异常为 "AbortError"。
  5. 将 document 的 [[orientationPendingPromise]] 设为一个新的 promise。
  6. 应用方向锁定 orientation 到 document。
  7. 返回 document 的 [[orientationPendingPromise]]。

5.3 unlock() 方法

当调用 unlock() 方法时，用户代理必须执行以下步骤：

  1. 令 document 为 this 的 相关全局对象的 关联 Document。
  2. 对 document 运行通用安全检查。如果异常被抛出，则重新抛出该异常并中止这些步骤。
  3. 如果屏幕的当前激活的方向锁定为 null，则返回 undefined。
  4. 如果 document 的 [[orientationPendingPromise]] 不为 null，则对 document 的当前锁定 promise 进行拒绝并置为 null，异常为 "AbortError"。
  5. 应用方向锁定 null 到 document。
注意：为什么 unlock() 不返回 promise？

unlock() 不返回 promise，因为它等价于锁定到 默认屏幕方向，而该方向可能会或者不会被 用户代理知晓。因此，用户代理 无法预测新的方向会是什么，甚至无法预测是否会发生方向变化。

5.4 常见安全检查

通用安全检查针对 Document document 的步骤如下：

  1. 如果 document 不是 完全激活，则抛出 "InvalidStateError" DOMException。
  2. 如果 document 设置了 沙箱化方向锁定浏览上下文标志， 抛出 "SecurityError" DOMException。
  3. 如果 document 的 visibility state 为 "hidden"，抛出 "SecurityError" DOMException。

5.5 type 属性

获取时，type 属性返回 this 的 [[type]]。

5.6 angle 属性

获取时，angle 属性返回 this 的 [[angle]]。

注意：angle 的取值代表什么？

angle 属性表示屏幕相对于其自然方向旋转了多少度。屏幕方向取值列表规定了屏幕旋转时 angle 如何变化。

5.7 onchange 事件处理程序属性

onchange 属性是一个 事件处理程序 IDL 属性， 用于 onchange 事件处理程序，其 事件处理类型 为 change。

6. OrientationLockType 枚举 

WebIDLenum OrientationLockType {
      "any",
      "natural",
      "landscape",
      "portrait",
      "portrait-primary",
      "portrait-secondary",
      "landscape-primary",
      "landscape-secondary"
    };

OrientationLockType 枚举 表示可以将屏幕锁定的屏幕方向。

注意：方向支持

用户代理可能只支持部分 OrientationLockType 值。例如，用户代理可能不支持锁定到"portrait-secondary"或 "landscape-secondary"方向。

  • “any” 表示 任意。
  • “natural” 表示 自然。
  • “landscape” 表示 横屏。
  • “portrait” 表示 竖屏。
  • “portrait-primary” 表示 portrait-primary。
  • “portrait-secondary” 表示 portrait-secondary。
  • “landscape-primary” 表示 landscape-primary。
  • “landscape-secondary” 表示 landscape-secondary。

7. OrientationType 枚举 

WebIDLenum OrientationType {
      "portrait-primary",
      "portrait-secondary",
      "landscape-primary",
      "landscape-secondary"
    };

OrientationType 枚举值用于表示屏幕的 当前方向类型。

  • "portrait-primary" 代表纵向-主。
  • "portrait-secondary" 代表纵向-次。
  • "landscape-primary" 代表横向-主。
  • "landscape-secondary" 代表 横向-次。

8. 算法

8.1 初始化ScreenOrientation对象

当创建 浏览上下文 context 时， 用户代理 必须：

  1. 让screenOrientation成为context的关联的ScreenOrientation。
  2. 初始化screenOrientation的[[initialType]]内部槽为屏幕的当前方向类型。
  3. 初始化screenOrientation的[[type]]内部槽为屏幕的当前方向类型。
  4. 初始化screenOrientation的[[angle]]内部槽为屏幕的当前方向角度。

8.2 拒绝文档的当前锁定承诺

当步骤要求用 DOMString exceptionName 对 Document document 的 当前锁定 promise 进行拒绝并置为 null 时， 用户代理必须：

  1. 断言：[[orientationPendingPromise]] 不为 null。
  2. 令 promise 为 document 的 [[orientationPendingPromise]]。
  3. 在 DOM 操作任务源上， 以 document 的 相关全局对象， 排入一个全局任务，以使用新的 exceptionName DOMException 拒绝 promise。
  4. 将 document 的 [[orientationPendingPromise]] 设为 null。

8.3 应用屏幕方向锁定

当步骤要求将 OrientationLockType? orientation 的方向锁定应用到 Document document 时， 用户代理必须执行以下步骤：

  1. 如果 document 在并行期间不再完全激活，且 [[orientationPendingPromise]] 不为 null，则对 document 的当前锁定 promise 进行拒绝并置为 null，异常为 "AbortError"。
  2. 令 topDocument 为 document 的顶层浏览上下文的 活动文档。
  3. 令 descendantDocs 为一个有序集合， 由 topDocument 的 后代 navigables 的 活动文档组成（如有），按树顺序排列。
  4. 对 descendantDocs 中每一个 doc：
    1. 如果 doc 是 document，则跳过。
    2. 如果 doc 的 [[orientationPendingPromise]] 为 null，则跳过。
    3. 对 doc 的当前锁定 promise 进行拒绝并置为 null，异常为 "AbortError"。
  5. 按并行执行以下子步骤：
    1. 如果 document 已不再完全激活， 且 document 的 [[orientationPendingPromise]] 不为 null，则对 document 的当前锁定 promise 进行拒绝并置为 null，异常为 "AbortError"，并中止这些步骤。
    2. 如果 orientation 为 null，则解锁屏幕方向。
    3. 否则，尝试将屏幕方向锁定为 orientation。根据平台约定，调整视口绘制方式以匹配 orientation。
    4. 如果由于用户先前设定的偏好、平台限制或其他原因导致尝试失败：
      1. 将屏幕的当前激活的方向锁定设为 null。
      2. 在 DOM 操作任务源 以 document 的 相关全局对象 排入一个任务，以：
        1. 如果 document 的 [[orientationPendingPromise]] 不为 null， 则对 document 的当前锁定 promise 进行拒绝并置为 null，异常为 "NotSupportedError"。
        2. 中止这些步骤。
      注意

      如果用户已设置不允许 web 应用更改屏幕方向的偏好，或者底层平台（而非用户代理）不允许锁定到指定 orientation，就会发生这种情况。

    5. 将屏幕的当前激活的方向锁定设为 orientation，并更新当前方向类型 和 当前方向角度，以反映 屏幕方向的任何变化。
  6. 在 DOM 操作任务源 以 document 的 相关全局对象 排入一个全局任务，以：
    1. 令 promise 为 document 的 [[orientationPendingPromise]]。
      注意

      由于 promise 需要在触发 "change" 事件后再 resolve， 因此需要保留引用，以防它被事件处理函数调用 lock() 时中止。

    2. 将 document 的 [[orientationPendingPromise]] 设为 null。
    3. 对 topDocument 运行屏幕方向变更步骤。
    4. 如果 promise 不为 null，则resolve promise，值为 undefined。

8.4 屏幕方向更改

当用户代理检测到某个顶层浏览上下文的屏幕方向发生变化，或用户将顶层浏览上下文移动到不同的屏幕时，应对该屏幕方向变更步骤，参数为顶层浏览上下文的 活动文档。

对于Document document，其屏幕方向变更步骤如下：

  1. 如果 document 的visibility state 为 "hidden"，则中止这些步骤。
  2. 令 type 和 angle 为屏幕的当前方向类型和当前方向角度。
  3. 令 screenOrientation 为 document 的相关全局对象的关联 ScreenOrientation。
  4. 如果 type 等于 screenOrientation 的 [[type]] 且 angle 等于 screenOrientation 的 [[angle]]，则中止这些步骤。
  5. 在 用户交互任务源上， 以 document 的相关全局对象，排入一个全局任务，执行以下步骤：
    1. 将 screenOrientation 的 [[angle]] 设为 angle。
    2. 将 screenOrientation 的 [[type]] 设为 type。
    3. 在 screenOrientation 上触发名为 "change" 的事件。
  6. 令 descendantDocs 为一个有序集合， 由 document 的后代 navigables 的活动文档组成（如有），按树顺序排列。
  7. 对 descendantDocs 中每一个 doc， 使用 doc 运行 屏幕方向变更步骤。

8.5 处理页面可见性更改

[HTML] 的“更新可见性状态”算法会运行屏幕方向变更步骤。

注意

开发者需要注意，文档如果其visibility state为 "hidden"，将不会收到方向变更事件。然而，一旦文档再次变为可见，将会收到变更事件。

8.6 处理卸载文档

每当卸载文档清理步骤针对 document 运行时，用户代理必须执行以下步骤：

  1. 如果 document 不是顶层浏览上下文的 活动文档，则中止这些步骤。
  2. 使用 document 运行完全解锁屏幕方向步骤。

8.7 完全解锁方向

针对 Document document，完全解锁屏幕方向步骤如下：

  1. 如果 document 的 [[orientationPendingPromise]] 不为 null，则对 document 的当前锁定 promise 进行拒绝并置为 null，异常为 "AbortError"。
  2. 令 topDocument 为 document 的顶层浏览上下文的 活动文档。
  3. 对 topDocument 应用方向锁定 null。

9. 与全屏 API 的交互

用户代理应当将 lock() 的使用限制在简单全屏文档上，作为锁定前置条件。[fullscreen]

当文档退出全屏时，也会运行完全解锁屏幕方向步骤。[fullscreen]

10. 与 Web 应用清单的交互

Web Application Manifest 规范允许 Web 应用通过 默认屏幕方向 和 orientation 成员来设置默认屏幕方向。

用户代理应当要求已安装的 Web 应用以 "fullscreen" 显示模式展示，作为锁定前置条件。

11. 可访问性考虑

由于用户的设备可能固定在一个方向上（例如安装在轮椅的扶手上），当开发人员希望用户在锁定屏幕方向时旋转设备时，开发人员需要了解Web 内容无障碍指南 (WCAG) 2.1中的方向成功标准。该标准要求无论屏幕方向如何，内容和功能必须可用。如果特定方向是必需的，Web 应用必须告知用户方向要求。

12. 隐私和安全考虑

屏幕的类型和 角度可能成为指纹识别的来源。以下缓解措施有助于保护用户隐私，不泄露设备的持握方式，同时也防止 secondary 方向类型及其关联角度被用于指纹识别。

为了抵抗指纹识别（如在隐私浏览模式下），用户代理可以：

  1. 在一个顶层浏览上下文的生命周期内，表现得好像 屏幕的自然方向为 [[initialType]]。
  2. 限制 type getter 的可能返回值为 “portrait-primary” 或 “landscape-primary”。返回值由屏幕宽高比决定。
  3. 如果当前方向类型等于 [[initialType]]，则 angle getter 返回 0；否则返回 90。
  4. 如果屏幕方向发生变化，仅在 change 事件中， 当前方向类型从 竖屏变为 横屏，或反之时触发。
