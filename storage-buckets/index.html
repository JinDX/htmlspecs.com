摘要

Storage Buckets API 为站点提供了一种将本地存储的数据组织到称为“存储桶”的分组中的方式。 这允许用户代理或站点独立地管理和删除存储桶，而不是对来自单一源的所有数据应用相同的处理。

1. StorageBucketManager 接口

[SecureContext]
interface mixin NavigatorStorageBuckets {
  [SameObject] readonly attribute StorageBucketManager storageBuckets;
};
Navigator includes NavigatorStorageBuckets;
WorkerNavigator includes NavigatorStorageBuckets;

每个环境设置对象都有一个关联的 StorageBucketManager 对象。

storageBuckets 获取器的 步骤是返回 this 的相关设置对象的 StorageBucketManager 对象。

用户代理有一个关联的 storage bucket manager，它是 启动新的并行队列的结果。 

[Exposed=(Window,Worker),
 SecureContext]
interface StorageBucketManager {
    Promise<StorageBucket> open(DOMString name, optional StorageBucketOptions options = {});
    Promise<sequence<DOMString>> keys();
    Promise<undefined> delete(DOMString name);
};

dictionary StorageBucketOptions {
  boolean persisted = false;
  unsigned long long quota;
  DOMHighResTimeStamp expires;
};

1.1. 创建存储桶

open(name, options) 方法步骤为：

  1. 令 environment 为 this 的相关设置对象。

  2. 令 shelf 为以 environment 为给定值运行 获取本地存储货架的结果。

  3. 如果 shelf 是 failure，则返回一个以 TypeError 拒绝的 promise。

  4. 如果以 name 运行验证存储桶名称的结果是 failure，则返回一个以 TypeError 拒绝的 promise。

  5. 令 p 为一个新的 promise。

  6. 将以下步骤入队到 storage bucket manager：

    1. 令 r 为以 shelf、name 和 options 运行打开存储桶的结果。

    2. 如果 r 是 failure，则排入一个存储任务，以用 TypeError 拒绝 p。

    3. 否则，排入一个存储任务，以用 r 解决 p。

  7. 返回 p。

要为 shelf，给定存储桶 name 和可选的 options，打开存储桶，运行以下 步骤：

  1. 令 expires 为 undefined。

  2. 如果 options["expires"] 存在，则：

    1. 将 expires 设为 options["expires"]。

    2. 如果 Unix 纪元之后 expires 毫秒的时刻早于 相关设置对象的当前墙上时间，则返回 failure。

  3. 令 quota 为 undefined。

  4. 如果 options["quota"] 存在，则：

    1. 将 quota 设为 options["quota"]。

    2. 如果 quota 小于或等于零，则返回 failure。

  5. 令 persisted 为 false。

  6. 如果 options["persisted"] 为 true，则：

    1. 令 permission 为请求使用 "persistent-storage" 权限的结果。

    2. 如果 permission 是 "granted"， 则将 persisted 设为 true。

  7. 令 bucket 为以 shelf 和 name 运行 获取或过期存储桶的结果。

  8. 如果 bucket 为 null，则：

    1. 将 bucket 设为一个名称为 name 的新存储桶。

    2. 将 bucket 的配额值设为 quota。

    3. 将 shelf 的存储桶映射[name] 设为 bucket。

  9. 如果 persisted 为 true，则将 bucket 的存储桶模式 设为 "persistent"。

  10. 将 bucket 的过期时间设为 Unix 纪元之后 expires 毫秒。

  11. 令 storageBucket 为一个新的 StorageBucket。

  12. 将 storageBucket 的存储桶 设为 bucket。

  13. 返回 storageBucket。

要给定字符串 name 验证存储桶 名称，运行以下步骤：

  1. 如果 name 包含任何不是 码点中的 ASCII 小写字母、ASCII 数字、 U+005F (_)，或 U+002D(-)，则返回 failure。

  2. 如果 name 的码点长度为 0 或超过 64，则返回 failure。

  3. 如果 name 以 U+005F (_) 或 U+002D(-) 开头，则返回 failure。

  4. 返回。

要在 shelf 上，给定字符串 name，获取或过期 存储桶，运行以下步骤：

  1. 如果存在，则令 bucket 为 shelf 的存储桶 映射[name]。否则返回 null。

  2. 如果 bucket 的过期时间非 null，并且早于相关设置对象的当前墙上时间，则：

    1. 将 bucket 的removed 设为 true。

    2. 返回 null。

  3. 返回 bucket。

1.2. 删除存储桶

delete(name) 方法 步骤为：

  1. 令 environment 为 this 的相关设置对象。

  2. 令 shelf 为以 environment 为给定值运行 获取本地存储货架的结果。

  3. 如果 shelf 是 failure，则返回一个以 TypeError 拒绝的 promise。

  4. 令 p 为一个新的 promise。

  5. 如果以 name 运行验证存储桶名称的结果是 failure，则用 InvalidCharacterError 拒绝 p。

  6. 否则，将以下步骤入队到 storage bucket manager：

    1. 以 shelf 和 name 运行移除存储桶。

    2. 排入一个存储任务，以解决 p。

  7. 返回 p。

要在 shelf 上，给定存储桶 name，移除存储桶， 运行以下步骤：

  1. 如果存在，则令 bucket 为 shelf 的存储桶 映射[name]。否则返回。

  2. 移除 shelf 的存储桶映射中的 键 name。

  3. 将 bucket 的removed 设为 true。

  4. 返回。

当存储桶被设置为 removed 时，特定存储端点可能需要运行额外动作来移除数据。

[IndexedDB-3] 需要定义当数据因配额被逐出时如何发生删除。

[FS] 需要定义当数据因配额被逐出时 Bucket File System 如何发生删除。

[Service-Workers] 需要定义当数据因配额被逐出时，CacheStorage 和 Service Workers 如何发生删除。

1.3. 枚举存储桶

keys() 方法步骤为：

  1. 令 shelf 为运行获取本地存储货架的结果。

  2. 如果 shelf 是 failure，则返回一个以 TypeError 拒绝的 promise。

  3. 令 p 为一个新的 promise。

  4. 令 keys 为一个新的列表。

  5. 将以下步骤入队到 storage bucket manager：

    1. 对于 shelf 的存储桶 映射中的每个 key，运行以下步骤：

      1. 令 bucket 为以 shelf 和 key 运行 获取或 过期存储桶的结果。

      2. 如果 bucket 非 null，则将 key 附加到 keys。

    2. 排入一个存储任务，以用 keys 解决 p。

  6. 返回 p。

2. StorageBucket 接口

[Exposed=(Window,Worker),
 SecureContext]
interface StorageBucket {
  readonly attribute DOMString name;

  [Exposed=Window] Promise<boolean> persist();
  Promise<boolean> persisted();

  Promise<StorageEstimate> estimate();

  Promise<undefined> setExpires(DOMHighResTimeStamp expires);
  Promise<DOMHighResTimeStamp?> expires();

  [SameObject] readonly attribute IDBFactory indexedDB;

  [SameObject] readonly attribute CacheStorage caches;

  Promise<FileSystemDirectoryHandle> getDirectory();
};

StorageBucket 有一个关联的存储桶。

存储桶有一个关联的 removed 标志，它是一个布尔值，初始为 false。当一个 存储桶被删除时将其设为 true。

StorageBucket 有一个 DOMString 对象 name，它是存储桶映射中映射到 存储桶的键。

浏览器可能会有 IPC 和磁盘访问，使得操作异步。Promise 从任务中解决，以允许这种工作。

2.1. 持久性

与已经定义 存储桶模式的 Storage § 4.5 Storage buckets 合并。

persist() 方法步骤为：

  1. 令 bucket 为 this 的存储 桶。

  2. 令 environment 为 this 的相关设置对象。

  3. 令 p 为一个新的 promise。

  4. 并行运行以下步骤：

    1. 如果 bucket 的removed 标志为 true，则排入一个存储任务，以用 InvalidStateError 拒绝 p。

    2. 如果 bucket 的存储桶 模式是 "persistent"，则令 persisted 为 true。

    3. 否则，

      1. 令 permission 为以 "persistent-storage" 和 environment 获取当前权限状态的结果。

      2. 如果 permission 是 "granted"， 则将 bucket 的存储桶模式设为 "persistent"，并 将 persisted 设为 true。

      3. 否则，将 persisted 设为 false。

    4. 排入一个存储任务，以用 persisted 解决 p。

  5. 返回 p。

persisted() 方法步骤为：

  1. 令 p 为一个新的 promise。

  2. 令 bucket 为 this 的存储 桶。

  3. 否则，并行运行这些步骤：

    1. 如果 bucket 的removed 标志为 true，则排入一个存储任务，以用 InvalidStateError 拒绝 p。

    2. 如果 bucket 的存储桶 模式是 "persistent"，则令 persistent 为 true，否则为 false。

    3. 排入一个存储任务，以用 persistent 解决 p。

  4. 返回 p。

2.2. 配额

存储桶有一个 配额 值，是一个 number-or-null，初始为 null。 指定可由该存储桶使用的以字节为单位的使用上限。用户代理可以进一步限制实际可用的存储空间。

存储桶的 存储使用量是其所有存储瓶所使用字节数的 由实现定义的粗略估计。

estimate() 方法步骤为：

  1. 令 environment 为 this 的相关设置对象。

  2. 令 shelf 为以 environment 运行获取本地存储货架的结果。

  3. 如果 shelf 是 failure，则返回一个以 TypeError 拒绝的 promise。

  4. 令 bucket 为 this 的存储 桶。

  5. 如果 bucket 的removed 标志为 true，则返回一个以 InvalidStateError 拒绝的 promise。

  6. 令 p 为一个新的 promise。

  7. 否则，并行运行以下步骤：

    1. 令 quota 为 shelf 的存储配额。

    2. 如果 bucket 的配额 值非 null，则将 quota 设为该值。

    3. 令 usage 为 bucket 的存储使用量。

    4. 令 dictionary 为一个新的 StorageEstimate 字典，其 usage 成员是 usage，且 quota 成员是 quota。

    5. 排入一个存储任务，以用 dictionary 解决 p。

  8. 返回 p。

如果配额值超过存储货架（即站点）可用的 总空间量，则会忽略该配额值。 它的预期用途是防止特定存储桶用尽整个站点的存储空间。

2.3. 过期

存储桶有一个 过期时间，它要么是 null，要么是时刻 在墙上时钟上的值，初始为 null。 指定存储桶生命周期的上限。

获取或过期 存储桶算法会在调用 keys() 或 open() 时移除已过期的存储桶。 当面临存储压力时，用户代理可以在存储桶模式为 "best-effort" 的存储桶达到其过期时间之前清除它们。 无论存储桶模式如何，当达到过期时间时， 用户代理可以在调用 open() 或 keys() 之前移除任何存储桶

setExpires(expires) 方法步骤 为：

  1. 令 p 为一个新的 promise。

  2. 令 bucket 为 this 的存储 桶。

  3. 否则，并行运行这些步骤：

    1. 如果 bucket 的removed 标志为 true，则排入一个存储任务，以用 InvalidStateError 拒绝 p。

    2. 否则，将 bucket 的过期时间设为 Unix 纪元之后 expires 毫秒。

    3. 排入一个存储任务，以解决 p。

  4. 返回 p。

expires() 方法步骤为：

  1. 令 p 为一个新的 promise。

  2. 令 bucket 为 this 的存储 桶。

  3. 否则，并行运行这些步骤：

    1. 如果 bucket 的removed 标志为 true，则排入一个存储任务，以用 InvalidStateError 拒绝 p。

    2. 否则，令 expiration 为 bucket 的过期时间。

    3. 排入一个存储任务，以用 expiration 解决 p。

  4. 返回 p。

2.4. 使用存储端点

存储端点，即存储瓶，可以按如下所述访问。

2.4.1. 使用 Indexed Database

IDBFactory 方法需要接受一个存储瓶映射，而不是 storageKey。

StorageBucket 有一个 IDBFactory 对象，初始为 null。indexedDB 获取器步骤为：

  1. 如果 this 的 indexedDB 为 null，运行以下步骤：

    1. 令 bucket 为 this 的存储桶。

    2. 令 bottle map 为以 bucket 和 "indexedDB" 运行获取本地存储瓶映射的结果。

    3. 令 indexedDB 为一个 IDBFactory 对象。

    4. 将 indexedDB 的存储瓶映射设为 bottle map。

    5. 将 this 的 indexedDB 设为 indexedDB。

  2. 返回 this 的 indexedDB。

2.4.2. 使用 CacheStorage

StorageBucket 有一个 CacheStorage 对象，初始为 null。caches 获取器 步骤为：

  1. 如果 this 的 caches 为 null，运行以下步骤：

    1. 令 bucket 为 this 的存储桶。

    2. 令 bottle map 为以 bucket 和 "cacheStorage" 运行获取本地存储瓶 映射的结果。

    3. 令 cacheStorage 为一个 CacheStorage 对象。

    4. 将 cacheStorage 的相关名称到缓存映射设为 bottle map。

    5. 将 this 的 caches 设为 cacheStorage。

  2. 返回 this 的 caches。

2.4.3. 使用 Bucket File System

[Storage] 需要定义辅助工具来检索给定（非默认）存储桶的 瓶映射。

[FS] 需要定义辅助工具来在给定瓶映射的情况下检索 OPFS。

getDirectory() 步骤为：

  1. 令 map 为以 this 的存储 桶和 "fileSystem" 运行获取本地存储瓶映射的结果。

  2. 返回以 map 运行 getDirectory 的结果。

参见 File System § 3 Accessing the Bucket File System。

2.5. Clear Site Data 集成

更新 Clear Site Data § 3.1 The Clear-Site-Data HTTP Response Header Field。

"storage:bucket-name"

如果类型字符串以 "storage:" 开头，则 : 之后的剩余字符将被视为指代 特定响应 URL 的源中的特定存储 桶。

将以下步骤添加到 Clear Site Data § 4.1 Parsing 中的算法。

要解析带存储桶的 Clear-Site-Data 标头，执行以下步骤：

  1. 对于 header 中的每个 type，执行以下步骤：

    1. 如果 type 不以 "storage:" 开头，则中止这些 步骤。

    2. 令 bucket name 为 type 中从 8 到结尾的按位置取码元子字符串。

    3. 如果以 bucket name 运行验证存储桶名称的结果是 failure，则中止这些步骤。

    4. 向 types 附加一个由 ("storage-bucket", bucket name) 组成的元组

将以下步骤添加到 Clear Site Data § 4.2 Clear data for response 中的算法。

要给定 bucket name 清除带存储桶的数据， 执行以下步骤：

  1. 令 environment 为 this 的相关设置对象。

  2. 令 shelf 为以 environment 为给定值运行 获取本地存储货架的结果。

  3. 如果 shelf 是 failure，则抛出一个 TypeError 并中止这些步骤。

  4. 对于 types 中的每个 type，执行以下步骤：

    1. 如果 type 不是一个元组，或 type[0] 不是 "storage-bucket"，则中止这些步骤。

    2. 如果存在，则令 bucket 为 shelf 的存储桶 映射[bucket name]。否则中止这些步骤

    3. 移除 bucket。
