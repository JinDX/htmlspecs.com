摘要

本CSS模块定义了一个二维基于网格的布局系统，针对用户界面设计进行了优化。在网格布局模型中，网格容器的子元素可以放置到预定义的灵活或固定尺寸的布局网格中的任意槽位。

CSS 是一种用于描述结构化文档（如HTML和XML）在屏幕、纸张等上的渲染语言。

1. 引言

本节不具有规范性。

网格布局是CSS中的一种新布局模型，具有强大的控制盒子及其内容大小和位置的能力。与弹性盒布局不同，它是面向单轴的，网格布局针对二维布局进行了优化：在这类布局中，内容在两个维度上的对齐是必要的。

此外，由于其能够在网格中明确定位项目，网格布局允许在不需要更改相应标记的情况下进行显著的视觉布局结构转换。通过将媒体查询与控制网格容器及其子项布局的CSS属性相结合，作者可以根据设备形状、方向和可用空间的变化来调整其布局，同时保持其内容在不同展示形式中的更理想语义结构。

尽管许多布局既可以用网格布局表达，也可以用弹性盒布局表达，但它们各有特长。网格布局强制二维对齐，采用自上而下的布局方法，允许项目明确重叠，并具有更强大的跨越能力。弹性盒布局专注于轴内的空间分配，采用更简单的自下而上的布局方法，可以使用基于内容大小的换行系统来控制其次轴，并依赖底层标记层次来构建更复杂的布局。预计两者都将成为CSS作者的有价值且互补的工具。

1.1. 背景和动机

随着网站从简单的文档演变为复杂的交互式应用，文档布局技术（如浮动）并不一定适合应用布局。通过使用表格、JavaScript或对浮动元素的精确测量的组合，作者发现了实现所需布局的解决方法。适应可用空间的布局往往比较脆弱，当空间变得受限时，会导致不符合直觉的行为。作为替代方案，许多Web应用的作者选择了固定布局，无法利用屏幕上可用渲染空间的变化。

网格布局的功能解决了这些问题。它为作者提供了一种机制，可以使用一组可预测的尺寸行为将可用的布局空间划分为列和行。然后，作者可以精确地将应用的构建块元素放置并调整到由这些列和行的交叉点定义的网格区域中。以下示例说明了网格布局的自适应功能，以及它如何实现内容和样式的更清晰分离。

1.1.1. 适应可用空间的布局

网格布局可以用于智能调整网页内元素的大小。旁边的图例展示了一个具有五个主要组件的游戏布局：游戏标题、统计区域、游戏板、得分区域和控制区域。作者的意图是将游戏的空间划分如下：

  • 统计区域始终出现在游戏标题下方。
  • 游戏板出现在统计区域和标题的右侧。
  • 游戏标题和游戏板的顶部应始终对齐。
  • 当游戏达到其最小高度时，游戏板的底部和统计区域的底部对齐。在其他所有情况下，游戏板将扩展以利用所有可用空间。
  • 控制区域位于游戏板下方的中间。
  • 得分区域的顶部与控制区域的顶部对齐。
  • 得分区域位于统计区域下方。
  • 得分区域与统计区域下方的控制区域对齐。

以下网格布局示例展示了作者如何以声明方式实现所有的尺寸、位置和对齐规则。

/**
 * 通过在 网格项目 上声明网格来定义每个网格项目的空间
 * 的 网格容器。
 */
#grid {
  /**
   * 两列：
   *  1. 第一列根据内容大小，
   *  2. 第二列接收剩余空间
   *     （但永远不小于游戏板或游戏控制的最小尺寸，它们占据这一列 [图4]）
   *
   * 三行：
   *  3. 第一行根据内容大小，
   *  4. 中间行接收剩余空间
   *     （但永远不小于游戏板或统计区域的最小高度）
   *  5. 最后一行根据内容大小。
   */
  display: grid;
  grid-template-columns:
    /* 1 */ auto
    /* 2 */ 1fr;
  grid-template-rows:
    /* 3 */ auto
    /* 4 */ 1fr
    /* 5 */ auto;
}

/* 使用每个网格项目的 'grid-row' 和 'grid-column' 属性指定其位置 */
#title    { grid-column: 1; grid-row: 1; }
#score    { grid-column: 1; grid-row: 3; }
#stats    { grid-column: 1; grid-row: 2; align-self: start; }
#board    { grid-column: 2; grid-row: 1 / span 2; }
#controls { grid-column: 2; grid-row: 3; justify-self: center; }

<div id="grid">
  <div id="title">游戏标题</div>
  <div id="score">分数</div>
  <div id="stats">统计</div>
  <div id="board">游戏板</div>
  <div id="controls">控制</div>
</div>

注意：有多种方式可以指定网格的结构以及定位和调整网格项目，每种方式都针对不同的场景进行了优化。

1.1.2. 源顺序独立性

延续前面的例子，作者还希望游戏适应不同的设备。此外，无论是以纵向还是横向查看游戏时，游戏都应优化组件的布局（图6和图7）。通过将网格布局与媒体查询结合使用，作者可以使用相同的语义标记，但根据源顺序独立重新排列元素的布局，以实现在两种方向上所需的布局。

以下示例展示了网格布局命名将由网格项目占用的空间的能力。这使得作者可以在网格定义发生变化时避免重写网格项目的规则。

@media (orientation: portrait) {
  #grid {
    display: grid;

    /* 网格的行、列和区域通过 grid-template-areas 属性进行视觉定义。
     * 每个字符串是一行，每个词是一个区域。
     * 字符串中的词数量决定了列数。
     * 请注意，每个字符串中的词数量必须相同。 */
    grid-template-areas: "title stats" 
                         "score stats" 
                         "board board" 
                         "ctrls ctrls";

    /* 可以通过 grid-template-columns 和 grid-template-rows 属性指定列和行的大小。 */ 
    grid-template-columns: auto 1fr; 
    grid-template-rows: auto auto 1fr auto; 
  } 
}

@media (orientation: landscape) { 
  #grid { 
    display: grid;

    /* 再次使用模板属性定义同名区域，但这次区域位置不同，以更适应横向方向。 */ 
    grid-template-areas: "title board" 
                         "stats board" 
                         "score ctrls"; 

    grid-template-columns: auto 1fr; 
    grid-template-rows: auto 1fr auto; 
  } 
}

/* grid-area 属性将网格项目放置到网格的命名区域中。 */ 
#title    { grid-area: title } 
#score    { grid-area: score } 
#stats    { grid-area: stats } 
#board    { grid-area: board } 
#controls { grid-area: ctrls } 

<div id="grid"> 
  <div id="title">游戏标题</div> 
  <div id="score">分数</div> 
  <div id="stats">统计</div> 
  <div id="board">游戏板</div> 
  <div id="controls">控制</div> 
</div>

注意：网格布局的重新排序功能故意仅影响视觉渲染，保留基于源顺序的语音顺序和导航。这使得作者能够操纵视觉呈现，同时保持源顺序完整，并优化非CSS用户代理和线性模型（如语音和顺序导航）。

网格项目的放置和重新排序不得替代正确的源顺序，因为这可能破坏文档的可访问性。

2. 概述

本节不具有规范性。

网格布局通过使用网格来控制内容的布局：网格是水平和垂直线条的交叉集合，为网格容器的内容创建一个尺寸和定位的坐标系统。网格布局的特性包括：

  • 固定、灵活和基于内容的轨道尺寸调整函数
  • 通过正向（正）和反向（负）的数字网格坐标、命名的网格线和命名的网格区域实现显式项目定位，以及将项目自动放置到空白区域中，包括通过使用 order 重新排序
  • 空间敏感的轨道重复和自动添加行或列以容纳更多内容
  • 通过边距、沟槽和对齐属性控制对齐和间距
  • 能够重叠内容并通过使用 z-index 控制层次

网格容器可以根据需要嵌套或与弹性容器混合使用，以创建更复杂的布局。

2.1. 声明网格

网格轨道（行和列）可以通过显式网格属性明确声明和调整大小，也可以在项目放置在显式网格之外时隐式创建。grid 简写及其子属性定义了网格的参数。详见§ 7 网格定义

以下是一些网格声明示例：
  • 下面声明了一个具有四个命名区域的网格：H、A、B 和 F。第一列根据内容大小调整（auto），第二列占据剩余空间（1fr）。行默认采用基于内容的尺寸调整（auto），最后一行固定大小为 30px。 
    main {
  grid: "H    H "
        "A    B "
        "F    F " 30px
  /     auto 1fr;
}
  • 下面声明了一个网格，具有尽可能多的至少 5em 的行，以适应网格容器的高度（100vh）。该网格没有显式列；而是随着内容的增加添加列，结果列宽相等（1fr）。由于溢出到右侧的内容无法打印，打印时的替代布局添加了行。 
    main {
  grid: repeat(auto-fill, 5em) / auto-flow 1fr;
  height: 100vh;
}
@media print {
  main {
    grid: auto-flow 1fr / repeat(auto-fill, 5em);
  }
}
  • 下面声明了一个网格，具有5个均匀大小的列和3行，其中中间行占据所有剩余空间（并且至少足以容纳其内容）。 
    main {
  grid: auto 1fr auto / repeat(5, 1fr);
  min-height: 100vh;
}

2.2. 项目放置

网格容器的内容被组织为单独的网格项目（类似于弹性项目），然后将它们分配到预定义的区域中。可以通过网格放置属性使用坐标显式放置，也可以通过自动放置到空白区域中。详见§ 8 网格项目的放置

以下是使用grid-area简写的网格放置声明示例： 
grid-area: a;          /* 放置到命名的网格区域 “a”     */
grid-area: auto;       /* 自动放置到下一个空白区域    */
grid-area: 2 / 4;      /* 放置到第2行，第4列         */
grid-area: 1 / 3 / -1; /* 放置到第3列，跨越所有行 */
grid-area: header-start / sidebar-start / footer-end / sidebar-end;
                       /* 使用命名线进行放置            */

这些等同于以下grid-row + grid-column声明：

grid-row: a;                         grid-column: a;
grid-row: auto;                      grid-column: auto;
grid-row: 2;                         grid-column: 4;
grid-row: 1 / -1;                    grid-column: 3;
grid-row: header-start / footer-end; grid-column: sidebar-start / sidebar-end;

它们还可以进一步分解为grid-row-start/grid-row-end/grid-column-start/grid-column-end的长格式，例如：

grid-area: a;
/* 等同于 grid-row-start: a; grid-column-start: a; grid-row-end: a; grid-column-end: a; */

grid-area: 1 / 3 / -1;
/* 等同于 grid-row-start: 1; grid-column-start: 3; grid-row-end: -1; grid-column-end: auto; */

2.3. 网格尺寸调整

一旦网格项目已经被放置，就会计算网格轨道（行和列）的大小，考虑到网格定义中指定的内容大小和/或可用空间。

调整大小后的网格根据对齐属性，在网格容器中进行对齐，具体使用该网格容器的align-content 和 justify-content 属性。详见§ 10 对齐和间距

以下示例通过分配多余空间来调整所有列的间距，当网格小于100vh时，将网格在网格容器中居中对齐。 
main {
  grid: auto-flow 1fr / repeat(auto-fill, 5em);
  min-height: 100vh;
  justify-content: space-between;
  align-content: safe center;
}

最后，每个网格项目根据其尺寸[CSS2]和对齐属性[CSS-ALIGN-3]在其分配的网格区域内进行调整和对齐。

3. 网格布局的概念和术语

在网格布局中，网格容器的内容通过在网格中进行定位和对齐来布局。网格是水平和垂直的网格线的交叉集合，它将网格容器的空间划分为网格区域，网格项目（代表网格容器的内容）可以放置在其中。有两组网格线：一组定义沿着块轴运行的列，另一组定义沿着行轴运行的正交行。[CSS3-WRITING-MODES]

3.1. 网格线

网格线是网格的水平和垂直分隔线。网格线存在于列或行的两侧。它们可以通过数字索引或作者指定的名称进行引用。网格项目通过引用网格线使用网格放置属性确定其在网格中的位置。

以下两个示例都创建了三列网格线和四行网格线。

第一个示例展示了作者如何使用网格线编号定位网格项目：

#grid {
  display: grid;
  grid-template-columns: 150px 1fr;
  grid-template-rows: 50px 1fr 50px;
}

#item1 { grid-column: 2;
         grid-row-start: 1; grid-row-end: 4; }

第二个示例使用显式命名的网格线：

/* 与前一个示例等效，但使用了命名的线 */
#grid {
  display: grid;
  grid-template-columns: 150px [item1-start] 1fr [item1-end];
  grid-template-rows: [item1-start] 50px 1fr 50px [item1-end];
}

#item1 {
  grid-column: item1-start / item1-end;
  grid-row: item1-start / item1-end;
}

3.2. 网格轨道和单元格

网格轨道是网格列或网格行的通用术语——换句话说，它是两个相邻网格线之间的空间。每个网格轨道都分配了一个尺寸调整函数，该函数控制列或行的宽度或高度的增长，从而决定其边界网格线的距离。相邻的网格轨道可以通过沟槽分开，但其他情况下是紧密排列的。

网格单元格是网格行和网格列的交叉点。它是布局中可以引用的最小单位，用于定位网格项目。

在以下示例中，有两列和三行。第一列固定为150px。第二列使用灵活的尺寸调整，它是未分配空间的函数，因此随着网格容器的宽度变化而变化。如果网格容器的使用宽度为200px，则第二列宽度为50px。如果网格容器的使用宽度为100px，则第二列宽度为0px，放置在该列中的任何内容都会溢出网格容器。 
#grid {
  display: grid;
  grid-template-columns: 150px 1fr;  /* 两列 */
  grid-template-rows: 50px 1fr 50px; /* 三行 */
}

3.3. 网格区域

网格区域是用于布局一个或多个网格项目的逻辑空间。网格区域由一个或多个相邻的网格单元格组成。它由四条网格线围绕，分别位于网格区域的每一侧，并参与它所交叉的网格轨道的尺寸调整。网格区域可以通过grid-template-areas属性显式命名，也可以通过其边界网格线隐式引用。网格项目通过网格放置属性分配到一个网格区域。

/* 使用模板语法 */
#grid  {
  display: grid;
  grid-template-areas: ". a"
                       "b a"
                       ". a";
  grid-template-columns: 150px 1fr;
  grid-template-rows: 50px 1fr 50px;
}

#item1 { grid-area: a }
#item2 { grid-area: b }
#item3 { grid-area: b }

/* 将项目2和3对齐到网格区域 "b" 中的不同位置。  */
/* 默认情况下，网格项目会拉伸以适应其网格区域 */
/* 这些项目会彼此重叠。 */
#item2 { align-self: start; }
#item3 { justify-self: end; align-self: end; }

网格项目的网格区域构成了其布局的包含块。放置在同一网格区域中的网格项目不会直接影响彼此的布局。然而，间接地，使用内在尺寸调整函数占据网格轨道的网格项目可以影响该轨道的尺寸（因此也影响其边界的网格线的位置），这反过来可能影响其他网格项目的位置或尺寸。

4. 重新排序与可访问性

网格布局为作者提供了对文档的强大重排能力。然而，这并不能替代文档源代码的正确排序。order 属性和网格放置不会影响非视觉媒体中的排序（例如语音）。同样，视觉上重新排列网格项目不会影响顺序导航模式的默认遍历顺序（例如链接循环，参见tabindex[HTML]）。

作者必须仅将order和网格放置属性用于视觉上的重排，而非内容的逻辑重排。使用这些特性来执行逻辑重排的样式表是不符合规范的。

注意：这是为了确保非视觉媒体和不支持 CSS 的用户代理能够依赖于逻辑的源代码顺序，而网格布局的放置和排序特性仅用于调整视觉排列。（由于视觉感知是二维且非线性的，所需的视觉顺序并不总是等同于所需的阅读顺序。）

许多网页在标记上具有相似的形状，顶部有页眉，底部有页脚，中间有内容区域和一到两列额外的列。通常，页面源代码中的内容应优先于额外的列。然而，这使得一些常见的设计（例如简单地将额外的列放在左边，内容区域放在右边）变得难以实现。多年来，这个问题通过多种方式得到解决，通常在有两列额外列的情况下被称为“圣杯布局”。网格布局使这个示例变得非常简单。例如，考虑以下页面代码和所需的布局草图： 
<!DOCTYPE html>
<header>...</header>
<article>...</article>
<nav>...</nav>
<aside>...</aside>
<footer>...</footer>

使用网格布局可以轻松实现此布局：

body { display: grid;
       grid: "h h h"
             "a b c"
             "f f f";
       grid-template-columns: auto 1fr 20%; }
article { grid-area: b; min-width: 12em;     }
nav     { grid-area: a; /* 自动最小宽度 */ }
aside   { grid-area: c; min-width: 12em;     }

额外的好处是，默认情况下，这些列将具有等高，主内容将根据需要扩展以填充屏幕。此外，这还可以与媒体查询结合，在窄屏幕上切换到全垂直布局：

@media all and (max-width: 60em) {
  /* 过窄，无法支持三列布局 */
  main { display: block; }
}

为了在所有呈现模式中保持作者预期的顺序，创作工具（包括 WYSIWYG 编辑器以及基于 Web 的创作辅助工具）必须重新排列底层文档源，而不是使用order或网格放置属性进行重排，除非作者明确表示底层文档顺序（决定语音和导航顺序）应与视觉顺序不同步。

例如，一个工具可能提供拖放网格项目排列的功能，以及根据屏幕尺寸范围处理媒体查询以进行替代布局的功能。

由于大多数情况下，重新排序应影响所有屏幕范围以及导航和语音顺序，因此该工具将通过同时重新排列 DOM 层来匹配结果的拖放视觉排列。然而，在某些情况下，作者可能希望为不同的屏幕尺寸提供不同的视觉排列。该工具可以通过结合使用网格放置属性和媒体查询来提供此功能，但同时将最小屏幕尺寸的排列与底层 DOM 顺序关联（因为这最有可能是逻辑线性呈现顺序），而在其他尺寸范围内使用网格放置属性重新排列视觉呈现。

此工具将符合规范，而仅使用网格放置属性来处理拖放网格重新排列的工具（无论其实现方式多么方便）将不符合规范。

5. 网格容器

5.1. 建立网格容器: grid 和 inline-grid display 值

名称: display
新值: grid | inline-grid
grid
这个值使元素生成一个网格容器框， 当它在流布局中时，表现为块级。
inline-grid
这个值使元素生成一个网格容器框， 当它在流布局中时，表现为内联级。

网格容器为其内容建立了一个独立的网格格式化上下文。这与建立独立的块格式化上下文相同，只是使用网格布局而不是块布局：浮动不会侵入网格容器，网格容器的外边距不会与其内容的外边距合并。网格容器的内容布局为网格，网格线形成了每个网格项目的包含块的边界。

网格容器不是块容器，因此一些设计用于假设块布局的属性在网格布局的上下文中不适用。特别是：

  • float 和 clear 对网格项目没有影响。然而，float属性仍然影响display在网格容器的子元素上的计算值，因为这是在网格项目确定之前发生的。
  • vertical-align 对网格项目没有影响。
  • ::first-line 和 ::first-letter 伪元素不适用于网格容器，并且网格容器不会为其祖先贡献第一格式化行或第一个字母。

如果元素的指定display为inline-grid，且该元素是浮动的或绝对定位的，则display的计算值为grid。因此，CSS 2.1 第9.7章中的表格应包含一个额外的行，"指定值"列中为inline-grid，"计算值"列中为grid。

5.2. 尺寸网格容器

请参阅[CSS-SIZING-3]以了解本节中的术语定义。

网格容器使用其所参与的格式化上下文的规则进行尺寸调整：

  • 作为块级盒子在 块级格式化上下文中， 它的尺寸计算方式类似于建立格式化上下文的块盒子， 其auto 行内尺寸 的计算方式与非替换的块盒子相同。
  • 作为一个行内格式化上下文中的行内级盒子， 它的尺寸计算方式与原子行内级盒子（如inline-block）相同。

在内联和块级格式上下文中，网格容器的 auto 块尺寸为其 max-content 大小。

块级布局规范应该定义这一点，但它尚未编写。

max-content 大小（ min-content 大小） 的 网格容器是相应轴上 网格容器的轨道尺寸（包括槽）之和， 当网格在max-content 约束（ min-content 约束）下进行大小调整时。

5.3. 可滚动网格溢出

overflow属性适用于网格容器。

与包含在内在尺寸中（见上文）一样，网格也包含在网格容器的可滚动溢出区域中。

注意: 当网格容器是滚动容器时，需注意与内边距的交互：根据需要添加的额外内边距将定义为添加到可滚动溢出矩形中，以实现可滚动内容的place-content: end对齐。参见CSS Overflow 3 §2.2 可滚动溢出。

5.4. 限制大网格

由于内存有限，用户代理（UA）可能会将隐式网格的可能大小限制在UA定义的范围内（应容纳范围为[-10000, 10000]的线），将该范围外的所有线删除。如果网格项目放置在该限制之外，其网格区域必须被限制在该有限网格内。

要限制网格区域：

  • 如果网格区域会跨越有限网格之外，则其跨度被限制为有限网格的最后一条线。

  • 如果网格区域将完全放置在有限网格之外，其跨度必须被截断为1，并且区域重新定位到该网格一侧的最后一个网格轨道中。

例如，如果UA只支持每个维度最多1000条轨道的网格，以下定位属性： 
.grid-item {
  grid-row: 500 / 1500;
  grid-column: 2000 / 3000;
}

最终等效于：

.grid-item {
  grid-row: 500 / 1001;
  grid-column: 1000 / 1001;
}

6. 网格项目

粗略地说，网格项目是网格容器的在流内容的表示框。

每个网格容器的在流子元素都会变成一个网格项目， 每个连续的子文本运行序列都包裹在一个匿名的块容器 网格项目中。 然而，如果整个子文本运行序列仅包含空白字符（即可以被white-space属性影响的字符）， 那么它将不呈现（就像其文本节点设置为display:none时一样）。

网格项目示例：

<div style="display: grid">

  <!-- 网格项目：块级子元素 -->
  <div id="item1">block</div>

  <!-- 网格项目：浮动元素；浮动被忽略 -->
  <div id="item2" style="float: left;">float</div>

  <!-- 网格项目：内联内容周围的匿名块框 -->
  anonymous item 3

  <!-- 网格项目：内联子元素 -->
  <span>
    item 4
    <!-- 网格项目不会 在块周围分割 -->
    <q style="display: block" id=not-an-item>item 4</q>
    item 4
  </span>
</div>

注意: 元素间的空白字符会消失： 它不会成为自己的网格项目， 即使元素间的文本确实会被包裹在匿名网格项目中。

注意: 匿名项目的框是不可样式化的， 因为没有可以分配样式规则的元素。 其内容将继承来自网格容器的样式（如字体设置）。

6.1. 网格项目显示

一个网格项目为其内容建立独立的格式化上下文。 然而，网格项目是网格级框，而不是块级框： 它们参与其容器的网格格式化上下文， 而不是块格式化上下文。

如果元素最近的祖先元素的计算display值 （跳过display:contents祖先）是grid或inline-grid， 那么元素自己的display值将被块化。 （参见CSS2.1§9.7[CSS2]和CSS Display 3 §2.7 自动框类型转换了解这种display值转换的详细信息。）

注意: 即使grid或inline-grid元素最终没有生成网格容器框， 例如当它被替换或位于display:none子树中，块化仍然会发生。

注意: 一些display值通常会触发在原始框周围创建匿名框。 如果这样的框是网格项目， 它会先被块化，因此不会发生匿名框创建。 例如，两个连续的网格项目设置为display: table-cell会变成两个独立的display: block 网格项目， 而不是被包裹成一个匿名表。

6.2. 网格项目尺寸调整

一个网格项目的尺寸是在其网格区域定义的包含块中调整的。

网格项目在给定维度中对于自动尺寸的计算根据其自身对齐值而有所不同：

normal

如果网格项目是一个具有相关维度的替换元素并且具有自然尺寸 （或具有首选宽高比且在另一个维度中具有尺寸）， 那么网格项目将按照align-self: start进行尺寸调整（与CSS 2 §10.3.4 块级替换元素的宽度计算规则一致）。

否则， 如果网格项目具有首选宽高比， 则网格项目将按照块级框的方式进行尺寸调整。

否则， 网格项目将按照stretch进行尺寸调整。

stretch

使用非替换框的内联尺寸计算规则 （在CSS 2 §10.3.3 块级非替换元素的正常流中的宽度计算中定义）。

注意: 这可能会扭曲具有首选宽高比的项目的宽高比， 如果其尺寸在另一个轴上也受到限制。

所有其他值

将项目尺寸调整为fit-content。

下表总结了网格项目的自动尺寸调整行为：
网格项目自动尺寸行为总结
对齐 非替换元素尺寸 替换元素尺寸
normal 填充网格区域 使用自然尺寸
stretch 填充网格区域 填充网格区域
start/center/等 fit-content 尺寸调整（类似浮动） 使用自然尺寸

注意: auto值 对min-width和min-height的影响 类似于它如何影响弹性项目的主尺寸。 参见§ 6.6 网格项目的自动最小尺寸。

6.3. 重新排序的网格项目：order属性

order属性也适用于网格项目。 它影响它们的自动布局和绘制顺序。

与重新排序的弹性项目一样， order属性只能在视觉顺序需要与语音和导航顺序不同步时使用； 否则，应该重新排列底层文档源。 参见CSS Flexbox 1 §5.4.1 重新排序和可访问性，在[CSS-FLEXBOX-1]中。

6.4. 网格项目的边距和填充

由于相邻的网格项目被独立包含在其网格区域中， 相邻网格项目的边距不会折叠。

百分比边距和填充在网格项目上， 与块框上的行为类似， 它们相对于其内联尺寸的包含块进行解析， 例如左/右/上/下百分比都相对于其包含块的宽度在水平书写模式中解析。

自动边距会扩展以吸收相应维度中的额外空间， 因此可以用于对齐。 参见§ 10.2 使用自动边距进行对齐

6.5. Z轴排序：z-index属性

网格项目在定位到相交的网格区域时可以重叠， 或者由于负边距或定位，即使在不相交的区域中也可以重叠。 网格项目的绘制顺序与内联块的顺序完全相同[CSS2]， 不同之处在于使用order修改的文档顺序代替了原始文档顺序， 并且当z-index值不为auto时， 即使position为static（行为与position为relative的情况完全相同）时也会创建堆叠上下文。 因此，可以轻松使用z-index属性来控制网格项目的Z轴顺序。

注意：定位在网格项目之外的后代仍然参与任何由该网格项目建立的堆叠上下文。

下图显示了几个重叠的网格项目， 使用隐式源顺序和显式z-index来控制它们的堆叠顺序。 
<style type="text/css">
#grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
  grid-template-rows: 1fr 1fr
}
#A { grid-column: 1 / span 2; grid-row: 2; align-self: end; }
#B { grid-column: 1; grid-row: 1; z-index: 10; }
#C { grid-column: 2; grid-row: 1; align-self: start; margin-left: -20px; }
#D { grid-column: 2; grid-row: 2; justify-self: end; align-self: start; }
#E { grid-column: 1 / span 2; grid-row: 1 / span 2;
     z-index: 5; justify-self: center; align-self: center; }
</style>

  <div id="A">A</div>
  <div id="B">B</div>
  <div id="C">C</div>
  <div id="D">D</div>
  <div id="E">E</div>
</div>

6.6. 网格项目的自动最小尺寸

注意：本节中使用的许多尺寸术语（以及规范的其余部分）在CSS 内在和外在尺寸中定义[CSS-SIZING-3]。

为了为网格项目提供更合理的默认最小尺寸， 如果满足以下所有条件，则其在给定轴上的自动最小尺寸的使用值为基于内容的最小尺寸：

  • 它不是滚动容器

  • 它在该轴上至少跨越一个轨道，且其最小轨道尺寸函数为auto

  • 如果它在该轴上跨越多个轨道，则这些轨道中没有一个是弹性轨道

否则，自动最小尺寸通常为零。

基于内容的最小尺寸在给定维度上的网格项目是其指定尺寸建议，如果存在； 否则为其转移尺寸建议； 如果都不存在，则为其内容尺寸建议，见下文。 然而， 如果在给定维度上 网格项目仅跨越那些具有固定最大轨道尺寸函数的网格轨道， 则其在该维度上的指定尺寸建议和内容尺寸建议（以及从该维度输入到相对维度的转移尺寸建议） 进一步限制为小于或等于网格区域最大尺寸中的拉伸适应值，表示为这些网格轨道的最大轨道尺寸函数之和加上任何中间的固定间隙。

在所有情况下，尺寸建议还会受到受影响轴上最大尺寸的进一步限制，如果它是明确的。

注意：传递给fit-content()的参数不会以与固定最大轨道尺寸函数相同的方式限制基于内容的最小尺寸。

内容尺寸建议，指定尺寸建议，和转移尺寸建议在此计算中会考虑相关的最小/最大/首选尺寸属性， 以防止基于内容的最小尺寸干扰任何作者提供的约束，定义如下：

指定尺寸建议
如果项目在相关轴上的首选尺寸是明确的， 则指定尺寸建议为该尺寸。 否则未定义。
转移尺寸建议
如果项目具有首选纵横比，且在相对轴上的首选尺寸是明确的， 则转移尺寸建议为该尺寸（如果它们是明确的，且受最小和最大尺寸的限制），通过纵横比转换。 否则未定义。
内容尺寸建议
内容尺寸建议是相关轴上的最小内容尺寸，若有首选纵横比，受任何明确的相对轴的最小和最大尺寸限制，通过纵横比转换。

为了计算盒子的内在尺寸（例如盒子的最小内容尺寸），基于内容的最小尺寸使得该轴上的盒子尺寸变得不明确（即使例如其宽度属性指定了明确的尺寸）。这意味着根据此尺寸计算的百分比将表现为auto。

然而，尽管在某些情况下这可能需要额外的布局遍历以重新解析百分比，但此值并不会阻止项目内百分比尺寸的解析。

注意，虽然基于内容的最小尺寸通常是合适的，有助于防止内容重叠或溢出其容器，但在某些情况下并不适用：

特别是，如果网格布局用于文档的主要内容区域，最好设置一个明确的字体相对最小宽度，例如min-width: 12em。 基于内容的最小宽度可能会导致大表格或大图像拉伸整个内容区域的大小，可能进入溢出区域，从而使文本行变得过长和难以阅读。

另请注意，当基于内容的尺寸用于包含大量内容的项目时，布局引擎必须遍历所有这些内容才能确定其最小尺寸，而如果作者设置了明确的最小值，则无需执行此操作。 （对于包含少量内容的项目，此遍历是微不足道的，因此不是性能问题。）

7. 定义网格

7.1. 显式网格

三个属性grid-template-rows，grid-template-columns和grid-template-areas一起定义了一个显式网格， 它通过指定其显式网格轨道来定义网格容器。 最终的网格可能会因为放置在显式网格外的网格项目而变得更大； 在这种情况下，将会创建隐式轨道， 这些隐式轨道将由grid-auto-rows和grid-auto-columns属性进行尺寸调整。

显式网格的大小由以下两者中较大的值决定： 由grid-template-areas定义的行/列数 和由grid-template-rows/grid-template-columns调整的行/列数。 由grid-template-areas定义但未由grid-template-rows/grid-template-columns调整尺寸的任何行/列 将从grid-auto-rows/grid-auto-columns属性中获取其尺寸。 如果这些属性没有定义任何显式轨道， 则显式网格仍然在每个轴上包含一个网格线。

在网格放置属性中的数字索引是从显式网格的边缘开始计算的。 正数索引从起始侧计算 （从最左侧的起始显式线开始，起始值为1）， 而负数索引从结束侧计算 （从最右侧的结束显式线开始，起始值为-1）。

grid和grid-template属性是速记， 它们可以用于同时设置所有三个显式网格属性 （grid-template-rows，grid-template-columns和grid-template-areas）。 grid速记还会重置控制隐式网格的属性， 而grid-template属性不会更改它们。

7.2. 显式轨道尺寸：grid-template-rows 和 grid-template-columns 属性

名称： grid-template-columns，grid-template-rows
值： none|<track-list>|<auto-track-list>
初始值： none
适用于： 网格容器
继承： 否
百分比： 参考内容区域的相应尺寸
计算值： 关键字none 或计算的轨道列表
规范顺序： 按语法
动画类型： 如果列表长度匹配，则按计算值类型处理计算的轨道列表中的每个项目（见§ 7.2.5 轨道列表的计算值 和 § 7.2.3.3 repeat()的插值/组合）；否则为离散

这些属性指定了作为空格分隔的轨道列表， 该列表定义了网格的线名称和轨道尺寸函数。 grid-template-columns属性指定了网格列的轨道列表， 而grid-template-rows则指定了网格行的轨道列表。

值有以下含义：

none
表示此属性不会创建任何显式网格轨道， （尽管显式网格轨道仍然可以由grid-template-areas创建）。

注意：在没有显式网格的情况下，任何行/列都会被隐式生成， 其大小将由grid-auto-rows和grid-auto-columns属性确定。

<track-list> | <auto-track-list>
指定了作为系列的轨道列表， 该列表包含轨道尺寸函数和线名称。 每个轨道尺寸函数可以被指定为长度， 百分比网格容器的大小， 占用列或行的内容的度量， 或网格中自由空间的一部分。 它也可以使用minmax()符号指定为一个范围， 该符号可以将之前提到的任何机制结合起来， 以为列或行指定单独的最小和最大轨道尺寸函数。

轨道列表的语法为：

<track-list>          = [ <line-names>? [ <track-size> | <track-repeat> ] ]+ <line-names>?
<auto-track-list>     = [ <line-names>? [ <fixed-size> | <fixed-repeat> ] ]* <line-names>? <auto-repeat>
                        [ <line-names>? [ <fixed-size> | <fixed-repeat> ] ]* <line-names>?
<explicit-track-list> = [ <line-names>? <track-size> ]+ <line-names>?

<track-size>          = <track-breadth> | minmax( <inflexible-breadth> , <track-breadth> ) | fit-content( <length-percentage> )
<fixed-size>          = <fixed-breadth> | minmax( <fixed-breadth> , <track-breadth> ) | minmax( <inflexible-breadth> , <fixed-breadth> )
<track-breadth>       = <length-percentage> | <flex> | min-content | max-content | auto
<inflexible-breadth>  = <length-percentage> | min-content | max-content | auto
<fixed-breadth>       = <length-percentage>
<line-names>          = '[' <custom-ident>* ']'

这些组件值的定义如下…

7.2.1. 轨道尺寸

<length-percentage>
一个非负的长度或百分比，按照CSS3值定义。[CSS-VALUES-3]

<percentage> 值相对于 内部 行内尺寸 网格容器的列轨道，和 内部 块尺寸 网格容器的行轨道。 如果 网格容器 的大小取决于其轨道的大小， 则 <percentage> 必须被视为 自动， 用于计算网格容器的固有大小，然后根据结果的 网格容器 大小解析以布局网格及其项目。

<flex>
带有单位fr 的非负维度，指定轨道的 弹性系数。 每个 <flex> 大小的轨道根据其 弹性系数 占用剩余空间的一部分。 例如，给定一个轨道列表 1fr 2fr， 轨道将分别占用 剩余空间 的1/3和2/3。 详情见 § 7.2.4 弹性长度：fr单位。

注意：如果弹性系数的总和小于1， 它们将仅占用相应比例的 剩余空间，而不是扩展到填满整个空间。

当出现在 minmax() 表达式之外时， 意味着一个自动最小值（即“minmax(自动, <flex>)”）。

minmax(最小值, 最大值)
定义一个大小范围， 大于或等于 最小值 并且小于或等于 最大值。 如果 最大值 小于 最小值，则 最大值 将被 最小值 限制（实际上相当于 minmax(min, min)）。 作为最大值，<flex> 值设置轨道的 弹性系数； 它作为最小值是无效的。

注意：本规范的未来版本可能允许 <flex> 最小值， 并且将更新 轨道尺寸算法 以正确处理此问题。

自动
作为 最大值：表示占用 最大内容贡献 的 网格项目 的最大值 网格轨道； 然而，与 最大内容 不同， 允许通过对齐内容 和 justify-content 属性进行扩展。

作为 最小值：表示占用 最小尺寸 （由 最小宽度/最小高度 指定） 的 网格项目 占用的 网格轨道。 （这通常等于 最小内容 的最小值，详见 § 6.6 网格项目的自动最小尺寸。）

当出现在 minmax() 表达式之外时： 相当于 minmax(auto, auto)， 代表上面描述的最小值和最大值之间的范围。 （在最基本的情况下，这类似于 minmax(最小内容, 最大内容)，但具有额外的功能。）

最大内容
表示 最大内容贡献 的 网格项目 占用的 网格轨道 的最大值。
最小内容
表示 最小内容贡献 的 网格项目 占用的 网格轨道 的最小值。
fit-content( <length-percentage> )
代表公式 max(最小值, min(限制, 最大内容))， 其中 最小值 代表 自动 最小值 （这通常等于 最小内容 的最小值）， 限制 是传递给 轨道尺寸函数 的参数， 并根据此参数计算为 minmax(自动, 最大内容) 和 minmax(自动, 限制) 中较小的一个。
给定以下 grid-template-columns 声明： 
grid-template-columns: 100px 1fr max-content minmax(min-content, 1fr);

创建了五条网格线：

  1. 在 网格容器 的起始边缘。
  2. 从 网格容器 的起始边缘100px。
  3. 从前一条线的距离等于 空闲空间（网格容器 的宽度减去不可伸缩的 网格轨道 的宽度）的一半。
  4. 从前一条线的距离等于属于这两条线之间列的任何 网格项目 的最大尺寸。
  5. 从前一条线的距离至少与属于这两条线之间列的任何 网格项目 的最大最小尺寸一样大， 但不超过 空闲空间 的另一半。

如果不可伸缩的尺寸 （100px, max-content 和 min-content） 的总和大于 网格容器 的宽度， 最后一条 网格线 将是距 网格容器 起始边缘等于其总和的距离（1fr 的大小都解析为 0）。 如果总和小于 网格容器 的宽度， 最后一条 网格线 将正好在 网格容器 的结束边缘。 这通常在 <flex> 值存在于 网格轨道 尺寸中时适用。

更多有效的 网格轨道 定义示例： 
/* 有效轨道定义的示例 */
grid-template-rows: 1fr minmax(min-content, 1fr);
grid-template-rows: 10px repeat(2, 1fr auto minmax(30%, 1fr));
grid-template-rows: calc(4em - 5px);

注意：网格的大小不仅仅是轨道尺寸的总和， 因为 行间距、列间距 和 justify-content、align-content 可能会在轨道之间增加额外的空间。

7.2.2. 命名网格线：[<custom-ident>*] 语法

虽然网格线总是可以通过其数字索引引用， 但线名称可以使网格放置属性 更易于理解和维护。线名称可以通过 显式分配给 grid-template-rows 和 grid-template-columns 属性， 或者通过隐式分配给命名网格区域，使用 grid-template-areas 属性。

例如，以下代码为网格中的所有线赋予了有意义的名称。 请注意，某些线有多个名称。 
#grid {
  display: grid;
  grid-template-columns: [first nav-start] 150px [main-start] 1fr [last];
  grid-template-rows: [first header-start] 50px [main-start] 1fr [footer-start] 50px [last];
}

线名称不能为span或auto， 即在<custom-ident> 中的<line-names> 生产 排除关键字 span 和 auto。

7.2.3. 重复行和列：repeat() 表示法

repeat() 表示法表示轨道列表的重复片段， 允许将显示重复模式的大量列或行以更紧凑的形式编写。

此示例显示了两种编写相同网格定义的等效方法。 两个声明都生成四个宽度为250px的“主”列， 由10px的“间隙”列包围。 
grid-template-columns: 10px [col-start] 250px [col-end]
                       10px [col-start] 250px [col-end]
                       10px [col-start] 250px [col-end]
                       10px [col-start] 250px [col-end] 10px;
/* 与上面相同，除了更容易编写 */
grid-template-columns: repeat(4, 10px [col-start] 250px [col-end]) 10px;
7.2.3.1. repeat() 语法

repeat() 语法的一般形式大致为：

repeat( [ <integer [1,∞]> | auto-fill | auto-fit ] , <track-list> )

第一个参数指定重复次数。 第二个参数是轨道列表， 重复该次数。 但是，有一些限制：

  • repeat() 表示法不能嵌套。

  • 自动重复（auto-fill 或 auto-fit） 不能与固有或灵活尺寸结合使用。

因此，repeat() 表示法的精确语法有几种形式：

<track-repeat> = repeat( [ <integer [1,∞]> ] , [ <line-names>? <track-size> ]+ <line-names>? )
<auto-repeat>  = repeat( [ auto-fill | auto-fit ] , [ <line-names>? <fixed-size> ]+ <line-names>? )
<fixed-repeat> = repeat( [ <integer [1,∞]> ] , [ <line-names>? <fixed-size> ]+ <line-names>? )

  • <track-repeat> 变体可以表示任何<track-size>的重复， 但仅限于固定次数的重复。

  • <auto-repeat> 变体可以自动重复以填充空间， 但需要确定的轨道尺寸，以便可以计算重复次数。 它只能在轨道列表中出现一次， 但同一个轨道列表也可以包含<fixed-repeat>。

如果repeat() 函数最终将两个<line-names>放置在相邻位置， 则名称列表将合并。 例如，repeat(2, [a] 1fr [b]) 等同于 [a] 1fr [b a] 1fr [b]。

7.2.3.2. 填充重复：auto-fill 和 auto-fit 重复

当auto-fill作为重复次数给出时， 如果网格容器在相关轴上具有 确定的大小或最大大小， 那么重复次数是不会导致网格溢出 其内容框的最大可能正整数（将每个轨道视为其 最大轨道尺寸函数，如果是 确定的，否则视为其最小轨道尺寸函数， 并且如果两者都是确定的，则取 最大轨道尺寸函数与最小轨道尺寸函数的最小值， 并考虑间隙）； 如果任何重复次数都会溢出，则为1次重复。 否则，如果网格容器在相关轴上具有 确定的最小大小， 则重复次数是满足该最小要求的最小可能正整数。 否则，指定的轨道列表 仅重复一次。

例如，以下代码将创建 尽可能多的25字符宽列，以适应窗口宽度。 如果有任何剩余空间， 它将分配给25字符宽的列。 
body {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(25ch, 1fr));
}

auto-fit 关键字的行为与auto-fill相同， 除了在网格项目放置之后，任何空的重复轨道都会折叠。 空轨道是指没有流入网格项目放置到其中或跨越它的轨道。 （这可能导致所有轨道都被折叠， 如果它们都是空的。）

折叠轨道 被视为具有 固定的轨道尺寸函数为0px， 并且其任一侧的间隙——包括通过分布对齐分配的任何空间——折叠。

为了找到自动重复轨道的数量， UA必须将轨道尺寸向下取整为UA指定的值， 以避免除以零。 建议该最小值为1px。

7.2.3.3. 插值/组合 repeat()

如果两个repeat() 表示法具有相同的第一个参数（重复计数） 并且其第二个参数（轨道列表）中的轨道数量相同， 它们通过 合并每个组件的计算轨道列表 按计算值合并 （就像合并顶层轨道列表一样）。 否则，它们会离散地合并。

7.2.4. 弹性长度：fr 单位

弹性长度 或 <flex> 是具有 fr 单位的维度， 代表 剩余空间 的一部分，位于 网格容器 中。 使用 fr 单位调整大小的轨道称为弹性轨道，因为它们会根据剩余空间进行调整，类似于弹性项目在 弹性容器中填充空间的方式。

在所有非弹性轨道尺寸函数达到最大值后， 剩余空间的分配会发生。 此类行或列的总大小从可用空间中减去，得到剩余空间， 然后按照它们的弹性系数在弹性大小的行和列之间分配。

每个列或行的剩余空间份额可以计算为列或行的 <flex> * <剩余空间> / <所有 弹性系数的总和>。

<flex> 值在 0fr 和 1fr 之间具有某种特殊行为： 当弹性系数的总和小于 1 时， 它们将占用少于 100% 的剩余空间。

轨道的 <flex> 值实际上是对剩余空间某个比例的请求， 其中 1fr 意味着“100% 的剩余空间”； 然后，如果该轴上的轨道总共请求的超过 100%， 请求将重新平衡以保持相同的比例，但正好使用 100% 的空间。 但是，如果轨道请求的空间少于总量 （例如三个轨道每个为 .25fr）， 那么它们将各自获得请求的确切空间 （每个占 25% 的剩余空间， 最终剩余 25% 的空间未填满）。 详见 § 11.7 扩展弹性轨道，了解剩余空间分配的具体细节。

这种模式对于当fr 值接近零时的连续行为是必需的 （这意味着轨道希望获得没有剩余空间）。 如果没有这种行为，1fr 轨道将占用所有剩余空间； 但 0.1fr 轨道也是如此， 以及 0.01fr 轨道， 等等， 直到该值小到不足以显示为零， 并且该轨道突然不再占用任何剩余空间。 通过这种行为， 轨道会随着其弹性系数小于 1fr 而逐渐占用较少的剩余空间， 平滑过渡到零时不再占用任何剩余空间。

除非这种“部分填充”行为是特别想要的， 否则作者应坚持使用值 ≥ 1； 例如，使用 1fr 和 2fr 通常比使用 .33fr 和 .67fr 更好， 因为它们在添加或删除轨道时更有可能按预期行为。

当可用空间是无限时 （当网格容器的宽度或高度是不确定的时会发生这种情况）， 使用弹性大小的网格轨道会根据内容调整大小，同时保持各自的比例。 每个弹性大小的 网格轨道 的使用大小通过 确定每个弹性大小的 网格轨道 的最大内容大小并将该大小除以相应的 弹性系数来计算“假设的 1fr 大小”。 这些中的最大值用作已解析的 1fr 长度（弹性分数）， 然后乘以每个 网格轨道 的 弹性系数，以确定其最终大小。

注意： <flex> 值不是 <长度>， （也不与 <长度> 兼容， 如某些<百分比>值）， 因此不能在calc() 表达式中表示或与其他单位类型结合使用。

7.2.5. 轨道列表的计算值

计算轨道列表是一个 列表，在线名称集和轨道部分之间交替， 第一个和最后一个项目是 线名称集。

线名称集是标识符的（可能为空的） 有序集合，代表线名称。

轨道部分可以是：

  • minmax() 函数符号，表示单个轨道的大小， 每个<长度-百分比> 都被计算出来

  • repeat() 函数符号，表示一个重复的轨道列表部分， 它的<整数>被计算， 它的<轨道列表>表示为计算轨道列表

7.2.6. 轨道列表的解析值

grid-template-rows 和 grid-template-columns 属性是 解析值特殊情况属性。 [CSSOM]

  • 相邻的线名称合并为一个括号集。

<string>+

将字符串标记为以下标记列表， 使用最长匹配语义：

  • 由 名称代码点组成的序列， 代表具有名称的 命名单元格标记。
  • 一个或多个“.”（U+002E 句号）的序列， 代表一个空单元格标记。

7.3.1. 模板字符串的序列化

在序列化 指定值 或 计算值 时，对于 <string> 类型的 grid-template-areas 值， 每个空单元格标记都会被序列化为一个 "."（U+002E 句号）， 并且连续的单元格标记之间用一个空格（U+0020 空格）分隔， 其他所有空白字符都会被省略。

7.3.2. 隐式分配的线名称

grid-template-areas 属性会根据模板中的 命名网格区域 生成隐式分配的线名称。 对于每个命名网格区域foo，会生成四个 隐式分配的线名称： 两个名为 foo-start 的名称，分别为该 命名网格区域的行起始线和列起始线命名， 以及两个名为 foo-end 的名称，分别为该 命名网格区域的行结束线和列结束线命名。

这些隐式分配的线名称与其他 线名称的行为完全相同， 只是它们不会出现在grid-template-rows/ grid-template-columns的值中。 即使定义了具有相同名称的 显式分配的线名称， 隐式分配的线名称也只是具有相同名称的更多线名称。

7.3.3. 隐式命名区域

由于 命名网格区域 是通过其生成的 隐式分配的线名称引用的， 明确添加相同形式的命名线（foo-start/ foo-end） 有效地创建了一个 命名网格区域。 这种隐式命名区域不会出现在 grid-template-areas的值中， 但仍可以通过网格放置属性引用。

7.5. 隐式网格

grid-template-rows、grid-template-columns 和 grid-template-areas 属性定义了一个固定数量的轨道， 它们构成了显式网格。 当 网格项目 被放置在这些边界之外时， 网格容器通过在 网格中添加 隐式网格轨道生成 隐式网格线。 这些线与 显式网格一起构成了 隐式网格。 grid-auto-rows 和 grid-auto-columns 属性为这些 隐式网格轨道定义了大小， 以及通过 grid-template-areas创建但未通过 grid-template-rows或 grid-template-columns显式指定大小的任何 显式网格轨道。

grid-auto-flow 属性控制未显式设置位置的 网格项目的自动放置。 一旦显式网格填满 （或者如果没有 显式网格）， 自动放置还会导致生成隐式网格轨道。

grid 简写属性可以同时设置 隐式网格属性 (grid-auto-flow、grid-auto-rows 和 grid-auto-columns) 以及显式网格属性。

7.7. 自动放置：grid-auto-flow 属性

名称： grid-auto-flow
值： [ row | column ] || dense
初始值： row
适用于： 网格容器
是否继承： 否
百分比： n/a
计算值： 指定的关键词
规范顺序： 按语法
动画类型： 离散

未显式放置的网格项目 将通过自动放置算法 被自动放置在网格容器 中的空位。grid-auto-flow 控制着 自动放置算法的工作方式， 指定自动放置的项目如何流入网格中。 请参见 § 8.5 网格项目放置算法 了解自动放置算法的详细工作原理。

row
自动放置算法通过依次填充每一行来放置项目， 并根据需要添加新行。 如果未指定row或column， 则默认假定为row。
column
自动放置算法通过依次填充每一列来放置项目， 并根据需要添加新列。
dense
如果指定了该值，自动放置算法 将使用一种“紧凑”算法， 尝试在网格中较小的项目出现时填补较早出现的空洞。 这可能导致项目的顺序被打乱，以填补较大项目留下的空洞。

如果未指定此值，将使用一种“稀疏”算法， 该算法在放置项目时只会在网格中向“前”移动，绝不回溯填补空洞。 这样可以确保所有自动放置的项目按顺序排列， 即使这会留下可以被后续项目填补的空洞。

注意： 未来版本的该模块预计将添加一个值，用于将自动放置的项目放在一个单一的“默认”单元格中。

grid 属性是一个简写，用于设置所有 显式网格属性（grid-template-rows、grid-template-columns和grid-template-areas）， 以及所有隐式网格属性（grid-auto-rows、grid-auto-columns和grid-auto-flow）， 在一个声明中完成。 （它不会重置gutter属性。） 其语法与grid-template匹配， 加上用于定义自动流布局的额外语法形式：

<'grid-template-rows'> / [ auto-flow && dense? ] <'grid-auto-columns'>?
[ auto-flow && dense? ] <'grid-auto-rows'>? / <'grid-template-columns'>
设置自动流， 通过显式设置一个轴的轨道 （指定 grid-template-rows 或 grid-template-columns， 并将另一个设置为none）， 并指定如何在另一个轴上自动重复轨道 （指定 grid-auto-rows 或 grid-auto-columns， 并将另一个设置为 auto）。grid-auto-flow 也会设置为 row 或 column， 以及 dense（如果已指定）。

所有其他 grid 的子属性 将重置为初始值。

注意： 需要注意的是，在一个 grid 声明中，你只能指定显式或隐式网格属性。 没有指定的子属性会被设置为它们的初始值， 这与简写的常规行为一致。

除了接受 grid-template 简写语法来设置 显式网格外， grid 简写还可以轻松设置自动布局网格的参数。 例如，grid: auto-flow 1fr / 100px; 等价于 
grid-template: none / 100px;
grid-auto-flow: row;
grid-auto-rows: 1fr;
grid-auto-columns: auto;

同样地，grid: none / auto-flow 1fr 等价于

grid-template: none;
grid-auto-flow: column;
grid-auto-rows: auto;
grid-auto-columns: 1fr;

8. 放置网格项

每个网格项都与一个网格区域相关联， 这是网格项所占据的相邻网格单元的矩形集合。 该网格区域定义了网格项的包含块，其中的自对齐属性（justify-self 和 align-self）决定其实际位置。 网格项占据的单元格也会影响网格的行和列的大小， 这在§ 11 网格大小调整中定义。

网格项的网格区域在网格中的位置由其放置定义， 该放置由网格位置和网格跨度组成：

网格位置
网格项在 网格中的每个轴上的位置。 一个网格位置可以是确定的（明确指定的） 或自动的（由自动放置决定）。
网格跨度
网格轨道中 网格项在每个轴上占据的轨道数量。 在第1级中， 网格项的网格跨度始终是确定的， 如果无法为某一轴确定，则默认值为 1。

网格放置属性—包括长属性 grid-row-start、grid-row-end、grid-column-start、grid-column-end，以及它们的简写属性 grid-row、grid-column 和 grid-area—允许作者通过提供以下六条信息中的任何一条或不提供来指定 网格项的放置：

行 列
开始 行起始线 列起始线
结束 行结束线 列结束线
跨度 行跨度 列跨度

在给定维度中，任意两个开始、结束和跨度的确定值意味着第三个值也是确定的。

下表总结了在何种条件下网格位置或跨度是确定的或自动的：

位置 跨度
确定的 至少指定一条线 显式、隐式或默认的跨度。
自动的 未显式指定线 不适用

8.1. 网格放置的常见模式

本节为说明性内容。

网格放置属性的长属性被组织为三个简写属性：

grid-area
grid-column grid-row
grid-column-start grid-column-end grid-row-start grid-row-end

8.1.1. 命名区域

可以通过在grid-area中指定区域名称，将项放置到命名网格区域中（例如通过grid-template-areas生成的区域）： 

article {
  grid-area: main;
  /* 将项目放置到名为 "main" 的区域中。 */
}

项也可以部分与命名网格区域对齐， 其他边缘则与其他线对齐：

.one {
  grid-row-start: main;
  /* 将行起始边对齐到 "main" 命名区域的起始边。 */
}

8.1.2. 数字索引和跨度

网格项可以按数字定位和调整大小， 这对于脚本驱动的布局特别有用：

.two {
  grid-row: 2;    /* 将项放在第二行。 */
  grid-column: 3; /* 将项放在第三列。 */
  /* 等同于 grid-area: 2 / 3; */
}

默认情况下，网格项的跨度为 1。 可以显式指定不同的跨度：

.three {
  grid-row: 2 / span 5;
  /* 从第2行开始，向下跨5行（结束于第7行）。 */
}

.four {
  grid-row: span 5 / 7;
  /* 结束于第7行，向上跨5行（从第2行开始）。 */
}

注意： 请注意，网格索引是书写模式相关的。 例如，在像阿拉伯语这样的从右到左的语言中， 第一列是最右边的列。

8.1.3. 命名行和跨度

行可以通过它们的行名称来引用， 而不是按数字计算：

.five {
  grid-column: first / middle;
  /* 从 "first" 行跨度到 "middle" 行。 */
}

注意： 请注意，如果命名网格区域与行名称同名， 放置算法将优先使用命名网格区域的行。

如果有多个同名行， 它们实际上建立了一组命名的网格行， 可以通过按名称过滤放置来唯一索引：

.six {
  grid-row: text 5 / text 7;
  /* 跨越第5到第7个名为 "text" 的行。 */
  grid-row: text 5 / span text 2;
  /* 同上——从第5个名为 "text" 的行开始，
     然后跨越另外两个 "text" 行，到第7行。 */
}

8.1.4. 自动放置

网格项可以自动放置到下一个可用的空网格单元中， 如果没有剩余空间，网格会增长。

.eight {
  grid-area: auto; /* 初始值 */
}

例如，这可以用来在目录网站上以网格模式列出若干销售项目。

如果项目应占用多个单元格， 自动放置可以与显式跨度结合使用：

.nine {
  grid-area: span 2 / span 3;
  /* 自动放置项，覆盖两行和三列。 */
}

无论自动放置算法是横向搜索并添加行， 还是纵向搜索并添加列， 都由grid-auto-flow属性控制。

注意： 默认情况下，自动放置算法线性查看网格而不会回溯； 如果为了放置较大的项目而跳过一些空位， 它不会返回填充这些空间。 要改变此行为， 请在dense中指定关键字在grid-auto-flow中。

8.2. 网格项放置与源顺序

“能力越大，责任越大。”

网格放置属性的功能允许内容在网格中自由排列和重新排序， 以至于视觉呈现可以与底层文档源顺序大致脱节。 这些功能让作者在不同设备和呈现模式中有很大的自由度， 例如使用媒体查询。 然而，它们不能代替正确的源顺序。

正确的源顺序对于语音、 顺序导航（如键盘导航）， 以及非CSS用户代理（如搜索引擎、触觉浏览器等）至关重要。 网格放置只影响视觉呈现！ 这使作者能够优化文档源以适应非CSS/非视觉交互模式， 并使用网格放置技术进一步操控视觉呈现， 从而保持源顺序不变。

8.3. 基于线的放置：grid-row-start、grid-column-start、grid-row-end 和 grid-column-end 属性

名称: grid-row-start, grid-column-start, grid-row-end, grid-column-end
值: <grid-line>
初始值: auto
应用于: 网格项 和包含块为 网格容器的绝对定位框
继承性: 否
百分比: 不适用
计算值: 指定的关键字、标识符和/或整数
规范顺序: 根据语法
动画类型: 离散型 
<grid-line> =
  auto |
  <custom-ident> |
  [ <integer> && <custom-ident>? ] |
  [ span && [ <integer> || <custom-ident> ] ]

grid-row-start、grid-column-start、grid-row-end 和 grid-column-end 属性 通过贡献一条线、一个跨度或无（自动）来确定网格项在 网格中的大小和位置， 从而为其 网格放置指定 内联起始、块起始、 内联结束 和 块结束 边缘， 以及其 网格区域。

这些值具有以下含义：

<custom-ident>
首先尝试将网格区域的边缘与命名的网格区域匹配： 如果有一个网格线的线名是<custom-ident>-start（对于grid-*-start）/ <custom-ident>-end（对于grid-*-end），则该行将作为第一个适用的网格线贡献给网格项的放置。

注意：命名的网格区域会自动生成这种形式的隐式分配的线名， 因此指定grid-row-start: foo时，将选择该命名的网格区域的起始边（除非在它之前明确指定了另一个名为foo-start的线）。

否则，将此视为与整数1一起指定的相同处理，连同<custom-ident>。

<integer> && <custom-ident>?
为网格项的放置贡献第N条网格线。 如果提供了负整数，它将从显式网格的终点边缘开始逆向计数。

如果指定了名称作为<custom-ident>，则只计算具有该名称的线。 如果没有足够的此类线，则假定所有隐式网格线都有该名称，用于找到这个位置。

如果<integer>值为零，则声明无效。

span && [ <integer> || <custom-ident> ]
为网格项的放置贡献一个网格跨度，这样网格项的网格区域对应的边与其相对方向相距N条线。 例如，grid-column-end: span 2 表示从grid-column-start线开始，向结束方向数第二条网格线。

如果指定了名称作为<custom-ident>，则只计算具有该名称的线。 如果没有足够的此类线，则假定所有隐式网格线在显式网格的相应方向上具有该名称，用于计算这个跨度。

例如，给定以下声明： 
.grid { grid-template-columns: 100px; }
.griditem { grid-column: span foo / 4; }

网格容器具有一个显式网格， 其中有两个网格线，编号为1和2。 该网格项的列结束边指定在第4条线， 因此在隐式网格的结束方向生成了两条线。

它的列起始边必须是之前能找到的第一个“foo”线。 然而在网格中没有“foo”线， 因此唯一的可能性是隐式网格中生成的线。 第3条线不是候选者，因为它位于显式网格的结束方向， 而列起始跨度强制它向起始方向搜索。 因此，唯一的选择是让隐式网格在显式网格的起始方向上生成一条线。

如果省略了<integer>，则默认值为1。 负整数或零是无效的。

auto
此属性对网格项的放置不作任何贡献， 表示自动放置或默认跨度为1。 （参见§ 8 放置网格项。）

在上述所有生产规则中，<custom-ident> 还排除了关键字 span 和 auto。

给定一个单行8列的网格和以下9条命名的线： 
1  2  3  4  5  6  7  8  9
+--+--+--+--+--+--+--+--+
|  |  |  |  |  |  |  |  |
A  B  C  A  B  C  A  B  C
|  |  |  |  |  |  |  |  |
+--+--+--+--+--+--+--+--+

以下声明将网格项放置在索引指示的行之间：

grid-column-start: 4; grid-column-end: auto;
/* 第4行到第5行 */

grid-column-start: auto; grid-column-end: 6;
/* 第5行到第6行 */

grid-column-start: C; grid-column-end: C -1;
/* 第3行到第9行 */

grid-column-start: C; grid-column-end: span C;
/* 第3行到第6行 */

grid-column-start: span C; grid-column-end: C -1;
/* 第6行到第9行 */

grid-column-start: span C; grid-column-end: span C;
/* 错误：忽略了结束跨度，自动放置的项目无法跨越到命名线。
   等价于 grid-column: span 1;。 */

grid-column-start: 5; grid-column-end: C -1;
/* 第5行到第9行 */

grid-column-start: 5; grid-column-end: span C;
/* 第5行到第6行 */

grid-column-start: 8; grid-column-end: 8;
/* 错误：第8行到第9行 */

grid-column-start: B 2; grid-column-end: span 1;
/* 第5行到第6行 */

8.3.1. 网格放置冲突处理

如果一个放置包含两条线，而 起始线位于 结束线的更末端， 则交换这两条线。如果起始线与结束线相等， 则删除结束线。

如果放置包含两个跨度， 则删除由结束贡献的 网格放置属性。

如果放置仅包含命名线的跨度， 则将其替换为跨度1。

8.4. 放置简写：grid-column，grid-row， 以及grid-area属性

名称: grid-row，grid-column
值: <grid-line> [ / <grid-line> ]?
初始值: auto
适用于: 网格项和包含块是 网格容器的绝对定位框
继承: 否
百分比: N/A
计算值: 见单个属性
动画类型: 离散
规范顺序: 按语法

grid-row 和grid-column属性是 grid-row-start/grid-row-end 和 grid-column-start/grid-column-end的简写。

如果指定了两个<grid-line> 值，grid-row-start/grid-column-start长写形式将设置为斜杠前的值， 而grid-row-end/grid-column-end长写形式将设置为斜杠后的值。

当省略第二个值时，如果第一个值是 <custom-ident>， grid-row-end/grid-column-end长写形式也将设置为该 <custom-ident>； 否则，它将设置为auto。

名称: grid-area
值: <grid-line> [ / <grid-line> ]{0,3}
初始值: auto
适用于: 网格项和包含块是 网格容器的绝对定位框
继承: 否
百分比: N/A
计算值: 见单个属性
动画类型: 离散
规范顺序: 按语法

grid-area属性是 简写，用于 grid-row-start，grid-column-start， grid-row-end和grid-column-end。

如果指定了四个<grid-line> 值，grid-row-start设置为第一个值，grid-column-start设置为第二个值，grid-row-end设置为第三个值， grid-column-end设置为第四个值。

当省略grid-column-end时， 如果grid-column-start是 <custom-ident>，则grid-column-end设置为该 <custom-ident>； 否则，设置为auto。

当省略grid-row-end时， 如果grid-row-start是 <custom-ident>，grid-row-end设置为该 <custom-ident>； 否则，设置为auto。

当省略grid-column-start时， 如果grid-row-start是 <custom-ident>， 则四个长写形式全部设置为该值。 否则，设置为auto。

注意: 此简写的解析顺序是 row-start/column-start/row-end/column-end， 对于LTR页面，它按逆时针 CCW顺序排列，与使用物理方向的相关四边属性（如margin）方向相反。

8.5. 网格项放置算法

以下网格项放置算法 解析自动位置的网格项到确定位置， 确保每个网格项有一个定义好的网格区域用于布局。 (网格跨度不需要特殊解析; 如果未明确指定， 它们默认为1。)

注意: 此算法可能会在隐式网格中创建新行或列， 如果在显式网格中没有空间放置自动定位的网格项。

每个网格单元 (在显式和 隐式网格中) 都可以是已占用或 未占用。 如果网格单元被网格区域覆盖， 并且该区域属于具有确定网格位置的 网格项， 则该单元被占用； 否则， 该单元处于未占用状态。 在此算法过程中，单元的已占用/未占用状态可能会发生变化。

为了清晰起见， 该算法假定grid-auto-flow 指定为row。 如果改为column， 则需在此算法中互换所有对行和列、行内和块的提及。

注意: 自动放置算法使用网格项的 修改后的文档顺序， 而非其原始文档顺序。

  1. 生成匿名网格项，如§ 6 网格项中所述。 (匿名网格项总是自动放置， 因为它们的框无法指定任何网格放置属性。)

  2. 定位任何非自动定位的项目。

  3. 处理锁定到给定行的项目。

    对于具有确定行位置的每个 网格项 (即，grid-row-start和 grid-row-end属性定义了 确定网格位置的项目)， 按修改后的文档顺序处理：

    “稀疏”打包（默认行为）

    将其放置的列起始线设置为确保该项目的 网格区域不会与任何已占用网格单元重叠的最早(正序中最小索引) 线索，并且在该步骤中位于之前放置的其他网格项之后。

    “密集”打包 (指定dense)

    将其放置的列起始线设置为确保该项目的 网格区域不会与任何已占用网格单元重叠的最早(正序中最小索引)线索。

  4. 确定隐式网格中的列。

    在隐式网格中创建列：

    1. 从显式网格中的列开始。

    2. 在所有具有确定列位置的项目(明确定位的项目、上一步中定位的项目以及尚未定位但具有确定列的项目)之间， 根据需要在隐式网格 的起始和结束位置添加列，以容纳这些项目。

    3. 如果所有不具有确定列位置的项目中的最大 列跨度大于 隐式网格的宽度， 则在隐式网格的末端添加列以容纳该列跨度。

    例如，在以下样式片段中： 
    #grid {
  display: grid;
  grid-template-columns: repeat(5, 100px);
  grid-auto-flow: row;
}
#grid-item {
  grid-column: 4 / span 3;
}

    所需列数为6。 显式网格 提供5列(由grid-template-columns 定义)，从第1到第6行， 但#grid-item的列位置意味着它结束于第7行， 这需要在隐式网格的末尾添加一列。

  5. 定位剩余的网格项。

    自动放置光标定义了网格中的当前“插入点”， 由行和列的一对网格线指定。 最初，自动放置光标 被设置为隐式网格中的起始行和列线。

    使用中的grid-auto-flow值决定了如何定位项目：

    “稀疏”打包（默认行为）

    对于每个尚未通过之前步骤定位的网格项， 按照修改后的文档顺序：

    如果项目具有确定的列位置：

    1. 将光标的列位置设置为 网格项的列起始线。 如果此值小于光标的前一个列位置， 则将行位置加1。

    2. 增加光标的行位置， 直到找到一个网格项不会重叠任何已占用网格单元的值 (根据需要在隐式网格中创建新行)。

    3. 将项目的行起始线设置为光标的行位置， 并根据该位置的跨度设置项目的行结束线。

    如果项目在两个轴上都有自动网格位置：

    1. 增加自动放置光标的列位置， 直到该项目的网格区域不与任何已占用网格单元重叠， 或者光标的列位置 加上项目的列跨度超出了隐式网格中的列数， 如之前的算法步骤确定的那样。

    2. 如果在上一步中找到一个不重叠的位置， 将项目的行起始线和列起始线设置为光标的位置。 否则， 增加自动放置光标的行位置， (根据需要在隐式网格中创建新行)， 将其列位置设置为隐式网格中最左侧的列线， 然后返回到上一步。

    “密集”打包 (指定dense)

    对于每个尚未通过之前步骤定位的网格项， 按照修改后的文档顺序：

    如果项目具有确定的列位置：

    1. 将光标的行位置设置为隐式网格中的最上方行线。 将光标的列位置设置为网格项的列起始线。

    2. 增加自动放置光标的行位置， 直到找到一个网格项不会重叠任何已占用网格单元的值 (根据需要在隐式网格中创建新行)。

    3. 将项目的行起始线索引设置为光标的行位置。 (隐式地也设置项目的行结束线，按其跨度。)

    如果项目在两个轴上都有自动网格位置：

    1. 将光标的行和列位置设置为隐式网格中最上方的行线和最左侧的列线。

    2. 增加自动放置光标的列位置， 直到该项目的网格区域不与任何已占用网格单元重叠， 或光标的列位置加上项目的列跨度 超出了隐式网格中的列数。

    3. 如果在上一步中找到一个不重叠的位置， 将项目的行起始线和列起始线设置为光标的位置。 否则， 增加自动放置光标的行位置 (根据需要在隐式网格中创建新行)， 重置其列位置为隐式网格中最左侧的列线， 然后返回到上一步。

9. 绝对定位

9.1. 使用网格容器作为包含块

如果绝对定位元素的包含块由网格容器生成， 则包含块对应于其网格定位属性 确定的网格区域。 然后，偏移属性 (top / right / bottom / left) 然后指示从该包含块的相应边缘向内的偏移，和正常情况一样。

注意： 虽然将元素绝对定位到网格容器允许它对齐到该容器的网格线， 但这些元素不会占据空间，也不会参与网格的布局。

.grid {
  grid: 1fr 1fr 1fr 1fr / 10rem 10rem 10rem 10rem;
  /* 4 等高行填充网格容器，
     每列为 10rem */
  justify-content: center;
  /* 将网格水平居中放置在 网格容器 内 */
  position: relative;
  /* 确立绝对定位 包含块 */
}

.abspos {
  grid-row-start: 1;     /* 网格容器的第一行网格线 = 网格容器的顶部 */
  grid-row-end: span 2;  /* 第三行网格线 */
  grid-column-start: 3;  /* 第三列网格线 */
  grid-column-end: auto; /* 右侧内边距 */
  /* 包含块 覆盖 网格容器 的右上象限 */

  position: absolute;
  top: 70px;
  bottom: 40px;
  left: 100px;
  right: 30px;
}

注意： 网格和网格定位属性 是流相对的， 而偏移属性 (left、right、top 和 bottom) 是物理上的， 因此如果direction或writing-mode属性发生变化， 网格将进行匹配变换， 但偏移不会。

与自动放置不同，auto值的网格定位属性为定位贡献一个特殊的线，其位置对应于网格容器的相应内边距边缘（如果网格容器 溢出，则为可滚动区域的内边距边缘）。 这些线成为用于定位绝对定位项的增强网格的第0条和第-0条线。

注意： 因此，默认情况下，绝对定位框的包含块 将对应于网格容器的内边距边缘， 正如它对块容器那样。

绝对定位在网格及其流内内容布局之后发生， 并且不会对任何网格轨道的大小产生贡献，也不会影响网格的大小/配置。 如果网格定位属性通过显式指定或跨越现有的隐式网格之外，引用了不存在的线， 则它将被视为指定了auto（而不是创建新的隐式网格线）。

注意： 记住隐式线假设拥有所有线名称，因此引用的线可能存在，尽管它没有显式命名。

如果定位只包含网格跨度， 请将其替换为该轴上的两个auto线。 （这种情况发生在轴上的两个网格定位属性最初都贡献了一个跨度， 并且§ 8.3.1 网格定位冲突处理导致忽略了第二个跨度。）

9.2. 使用网格容器作为父元素

绝对定位的网格容器子元素是脱离文档流的，不是网格项， 因此不会影响其他项的放置或网格的大小。

绝对定位的网格容器子元素的 静态位置 [CSS2]是按照该元素为唯一网格项计算的， 其网格区域的边缘与 网格容器的内容边缘重合。 然而，如果网格容器父元素也是绝对定位元素的包含块的生成器， 则使用§ 9.1 使用网格容器作为包含块中确定的网格区域。

注意： 请注意，此位置受子元素的justify-self和align-self属性的影响， 并且，如同其他大多数布局模型一样，绝对定位的子元素不会影响包含块的大小或其内容的布局。

10. 对齐与间距

在网格容器的网格轨道尺寸确定后， 并且所有网格项的尺寸也最终确定，网格项可以在其网格区域内进行对齐。

margin属性可以用于类似块布局中的方式对齐元素。网格项还会遵循盒子对齐属性， 这些属性来自CSS Box Alignment Module，通过关键词可以轻松实现行和列中的对齐。

默认情况下，网格项会扩展填充其网格区域。 然而，如果justify-self或align-self的计算值不是stretch，或者margins被设置为auto，那么网格项将会自动调整以适应其内容。

10.1. 间隙：行间隙、列间隙和间隙属性

行间隙和列间隙属性（以及它们的gap简写属性），当在网格容器上指定时，定义了网格间隙。

这些属性的效果类似于受影响的网格线具有一定的厚度：网格轨道之间的空间相当于表示它们的网格间隙。在轨道尺寸计算中，每个网格间隙被视为一个额外的空白固定大小轨道，其大小由指定的值决定，跨越相应的网格线的所有网格项也跨越这些轨道。

注意： 由于justify-content/align-content，可能会在轨道之间增加额外的间距。参见§ 11.1 网格尺寸算法。这些空间会有效地增加网格间隙的大小。

如果一个网格在轨道之间被分割， 那么那些轨道之间的网格间隙必须被抑制。注意，即使在强制换页后，网格间隙也会被抑制，不同于外边距。

网格间隙只会出现在隐式网格的轨道之间；在第一个轨道之前或最后一个轨道之后没有网格间隙。（特别是，在隐式网格的第一个/最后一个轨道与扩展网格中的“自动”线之间没有网格间隙。）

当一个折叠轨道的网格间隙折叠时，它们会完全重合——两者的开始和结束边重叠。如果某个折叠轨道的一侧没有网格间隙（例如，它是隐式网格的第一个或最后一个轨道），那么折叠它的网格间隙结果是该折叠轨道的两侧都没有网格间隙。

10.2. 使用auto的边距对齐

本节是非规范性的。 关于边距如何影响网格项的规范定义，请参见§ 11 网格尺寸。

在网格项上使用的自动边距的效果与块流中的自动边距非常相似：

  • 在计算网格轨道尺寸时，自动边距被视为0。
  • auto边距会在通过盒子对齐属性进行对齐之前吸收正的空余空间。
  • 溢出的元素忽略它们的auto边距，按照它们的盒子对齐属性规定的方式溢出。

10.3. 内联轴对齐：justify-self和justify-items属性

网格项可以使用justify-self属性或在justify-items属性对齐在内联轴上进行对齐，如在[CSS-ALIGN-3]中定义。

例如， 对于英文文档， 内联轴是水平的， 因此justify-*属性在水平方向上对齐网格项。

如果基线对齐在一个网格项上指定，而该项在该轴的尺寸依赖于本质上尺寸的轨道， （其尺寸依赖于该项的尺寸和基线对齐，形成了循环依赖）， 那么该项不参与基线对齐，而是使用其回退对齐，如同该回退对齐最初被指定的一样。 对于此目的，当网格容器在相关轴上具有不确定的尺寸时，<flex>轨道尺寸被视为“本质尺寸”。

注意： 是否使用回退对齐不会在布局过程中发生变化：如果存在循环，它就一直存在。

10.4. 块轴对齐：align-self和align-items属性

网格项也可以使用align-self或在align-items属性在块轴上进行对齐，如在[CSS-ALIGN-3]中定义。

如果基线对齐在一个网格项上指定，而该项在该轴的尺寸依赖于本质上尺寸的轨道， （其尺寸依赖于该项的尺寸和基线对齐，形成了循环依赖）， 那么该项不参与基线对齐，而是使用其回退对齐，如同该回退对齐最初被指定的一样。 对于此目的，当网格容器在相关轴上具有不确定的尺寸时，<flex>轨道尺寸被视为“本质尺寸”。

10.5. 对齐网格：justify-content和align-content属性

如果网格的外边缘与 网格容器的内容边缘不一致 （例如，如果没有列是弹性尺寸）， 则根据justify-content和align-content属性在内容框中对齐网格轨道。

例如，以下网格垂直居中并与其网格容器的右边缘对齐： 
.grid {
  display: grid;
  grid: 12rem 12rem 12rem 12rem / 10rem 10rem 10rem 10rem;
  justify-content: end;
  align-content: center;
}

如果没有网格轨道（显式网格为空，且 在隐式网格中没有创建轨道）， 则每个轴的唯一网格线 与网格容器的起始边对齐。

请注意，某些justify-content和align-content的值可能导致轨道分开排列 （space-around、space-between、space-evenly） 或调整大小（stretch）。 如果网格在轨道之间被分段， 则必须抑制轨道之间的任何此类额外间距。

例如，在以下网格中， 跨越项的网格区域由于对齐而增加，以容纳分配给缝隙的额外空间： 
.wrapper {
  display: grid;
  /* 3行/4列的网格容器 */
  grid: repeat(3, auto) / repeat(4, auto);
  gap: 10px;
  align-content: space-around;
  justify-content: space-between;
}

.item1 { grid-column: 1 / 5; }
.item2 { grid-column: 1 / 3; grid-row: 2 / 4; }
.item3 { grid-column: 3 / 5; }
/* 最后两个项自动放置到最后两个网格单元中 */

请注意，对齐（不同于缝隙间距） 是在网格轨道尺寸确定之后发生的， 因此，如果轨道尺寸是由跨越项的内容决定的， 它将在对齐阶段获得多余的空间以容纳对齐间距。

10.6. 网格容器基线

网格容器的第一个（最后一个）基线按以下方式确定：

  1. 找到包含至少一个网格项的网格容器的第一个（最后一个）行。

    如果与该行相交的任何网格项 参与该行中的基线对齐， 则网格容器的基线集是根据这些网格项的共享 对齐基线生成的。

    否则， 网格容器的第一个（最后一个）基线集是根据第一个（最后一个）网格项的 对齐基线生成的， 按照网格容器的网格顺序（根据 书写模式）。 如果该网格项在网格的内联轴中没有对齐基线， 则首先根据其边框边缘合成一个基线。

  2. 如果网格容器不包含任何 网格项， 则网格容器没有第一个（最后一个）基线集， 并根据其对齐上下文的规则， 在需要时合成一个。 退出此算法。

网格修改后的文档顺序（网格顺序）是当遍历网格的 网格单元时遇到网格项的顺序。 如果同时遇到两个项， 则它们按修改后的文档顺序排列。

根据上述规则计算基线时， 如果贡献基线的框具有允许滚动的overflow值， 则在确定其基线时，必须将该框视为处于其初始滚动位置。

当确定表格单元的基线时， 网格容器提供的基线与行框或表行提供的基线相同。[CSS2]

有关基线的更多信息，请参见CSS书写模式3 §4.1 基线简介和 CSS框对齐模块3 §9 基线对齐详情。

11. 网格尺寸计算

本节定义了网格尺寸算法， 该算法确定所有网格轨道的尺寸， 从而确定整个网格的尺寸。

每个轨道都有指定的最小和最大 尺寸函数（两者可能相同）。 每个尺寸函数可以是以下之一：

  • 固定尺寸函数 (<length> 或可解析的<percentage>）。
  • 内在尺寸函数 (min-content，max-content，auto，fit-content()）。
  • 灵活尺寸函数 (<flex>）。

网格尺寸算法定义了如何将这些尺寸约束解析为实际使用的轨道尺寸。

11.1. 网格尺寸算法

  1. 首先，使用 轨道尺寸算法 来解析 网格列 的尺寸。

    如果在此步骤中计算 网格项目 的布局取决于 可用空间 在 块轴 中的分配， 假设其具有在行具有 确定的 最大轨道尺寸函数 时的大小，并且所有其他行都是无限的。 如果 网格容器 和所有轨道都具有 确定的 尺寸， 则应用 align-content 来找到由此类项目跨越的任何间隙的最终有效尺寸； 否则忽略轨道对齐效果的估算。

    是否有助于使用 启发式方法 以更准确地估计初始值？ 例如，假设它的 可用空间 为以下最大值之一：
    • 它跨越的所有 确定的 轨道尺寸之和 （如果确定了最小和最大尺寸函数，则使用轨道的最小和最大尺寸函数中的最大值， 如果两者都已确定， fit-content() 的参数如果已确定）。
    • 项目的 min-content 尺寸， 如果它跨越的任何轨道 具有 min-content 或 fit-content() 尺寸函数。
    • 项目的 自动最小尺寸， 如果它跨越的任何轨道 具有 auto 最小尺寸函数。
    • 无限大，如果它跨越的任何轨道 具有 max-content 最小尺寸函数 或 max-content、auto 或 <flex> 最大尺寸函数。

    这可能会减少必要的重布局次数， 但在某些情况下它会产生不同或更好的结果吗？ 我们应该将其纳入规范吗？

  2. 接下来，使用 轨道尺寸算法 解析 网格行 的尺寸。

    为了找到任何项目的 行轴 可用空间， 使用上一步计算的 网格列 尺寸。 如果 网格容器 的 行轴尺寸 是 确定的， 还应应用 justify-content 来计算列间隙的有效尺寸。

  3. 然后，如果任何网格项目的 最小内容贡献 基于第2步计算的行尺寸和对齐方式发生了变化， 使用新的 最小内容 和 最大内容贡献 重新解析 网格列 的尺寸（仅一次）。

    为了找到任何项目的 块轴 可用空间， 使用上一步计算的 网格行 尺寸。 如果 网格容器 的 块尺寸 是 确定的， 还应应用 align-content 来计算行间隙的有效尺寸。

    这种重复在某些情况下是必要的，例如当 行轴尺寸 取决于 块尺寸 时的网格项目， 例如包裹列 弹性容器 (flex-flow: column wrap)， 正交流 (writing-mode)， 以及 多列容器。
  4. 接下来，如果任何网格项目的 最小内容贡献 基于第3步计算的列尺寸和对齐方式发生了变化， 使用新的 最小内容 和 最大内容贡献 重新解析 网格行 的尺寸（仅一次）。

    为了找到任何项目的 行轴 可用空间， 使用上一步计算的 网格列 尺寸。 如果 网格容器 的 行轴尺寸 是 确定的， 还应应用 justify-content 来计算列间隙的有效尺寸。

  5. 最后，使用得到的 网格 的尺寸来确定 网格容器 的大小， 并根据 网格容器 的 align-content 和 justify-content 属性对轨道进行对齐。

    注：这可能会在轨道之间引入额外的空间， 从而将跨越这些间隙的任何网格项目的网格区域扩大， 超出轨道尺寸期间分配的空间。

一旦每个 网格区域 的大小确定， 网格项目 将被放置到它们各自的包含块中。 网格区域 的宽度和高度被认为是 确定的。

注：由于仅使用确定尺寸计算的公式， 例如 stretch fit 公式， 也是确定的， 因此拉伸的网格项目的尺寸也被认为是确定的。

11.2. 轨道尺寸术语

最小轨道尺寸函数
如果轨道是使用 minmax() 函数进行尺寸调整的， 则这是该函数的第一个参数。 如果轨道是使用 <flex> 值或 fit-content() 函数调整尺寸，auto。 否则，轨道的尺寸函数。
最大轨道尺寸函数
如果轨道是使用 minmax() 函数进行尺寸调整的， 则这是该函数的第二个参数。 否则，轨道的尺寸函数。 在所有情况下，将 auto 和 fit-content() 视为 max-content， 除非另有规定 fit-content()。
可用网格空间
在每个维度中独立计算，可用网格空间 为：

  • 如果 网格容器 的尺寸是确定的， 则使用其内容框的大小。

  • 如果 网格容器 的尺寸根据 最小内容约束 或 最大内容约束 进行尺寸调整， 则 可用网格空间 即为该约束（且为不确定）。

注： 表示基于内容的尺寸的 auto 尺寸（例如水平书写模式中块级框的高度）相当于 max-content。

在所有情况下，根据 可用网格空间 约束 网格容器 的最小/最大宽度/高度属性， 如果它们是确定的。

空余空间
等于 可用网格空间 减去 所有网格轨道（包括槽）的 基本尺寸 之和， 最小为零。 如果 可用网格空间 是 不确定的， 则 空余空间 也是 不确定的。
跨度计数
适用维度中由 网格项目 跨越的 网格轨道 的数量。

注： 请记住，槽 被视为固定大小的轨道—— 轨道的最小和最大尺寸函数都设置为槽的使用尺寸——用于网格尺寸算法。 它们的宽度需要相应地纳入 轨道尺寸 算法 的计算中。

11.3. 轨道尺寸算法

本节余下部分是 轨道尺寸算法， 它通过 最小 和 最大轨道尺寸函数 来计算使用的轨道尺寸。 每个轨道有一个 基本尺寸， 一个 <length> 在算法执行过程中逐渐增长， 并最终成为轨道的最终尺寸， 还有一个 增长限制， 一个 <length> 提供了 基本尺寸 的最大期望尺寸。 算法有5个步骤：

  1. 初始化轨道尺寸
  2. 解析内在轨道尺寸
  3. 最大化轨道
  4. 扩展弹性轨道
  5. [[#algo-stretch|扩展拉伸的 auto 轨道]]

11.4. 初始化轨道尺寸

初始化每个轨道的基本尺寸和增长限制。 对于每个轨道， 如果该轨道的 最小轨道尺寸函数 是：

固定尺寸函数
解析为一个绝对长度，并将该尺寸用作轨道的初始 基本尺寸。

注： 不确定 长度不会出现， 因为它们被视为 auto。

内在尺寸函数
使用初始 基本尺寸 为零。

对于每个轨道， 如果该轨道的 最大轨道尺寸函数 是：

11.5. 解析内在轨道尺寸

此步骤将内在轨道 尺寸函数 解析为绝对长度。 首先根据完全包含在单个轨道内的项目解析这些尺寸。 然后逐步加入跨越多个轨道的项目的空间需求， 尽可能将额外空间均匀分配到这些轨道中。

注： 当此步骤完成时， 所有内在的 基本尺寸 和 增长限制 都将解析为绝对长度。

注： 请记住 fit-content() 和 auto 最大轨道尺寸函数 被视为与 max-content 相同，除非另有明确说明。

  1. 填充基线对齐的项目， 以便它们的内在尺寸贡献反映其基线对齐。 对于每个 基线共享组 中的项目， 在每个项目的开始/结束侧（对于首/尾基线对齐） 添加一个“填充”（实际上是额外的边距）， 使得当它们一起对齐时， 它们的 基线按规定对齐。

    在轨道尺寸调整的过程中， 将这些“填充”视为项目的内在尺寸贡献的一部分。 如果一个项目使用了多个内在尺寸贡献， 则每个贡献可能会有不同的填充。

    例如，当 网格容器 具有 不确定 尺寸时， 首先在最小/最大内容约束下布局以确定尺寸， 然后以该尺寸“真正”布局 （这可能会影响百分比轨道等内容）。 每个阶段添加的“填充”是独立的， 仅在该阶段布局期间影响布局。

    注： 请注意，此步骤中考虑了 基线自对齐 和 基线内容对齐 的项目。

    注： 由于 网格项目 自身的尺寸 取决于一个内在尺寸的轨道 不参与基线对齐， 它们不会进行填充。

  2. 调整轨道以适应非跨轨项目： 对于每个具有内在 轨道尺寸函数 而不是 弹性尺寸函数 的轨道， 考虑其中跨度为1的项目：
    对于最小内容最小值：
    如果轨道具有 最小内容 最小轨道尺寸函数， 则将其 基本尺寸 设置为项目的 最小内容贡献 的最大值， 最小为零。
    对于最大内容最小值：
    如果轨道具有 最大内容 最小轨道尺寸函数， 则将其 基本尺寸 设置为项目的 最大内容贡献 的最大值， 最小为零。
    对于自动最小值：
    如果轨道具有 auto 最小轨道尺寸函数， 且 网格容器 正在根据 最小/最大内容约束 进行尺寸调整， 则将该轨道的 基本尺寸 设置为 其项目的 受限的最小/最大内容贡献（分别）， 最小为零。 项目的 受限的最小/最大内容贡献 是其 最小/最大内容贡献， 受 最大轨道尺寸函数 的限制（该函数可能是 fit-content() 轨道尺寸函数的参数）， 如果这是 固定 的，并最终受其 最小贡献 （如下所定义） 的限制。

    否则， 将该轨道的 基本尺寸 设置为项目的 最小贡献 的最大值， 最小为零。 项目的 最小贡献 是它能拥有的最小 外部尺寸。 具体来说， 如果项目的计算 首选尺寸 表现为auto 或依赖于其 包含块 在相关轴上的尺寸， 则它的 最小贡献 是假设项目的 最小使用尺寸 作为其 首选尺寸 时得到的 外部尺寸； 否则项目的 最小贡献 是它的 最小内容贡献。 由于 最小贡献 通常取决于项目内容的大小， 它被视为 内在尺寸贡献 的一种类型。

    注： 对于具有指定最小尺寸为 auto（初始值）的项目， 其 最小贡献 通常等同于 最小内容贡献—但在某些情况下可能有所不同，详见 § 6.6 网格项目的自动最小尺寸。 此外，最小贡献 ≤ 最小内容贡献 ≤ 最大内容贡献。

    对于最小内容最大值：
    如果轨道具有 最小内容 最大轨道尺寸函数， 则将其 增长限制 设置为项目的 最小内容贡献 的最大值。
    对于最大内容最大值：
    如果轨道具有 最大内容 最大轨道尺寸函数， 则将其 增长限制 设置为项目的 最大内容贡献 的最大值。 对于 fit-content() 最大值， 还要通过 fit-content() 参数 限制此 增长限制。

    在所有情况下，如果轨道的 增长限制 现在小于其 基本尺寸， 则增加 增长限制 以匹配 基本尺寸。

    注： 此步骤是下面处理跨轨项目步骤的简化版， 应该会产生与对跨度为1的项目运行这些指令相同的行为。

  3. 增加尺寸以容纳跨越内容尺寸轨道的项目： 接下来，考虑跨度为2且不跨越具有 弹性尺寸函数 的轨道的项目。
    1. 对于内在最小值： 首先增加具有 内在 最小轨道尺寸函数 的轨道的 基本尺寸，通过 分配额外空间（如有需要） 来容纳这些项目的 最小贡献。

      如果网格容器根据 最小 或 最大内容约束 进行尺寸调整， 则在此处使用项目的 受限最小内容贡献 代替其 最小贡献。 （对于跨越多个轨道的项目， 用于计算其 受限最小/最大内容贡献 的上限是它跨越的 固定 最大轨道尺寸函数 之和，并且仅在其跨越这些轨道时应用。）

    2. 对于基于内容的最小值： 接下来继续增加具有 最小轨道尺寸函数 的轨道的 最小内容 或 最大内容 的 分配额外空间，以应对这些项目的 最小内容贡献。
    3. 对于最大内容的最小值： 接下来，如果网格容器根据 最大内容约束 进行尺寸调整， 继续增加具有 最小轨道尺寸函数 的轨道的 auto 或 最大内容 的 分配额外空间，以应对这些项目的 受限最大内容贡献。

      在所有情况下， 继续增加具有 最小轨道尺寸函数 的轨道的 最大内容 的 分配额外空间，以应对这些项目的 最大内容贡献。

    4. 如果此时任何轨道的 增长限制 现在小于其 基本尺寸， 则将其 增长限制 增加到与其 基本尺寸 相匹配。
    5. 对于内在最大值： 接下来，增加具有 最大轨道尺寸函数 的轨道的 分配额外空间 以应对这些项目的 最小内容贡献。 在此步骤中，将任何由无限变为有限的轨道标记为 无限可增长，为下一步做准备。

    对于跨度更大的项目，逐步重复此过程，直到所有项目都被考虑。

  4. 增加尺寸以容纳跨越 弹性轨道 的项目： 接下来，重复上一步 但考虑所有跨越具有 弹性尺寸函数 的轨道的项目， 并且
    • 仅将空间分配给 弹性轨道（即将所有其他轨道视为具有 固定尺寸函数）
    • 如果项目跨越的所有 弹性轨道 的 弹性尺寸函数 之和大于零， 则根据其 弹性尺寸函数 的比率分配空间， 而不是平均分配空间。
  5. 如果任何轨道仍具有 无限的 增长限制 （例如，因为它没有项目放置其中， 或者它是一个 弹性轨道）， 将其 增长限制 设置为其 基本尺寸。

注： 当项目跨越多个轨道时，没有单一的方式来满足内在尺寸约束。 该算法包含了一些已被证明在现实世界案例中产生良好结果的启发式方法， 例如本规范前面提到的“game”示例。 将来可能会根据识别出的更高级的启发式方法更新此算法。

11.5.1. 在跨越轨道上分配额外空间

要通过增加一组轨道的受影响尺寸来 分配额外空间，如一组内在尺寸贡献所需：

  1. 分别为每个受影响的 基本尺寸 或 增长限制 维护一个 计划增加值，初始值为0。 （这可以防止尺寸增加变得依赖顺序。）
  2. 对于每个被考虑的项目，
    1. 找到要分配的空间： 从每个跨越轨道的项目尺寸贡献中减去对应的尺寸（基本尺寸 或 增长限制）。 对于无限的 增长限制，使用该轨道的 基本尺寸 替代。将结果取零。 
      extra-space = max(0, size-contribution - ∑track-sizes)
    2. 分配空间至限制： 对于每个跨越的轨道，按以下方法分配每个轨道的 项目增加值： 将空间在这些轨道之间平均分配， 当轨道的受影响尺寸加上 项目增加值 达到其 限制 时冻结该轨道， 并继续增长未冻结的轨道。

      对于 基本尺寸， 限制 是其 增长限制。 对于 增长限制， 如果它被标记为 无限可增长， 则 限制 为无限；否则等于 增长限制。

      注： 如果受影响的尺寸是 增长限制， 且轨道未被标记为 无限可增长， 那么每个 项目增加值 将为零。

    3. 分配超出限制的空间： 如果在所有轨道冻结后仍有剩余空间， 则解冻并继续分配空间给…的 项目增加值：
      • 当 容纳最小贡献 或 容纳最小内容贡献 时： 任何受影响的同时具有内在 最大轨道尺寸函数 的轨道； 如果没有这样的轨道，则为所有受影响的轨道。
      • 当 容纳最大内容贡献 时： 任何受影响的同时具有 最大内容 的 最大轨道尺寸函数 的轨道； 如果没有这样的轨道，则为所有受影响的轨道。
      • 当处理任何内在 增长限制 时： 所有受影响的轨道。

      在此情况下， 最大轨道尺寸函数 是 fit-content() 轨道， 直到达到 fit-content() 参数指定的限制， 之后它被视为具有该参数的 固定尺寸函数。

      注： 此步骤优先分配空间给超出当前增长限制的 最小轨道尺寸函数 所需的空间， 基于其 最大轨道尺寸函数 的类型。

    4. 对于每个受影响的轨道， 如果轨道的 项目增加值 大于轨道的 计划增加值， 则将轨道的 计划增加值 设置为该值。
  3. 更新轨道的受影响尺寸，通过增加 计划增加值 以便下一轮空间分配考虑该增加。 （如果受影响的尺寸是无限的 增长限制， 则将其设置为轨道的 基本尺寸 加上 计划增加值。）

11.6. 最大化轨道

如果 可用空间 是正值， 则将其平均分配给所有轨道的 基本尺寸， 并在轨道达到其 增长限制 时冻结轨道（并在需要时继续增长未冻结的轨道）。

为本步骤的目的： 如果网格容器在 最大内容约束 下进行尺寸调整， 则 可用空间 是无限的； 如果在 最小内容约束 下进行尺寸调整， 则 可用空间 为零。

如果这将导致网格的尺寸大于 网格容器 的 内部尺寸，并受其 最大宽度/高度 限制， 则重新执行此步骤， 将 可用网格空间 视为等于 当网格容器的尺寸达到其 最大宽度/高度 时 网格容器的 内部尺寸。

11.7. 扩展弹性轨道

此步骤通过使用可以分配给 弹性轨道 的最大值来调整其尺寸，且不会超过 可用空间。

首先，找到网格的使用 弹性份额：

如果 自由空间 为零 或者在 网格容器 在 最小内容约束 下调整大小：
使用的 弹性分数 为 零。
否则，如果 自由空间 是 一个 确定长度：
使用的 弹性分数 是 使用所有的 网格轨道 和 需要填充的空间 的 找到一个 fr 的大小 算法的结果。
否则，如果 自由空间 是 一个 不确定长度：
使用的 弹性分数 是以下各项的最大值：
  • 对于每个可伸缩轨道， 如果该可伸缩轨道的 弹性因子 大于一， 则结果为该轨道的 基础大小 除以其 弹性因子； 否则， 结果为该轨道的 基础大小。
  • 对于每个跨越可伸缩轨道的 网格项， 结果为使用所有该项跨越的网格轨道和该项的 找到一个 fr 的大小 算法的结果， 以及该项的 需要填充的空间 中 的 最大内容贡献。

如果使用此 弹性分数 会导致 网格 小于 网格容器 的 最小宽度/高度（或大于 网格容器的 最大宽度/高度）， 则重新执行此步骤， 将 自由空间 视为 确定的，并将 可用网格空间 设为 网格容器的 内部大小 当它根据其 最小宽度/高度（最大宽度/高度）调整大小时。

对于每个 可伸缩轨道， 如果使用的 弹性分数 与轨道的 弹性因子 的乘积大于轨道的 基础大小， 则将其 基础大小 设置为该乘积。

11.7.1. 查找 fr 的大小

该算法查找一个 fr 单位在不超过目标大小的情况下可以达到的最大尺寸。 它必须与一组 网格轨道 和一定数量的 需要填充的空间 一起调用。

  1. 设定 剩余空间 为 需要填充的空间 减去 非可伸缩 网格轨道 的 基础大小。
  2. 设定 弹性因子总和 为 弹性因子 的总和，来自 可伸缩轨道。 如果该值小于1， 则将其设为1。
  3. 设定 假设的 fr 大小 为 剩余空间 除以 弹性因子总和。
  4. 如果 假设的 fr 大小 与 可伸缩轨道 的 弹性因子 的乘积 小于该轨道的基础大小， 则重新启动此算法， 将所有此类轨道视为不可伸缩。
  5. 返回 假设的 fr 大小。

11.8. 拉伸 auto 轨道

此步骤通过将任何剩余的正的、确定的 自由空间 平等地分配给 拥有 auto 最大轨道大小函数 的轨道，从而扩展这些轨道。 如果 自由空间 是 不确定的， 但 网格容器 有 一个 确定的 最小宽度/高度， 则改用该尺寸来计算此步骤的 自由空间。

12. 分页网格布局

网格容器 可以跨页 在行或列之间以及项目内部断开。 break-* 属性应用于网格容器， 像它们参与的格式化上下文中一样正常。 本节定义了它们如何应用于网格项 以及网格项的内容。

以下断开规则将 碎片化容器 称为“页面”。 相同的规则适用于任何其他 碎片化上下文。 （根据需要，用适当的 碎片化容器 类型替换“页面”。） 参见 CSS 碎片化模块 [CSS3-BREAK]。

碎片化网格容器的确切布局在此级别的网格布局中未定义。 但是，网格容器内部的断开遵循以下规则：

  • break-before 和 break-after 属性应用于 网格项，并传播到它们的网格行。 第一个行的 break-before 属性和最后一行的 break-after 属性传播到网格容器。
  • 网格项内部的强制断开实际上会增加其内容的大小； 它不会触发同级项内部的强制断开。
  • A 类断开机会 发生在行或列之间（取决于适当的轴）， 而 C 类断开机会 发生在第一/最后一行（列）与网格容器内容边缘之间。 [CSS3-BREAK]
  • 当网格容器在断开后继续时， 其 网格项（在碎片化上下文的块流方向上）的可用空间 会因前几页上网格容器碎片所占用的空间而减少。 网格容器碎片所占用的空间是该页上其内容框的大小。 如果因此调整，可用空间变为负数， 则将其设为零。
  • 除了由前一点强加的项目重新排列外， 用户代理应尽量减少网格容器相对于未碎片化流的失真。

样式表
CSS 样式表。
渲染器
解释样式表语义并渲染使用它们的文档的 用户代理。
编辑工具
编写样式表的 用户代理。

如果样式表中使用的所有语句符合通用 CSS 语法和本模块中定义的各个特性的语法，则该样式表符合本规范。

如果渲染器不仅解释样式表，还正确解析并支持本规范中定义的所有特性，并相应地渲染文档，则渲染器符合本规范。然而，由于设备限制而无法正确渲染文档的用户代理并不违反符合性要求。（例如，用户代理不要求在单色显示器上渲染颜色。）

const s = getComputedStyle(gridEl);

这是早期实现的一个意外特性，未经深思熟虑就泄露到后来的实现中。我们打算将其从规范中删除，但在我们定义一个用于获取隐式轨道信息的CSSOM API之前不会这样做，因为目前这是获取该信息的唯一方式，许多页面依赖于此。
