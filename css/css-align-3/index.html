摘要

本模块包含与各种CSS盒子布局模型（块布局、表格布局、弹性布局和网格布局）中盒子在其容器内对齐相关的CSS特性。(文本和行级内容的对齐在[CSS-TEXT-3]和[CSS-INLINE-3]中定义。)

CSS是一种用于描述结构化文档（如HTML和XML）在屏幕、纸张等上呈现的语言。

1. 介绍

CSS Level 1 和 2 允许通过 text-align 对文本进行对齐，并通过平衡 auto 边距对块进行对齐。 然而，除了在表格单元格中， 垂直对齐是不可能的。 随着 CSS 添加更多功能， 在不同维度中对齐盒子的能力变得更加重要。 本模块试图创建一个一致的、通用的盒子对齐模型，以便在所有 CSS 中共享。

注意： 文本和行内级内容的对齐 定义在 [CSS-TEXT-3] 和 [CSS-INLINE-3] 中。

注意： 本规范并不意图改变任何行为 定义在 CSS2.1 中，当这些属性设置为 其初始值时。 如果实施者或其他人注意到任何差异， 请将其报告给 CSSWG 作为错误。

本节（上述内容）不是规范性内容。

1.1. 模块交互

本模块为在 [CSS2] 第 9 章 和 第 10 章 中描述的块布局模型添加了一些新的对齐能力，重新定义了如何解析过度约束的块级盒子边距， 并定义了这些新对齐属性与表格单元格内容对齐的交互，使用 vertical-align， 如 [CSS2] 第 17 章 中定义的那样。

这些属性与网格布局 [CSS-GRID-1] 和弹性盒子布局 [CSS-FLEXBOX-1] 的交互在其各自的 模块中定义。 此处的属性定义优先于 [CSS-FLEXBOX-1]（其具有更小、更早的 允许值子集）。

本模块中的没有属性适用于 ::first-line 或 ::first-letter 伪元素。

1.2. 值定义

本规范遵循 CSS 属性 定义约定，来自 [CSS2]，使用 值定义语法，来自 [CSS-VALUES-3]。 本规范中未定义的值类型在 CSS 值与单位 [CSS-VALUES-3] 中定义。 与其他 CSS 模块的组合可能会扩展这些值类型的定义。

除了在其定义中列出的属性特定值外， 本规范中定义的所有属性 也接受 CSS-wide 关键字 作为其属性值。 为了可读性，这些值没有被明确重复。

1.3. 部分实现

由于预期对本模块中的特性的支持将分阶段部署，涉及到的各种布局模型， 特此澄清 部分实现的规则要求将任何不支持的特性视为无效 适用于任何对齐关键字 该关键字在其适用的所有布局模块中未被支持 对于实施一般支持该属性的布局模型。

例如， 如果一个实现支持 align-self 在 [CSS-GRID-1] 和 [CSS-FLEXBOX-1]， 那么它必须将 start 视为无效 除非它在网格和弹性容器中都得到支持。 然而，如果该实现根本不支持 align-self 的块级元素， 那么缺乏对 align-self: start 的实现并不会触发将其视为无效的要求。

2. 对齐属性概述

CSS中的盒子对齐属性是一组6个属性 用于控制盒子在其他盒子中的对齐。 它们可以沿着两个轴描述：

  • 适用于哪个维度（主/行内与交叉/块），以及
  • 它们是否控制盒子在其父级内的位置，或盒子内容在其内部的位置。

注意：本规范使用“对齐”和“对齐”这两个术语来区分 主/行内和交叉/块维度的对齐。 这种选择在某种程度上是任意的，但拥有这两个术语使得可以在CSS所有布局模型中 （包括CSS Flexbox 1 § 2 Flex Layout Box Model 和术语）中保持一致的命名方案。

以下表格总结了盒子对齐属性及其适用的显示类型。

常见 轴 对齐 适用对象
justify-content 主/行内 内容 在 元素内 (有效调整填充) 块级容器，弹性 容器和网格 容器
align-content 交叉/块
justify-self 行内 元素在 父级内 (有效调整边距) 块级盒子、绝对定位盒子和网格项
align-self 交叉/块 绝对定位盒子，弹性项， 和网格项
justify-items 行内 盒子内 的项目 (控制子项目的justify-self: auto) 块级容器和网格 容器
align-items 交叉/块 弹性容器和网格 容器

注意： *-items属性不会影响元素本身。 当在容器上设置时， 它们指定了任何*-self: auto在容器子项上的解释。

3. 对齐术语

由于该模块定义了CSS中所有布局模式的对齐属性， 引入了一些抽象术语：

对齐对象
对齐对象是由属性对齐的事物。 对于justify-self和align-self， 对齐对象是该属性设置的盒子的外边距框， 并假设该盒子的书写模式。 对于justify-content和align-content， 对齐对象由布局模式定义， 并指其内容的某些方面； 它还假设该属性设置的盒子的书写模式。
对齐容器
对齐容器是对齐对象所对齐的矩形。 这由布局模式定义， 但通常是对齐对象的包含块， 并假设建立包含块的盒子的书写模式。
后备对齐
一些对齐只能在特定情况下满足 或受到空间消耗的限制； 例如，space-between只能在存在多个对齐对象时操作， 基线对齐在满足后，可能不足以吸收所有多余的空间。 在这些情况下，后备对齐生效 （如下所定义） 以完全消耗多余空间。

4. 对齐关键字

所有对齐属性使用一组共同的关键字值， 这些值在本节中定义。 关键字分为三类：

位置对齐
这些关键字将对齐定义为在对齐容器内的绝对位置。
基线对齐
这些关键字将对齐定义为多个对齐主题在一个对齐上下文中的关系。
分布对齐
这些关键字将对齐定义为在对齐主题之间的空间分配。

4.1. 定位对齐：center，start，end，self-start，self-end，flex-start，flex-end，left，和right 关键字

定位对齐 关键字指定了 对齐对象相对于其对齐容器的位置。

这些值具有以下含义：

center (self，content)
将对齐对象居中 于其对齐容器内。
start (self，content)
将对齐对象对齐 至与对齐容器的开始边缘相吻合的适当轴上。
end (self，content)
将对齐对象对齐 至与对齐容器的结束边缘相吻合的适当轴上。
self-start (self)
将对齐对象对齐 至与对齐容器的边缘相吻合，该边缘对应于对齐对象在适当轴上的开始侧。
self-end (self)
将对齐对象对齐 至与对齐容器的边缘相吻合，该边缘对应于对齐对象在适当轴上的结束侧。
flex-start (自我，内容)
仅在弹性布局中使用。 [CSS-FLEXBOX-1] 将对齐对象对齐到对齐容器的边缘，适用于弹性容器的主起始或交叉起始侧。

在弹性格式上下文外使用时，此值表现为start。 即在不是弹性项目（或 假装是弹性项目， 例如在确定静态位置时，如果是一个弹性容器的子元素） 时，此值在自我对齐属性中表现为start， 而在不是弹性容器的盒子中，此值在内容分配属性中表现为start。

flex-end (自我，内容)
仅在弹性布局中使用。 将对齐对象对齐到对齐容器的边缘，适用于弹性容器的主末端或交叉末端。

在弹性格式上下文外使用时，此值表现为end。 即在不是弹性项目（或 假装是弹性项目， 例如在确定静态位置时，如果是一个弹性容器的子元素） 时，此值在自我对齐属性中表现为end， 而在不是弹性容器的盒子中，此值在内容分配属性中表现为end。

left (仅 justify-*)
将对齐对象对齐到 对齐容器的左边或物理左边边缘， 取决于在适当的轴上哪个边缘。

如果属性的轴与左↔右轴不平行， 此值表现为start。目前，属性轴 不平行于左↔右轴的唯一情况 是在列弹性盒子中。

right (仅 justify-*)
将对齐对象对齐到 对齐容器的右边或物理右边边缘， 取决于在适当的轴上哪个边缘。

如果属性的轴与左↔右轴不平行， 此值表现为start。目前，属性轴 不平行于左↔右轴的唯一情况 是在列弹性盒子中。

有两个语法术语用于表示这些值的某些子集：

<self-position>
此集合由 justify-self 和 align-self 使用，以在其 对齐容器 内对齐框， 还被 justify-items 和 align-items 使用（以指定 justify-self 和 align-self 的默认值）。 
<self-position> = center | start | end | self-start | self-end |
        flex-start | flex-end
<content-position>
此集合由 justify-content 和 align-content 使用，以对齐框的内容。 
<content-position> = center | start | end | flex-start | flex-end

注意： left 和 right 被排除在 <self-position> 和 <content-position> 之外， 尽管它们是有效的 位置对齐 值，用于 justify-content/justify-self/justify-items， 因为它们不允许出现在 align-* 属性中。 它们被明确包含在 justify-* 属性的语法中。

start 和 end 关键词是 流动相关的： 它们使用 书写模式 来确定对齐的侧边。

left 和 right 关键词 是绝对的（不是 流动相关的）。

start 和 end 关键词 在 行内 和 块 轴中都有意义：start 总是定向于文本的开始 （对于像英语这样的从左到右/从上到下的语言为左上） 而 end 总是定向于 文本的结束。

start 和 end 关键词 使用 书写模式 的 对齐容器， 以帮助保持一致性。 但如果需要使用 对齐主题 的 书写模式 进行对齐， 可以使用 self-start 和 self-end 关键词。

对齐关键词的行为在 垂直书写模式 中是相似的：start 和 end 相对于相关轴的文本的开始/结束。 left 和 right 关键字 被解释为 line-left 和 line-right， 相对于 LTR/RTL 文本的“左”侧和“右”侧。
对于大多数布局模型（块、表格、网格等）， justify-* 属性始终在 行内轴 中对齐，而 align-* 属性始终在 块轴 中对齐。

另一方面，Flexbox 中 justify-* 在 主轴 中对齐，align-* 属性 在 交叉轴 中对齐。 这取决于 flex-direction 的值： 当 flex-direction 为 row 或 row-reverse 时，它与其他布局模式相匹配 （行内轴 对应 justify-*，块轴 对应 align-*）； 当 flex-direction 为 column 或 column-reverse 时，则对应关系相反。

更容易理解 "start" 和 "end" 在正交流中的双轴特性。

4.2. 基线对齐：baseline 关键字 和 first/last 修饰符

参见CSS Writing Modes 3 § 4.1 基线简介。

基线对齐 是一种位置对齐形式 ，它通过匹配多个 对齐对象 的 对齐基线，在 共享对齐上下文 （例如一行中的单元格）内对齐多个对象。 如果 对齐对象 在一个 基线共享组 中的位置 并未完全受 基线对齐 约束（即，它们可以在各自的 对齐容器 中移动，同时保持基线对齐）， 则它们将尽可能保留其 基线对齐 的同时被 后备对齐。

基线对齐关键字用 <baseline-position> 语法术语表示：

<baseline-position> = [ first | last ]? && baseline

first 和 last 值为一个框提供了 基线对齐偏好： 要么是“first”，要么是“last”，默认为“first”。

值具有以下含义：

baseline
计算为 first baseline，定义如下。
first baseline
指定参与 first-baseline 对齐： 将框的 first baseline set 的 对齐基线 与其 基线共享组 的相应基线对齐。 详情见 § 9.3 基线对齐框。

后备对齐 对于 first baseline 是 安全自开始（用于 自对齐） 或 安全开始（用于 内容分配）。

last baseline
指定参与 last-baseline 对齐： 将框的 last baseline set 的 对齐基线 与其 基线共享组 的相应基线对齐。 详情见 § 9.3 基线对齐框。

后备对齐 对于 last baseline 是 安全自结束（用于 自对齐） 或 安全结束（用于 内容分配）。

当为 align-content 指定时，这些值会触发 基线内容对齐，移动框内的内容，可能还会影响框本身的大小。参见 § 5.4 基线内容对齐。

当为 align-self/justify-self 指定时，这些值会触发 基线自对齐，移动整个框在其容器内，这可能会影响其容器的大小。参见 § 6.4 基线自对齐。

当在框上指定 基线自对齐 时，首先执行 内容分配，然后将结果框及其内容 自对齐。然而，如果该框在同一轴上也具有 基线内容对齐，或者其在同一轴上的 内容分配属性 为 normal，则其在该轴上的 使用 内容分配属性 为 start 或 安全结束，用于其 基线对齐偏好 的 基线自对齐 为“first”或“last”。

4.3. 分布对齐: 拉伸, 分隔, 环绕, 和 均匀分布 关键词

分布对齐 值被 justify-content 和 align-content 用来在其 对齐对象 之间分配容器的额外空间。

当空间无法以这种方式分配时，这些值的行为就像它们的 后备对齐。每个分配值都有一个关联的默认 后备对齐。 (该模块的未来版本可能允许显式指定 后备对齐。)

分隔
对齐对象 在 对齐容器 中均匀分布。 第一个 对齐对象 紧贴 对齐容器 的起始边缘， 最后一个 对齐对象 紧贴 对齐容器 的结束边缘， 剩余的 对齐对象 分配以使任何两个相邻的 对齐对象 之间的间距相同。

该值的默认 后备对齐 是 flex-start。(对于弹性布局以外的布局模式， flex-start 与 start 相同。)

环绕
对齐对象 在 对齐容器 中均匀分布， 两端有一半大小的空间。 对齐对象 分配以使任何两个相邻的 对齐对象 之间的间距相同， 且第一个和最后一个 对齐对象 之前和之后的间距是其他间距的一半。

该值的默认 后备对齐 是 安全中心。

均匀分布
对齐对象 在 对齐容器 中均匀分布， 两端有一整大小的空间。 对齐对象 分配以使任何两个相邻的 对齐对象 之间的间距相同， 在第一个 对齐对象 之前和最后一个 对齐对象 之后的间距也是相同的。

该值的默认 后备对齐 是 安全中心。

拉伸
如果 对齐对象 的总大小小于 对齐容器 的大小， 任何 自动 大小的 对齐对象 将其大小均等增加（而不是按比例增加）， 同时仍然遵守 最大高度/最大宽度（或等效功能）的约束， 使得总大小恰好填充 对齐容器。

该值的默认 后备对齐 是 flex-start。(对于弹性布局以外的布局模式， flex-start 与 start 相同。)

这些值用 <content-distribution> 语法术语表示:

<content-distribution> = space-between | space-around | space-evenly | stretch

5.3. 溢出和滚动位置

当内容分布属性在滚动容器上设置且有溢出的对齐主题时， 它们不是改变内容的布局位置， 而是改变其初始滚动位置，使得滚动容器中初始可见的内容 满足对齐主题和对齐容器的预期对齐。

注意：滚动条的存在可能会改变滚动容器的内容框大小， 从而影响对齐容器和/或对齐主题的大小。

例如， 如果一个可滚动的弹性容器设置为justify-content: flex-end（或 justify-content: flex-start与flex-flow: row-reverse）， 它将最初显示为滚动到其内容的主末边， 并且其内容将看起来溢出其主起始边缘。

所有这些并没有改变滚动坐标的分配： 原点仍然是在start-对齐 的容器中， 只是它可能最初位于滚动区域外。

注意： 对齐主题不一定与可滚动溢出区域相同： 溢出对齐主题的内容， 如绝对定位或流出元素， 扩大了可滚动溢出区域但不扩大对齐主题， 不会与其在可视范围内的位置相匹配。

5.4. 基线内容对齐

参与类似行的布局上下文的盒子(共享对齐上下文) 可以相互基线对齐。基线内容对齐有效地增加了盒子的填充 以将其内容的对齐基线 与其基线共享组中其他基线对齐的盒子的对齐基线对齐。

基线 内容对齐仅在align-content轴 与盒子的块轴 平行时适用（即它不适用于“列”弹性容器）； 否则使用回退对齐。

参与基线内容对齐的盒子集依赖于布局模型：

表格单元格：
一个(未替换)表格单元格在其最左/右行中参与 第一个/最后一个基线内容对齐， 如果其内联轴与表格本身的内联轴平行且 其计算的align-content为第一个基线（最后一个基线）。

注意：基线 内容对齐共享列的单元格不支持； 但是如果有足够的需求和实现者的兴趣，将来可能会添加此功能。

弹性项目：
一个未替换 弹性项目 参与 第一个/最后一个基线内容对齐， 如果其计算的align-content为第一个基线/最后一个基线，并且其内联轴与主轴平行。
网格项目：
一个未替换 网格项目 参与 第一个/最后一个基线内容对齐， 在其最左/右行或列中（ 取决于其内联轴的方向） 并且如果其计算的align-content为第一个基线/最后一个基线。

何时协调自对齐偏好？

一个框的 起始 边距边缘 对齐到其 包含块 的相应边缘，并且它对“第一” 基线对齐偏好 具有协调自对齐偏好时，在相关轴中：

  • 没有 auto 边距，且相关 自对齐属性 不是或与 拉伸 或 自启动 一致；或者

  • 只有一个 auto 结束-边缘 边距， 它吸收任何正的自由空间并禁用任何 自对齐属性 的效果，并且 其 边距框 在情况下不会溢出其 包含块，这将导致它有效地结束对齐（例如，具有相反的 包含块 书写模式）。

一个框的 结束 边距边缘 对齐到其 包含块 的相应边缘，并且它对“最后” 基线对齐偏好 具有协调自对齐偏好时，在相关轴中：

  • 没有 auto 边距，且相关 自对齐属性 不是或与 自结束 一致并且 其自对齐是将导致的结果 从 不安全 溢出对齐；或者

  • 只有一个 auto 起始-边缘 边距， 它吸收任何正的自由空间并禁用任何 自对齐属性 的效果，并且 其 边距框 在情况下不会溢出其 包含块，这将导致它有效地开始对齐。

请参阅 § 9.3 按基线对齐框 获取更多详细信息。 基线内容对齐 可以增加框的固有大小。

6.5. 静态定位插图的绝对定位盒子的大小影响

对于绝对定位的盒子， 其行内轴偏移量均为 auto， 计算 可用空间 的 行内大小 也受对齐的影响。

注意： 在 [CSS2] 中，可用空间 取决于 direction 属性 的 静态定位包含块。 （见 CSS2§10.3.7 和 CSS2§10.3.8。） 从根本上讲，这些规则将一个 auto 插图（默认情况下，起始边缘插图）设置到 静态定位矩形 的相应边缘，而另一个设置到 包含块 的相应边缘（即将插图设置为零）。 就像 自我对齐属性 替换 包含块 的 direction 查找以进行放置一样， 它们也替换此处的查找以进行大小调整。

因此， 在解释 CSS2§10.3.7 和 CSS2§10.3.8 中的规则时， 每当引用 direction 属性的 静态定位包含块 时， 请改为引用 align-self 或 justify-self 属性（以适用的轴为准）， 将左等效对齐视为定义于 ltr，将右等效对齐视为定义于 rtl。 将 normal 视为 start，任何 分布对齐 值视为其 后备对齐。

在 center 对齐的情况下， 该盒子的 可用空间 为 静态定位矩形的中心与包含块的最近边缘之间的距离的两倍。

例如， 当盒子的 包含块 的 direction 为 ltr，而其自身的 justify-self 为 end， 则应用 direction: rtl 的规则； 当 direction 为 rtl 且 justify-content 为 space-between， 则应用 direction: rtl 的规则； 等等。 对于 center（或其等价物）， 将两侧设置为匹配 静态 定位。 然后，绝对定位盒子将大小调整为 结果空间（下限为零）。

注意： align-self/justify-self 属性 还可以修改大小的其他方面： 例如，justify-self: stretch 将替换 “适应内容”（fit-content）大小调整为 可用空间 的 拉伸适配大小（消耗所有的 可用空间）。 这是与此处的可用空间调整独立的效果。

注意： 这仅影响如何计算 可用空间 以调整绝对定位盒子的大小； 其对齐如前面章节所规定。

8. 盒子之间的间隙

虽然 margin 和 padding 可用于指定单个盒子周围的视觉间距，但有时在给定布局上下文中全局指定相邻盒子之间的间距会更方便，特别是当盒子之间的间距与容器边缘的第一/最后盒子之间的间距不同时。

gap 属性及其 row-gap 和 column-gap 子属性，为 多列、flex 和 grid 布局 提供了此功能。

8.1. 行间距和列间距: row-gap 和 column-gap 属性

名称: row-gap, column-gap
值: normal | <length-percentage [0,∞]>
初始值: normal
适用对象: 多列容器, flex容器, grid 容器
继承: 无
百分比: 见§ 8.3 百分比在间距属性中
计算值: 指定关键字，其他为计算后的<length-percentage> 值
规范顺序: 按语法
动画类型: 按计算值类型

这些属性指定容器中项目之间的固定长度的间距，在它们之间添加空间——与space-between关键字相似， 属于内容分布属性，但大小为固定值，而不是剩余空间的分数。 column-gap属性指定“列”之间的间距， 在容器的内联轴 中分隔框，与内联轴的边距相似； 而row-gap表示“行”之间的间距， 在容器的块轴中分隔框。

值的含义如下:

<length-percentage [0,∞]>

指定“行”或“列”之间的间距， 由应用的布局模式定义； 详细信息见下文。

负值无效。 对于百分比， 见§ 8.3 百分比在间距属性中。

normal

normal表示在多列容器中使用的值为1em， 在其他上下文中使用的值为0px。

间距效果在项目之间形成最小间距： 通过justify-content/align-content可以添加额外的间距。 这样的额外空间实际上会增加这些间距的大小。

这些属性的具体处理方式因布局容器而异：

多列容器
column-gap指定相邻列框之间的间距。 见[CSS-MULTICOL-1]了解这如何影响多列元素的布局。 row-gap当前不适用。
flex 容器
当应用于主轴（例如，在行的column-gap 的flex容器）， 表示项目之间的最小间距 （好像在相邻flex项目之间插入了额外的固定大小的边距）。

当应用于交叉轴（例如，在行的row-gap 的flex容器）， 表示相邻flex行之间的最小间距。

grid 容器
row-gap 和 column-gap属性， 当在grid容器上指定时， 定义间距在grid 行和grid列之间， 分别。 见CSS Grid Layout 1 § 10.1 间距: row-gap, column-gap 和 gap 属性了解详细信息。

注意: 表格框不使用gap属性 来指定其单元格之间的分隔。 相反，它们使用border-spacing属性， 该属性具有略微不同的功能: 它是可继承的， 并且它还指定最外层单元格和表格边框之间的额外间距 （类似于space-evenly而不是space-between）。

8.3. 在gap属性中的百分比

一般而言， 通过gap属性引入的间隙 意图作为具有间隙大小的空项目/轨道/etc 也就是说， 作者应该能够通过在容器中插入额外的空项目/轨道/etc来重现gap的效果。

在网格布局中

8.4. 旧版间隙属性: grid-row-gap, grid-column-gap, 和 grid-gap 属性

  • grid-row-gap必须被视为row-gap属性的缩写

  • grid-gap必须被视为gap属性的缩写

在所有这三种情况下，旧版属性必须与它们别名的属性具有相同的语法，并将值“转发”到它们所别名的属性。

9. 基线对齐细节

在基线共享组中的盒子通过它们的对齐基线相互对齐。 例如，在水平书写模式中， 在同一行的表格单元格上指定align-content: baseline 将对齐它们的第一格式化行 的基线。 本节明确了在考虑到国际化现代CSS中存在的多种基线和书写模式 时，基线对齐的具体执行方式。

基线集合是一组 与共同基线表关联的基线（字母、中央等）。 通常，排版传统只使用其中一种， 但不同的书写系统使用不同的基线， 混合书写系统可能导致在单行中使用多个基线。 有关基线和书写模式的更多信息，请参见CSS书写模式 3 § 4.1 基线介绍。

9.1. 确定框的基线

每个框在给定轴上，可能具有 一个 首基线集（和 末基线集） 这通常对应于框内第一/最后一行文本的 基线集。 对齐基线， 是用于在其 对齐上下文中对齐框的基线， 是其 基线集 中的一个基线， 通常是与 共享对齐上下文 关联的主导基线。 （参见 dominant-baseline 和 alignment-baseline 属性，在 [CSS-INLINE-3]。）

框的首个和最后一个 基线集 的确定方式取决于布局模型，具体如下：

行盒子
行盒子的第一个/最后一个基线集合 是从其根行盒 的主导基线和字体设置生成的。
块容器
基线集合的第一个/最后一个 块容器取自块容器中第一个/最后一个流动行盒 或第一个/最后一个流动块级子元素，这些元素贡献了一组第一个/最后一个基线， 以先出现的为准。 如果没有这样的行盒或子元素， 则该块容器没有基线集合。

然而，出于遗留原因， 块级或行内级 滚动容器始终具有最后一个基线集合， 其所有基线都对应于其块结束 边缘。

多列容器
基线集合的第一个 多列容器是与其列或多列跨行器的第一个基线集合，其具有对应于多列容器 的对齐基线的最高（块开始最上）基线。 如果没有这样的行盒或子元素， 则多列容器没有第一个基线集合。

最后一个基线集合是类似的， 但使用最后的基线集合和最低 (块结束最下)基线。

表格
表格盒子的第一个/最后一个基线集合 是其第一/最后一行的第一个/最后一个基线集合。

在查找行内块的第一个/最后一个基线集合时， 必须跳过由表格盒子贡献的任何基线。 （此特性是来自[CSS2]的遗留行为。）

表格行
如果行中的任何单元格参与沿第一个基线/最后基线对齐 沿着行内轴， 行的第一个/最后一个基线集合 是生成 自它们共享的对齐基线和行的首个可用字体， 在对齐完成后。 否则，行的第一个/最后一个基线集合 是合成 自行中的最低和最高内容边缘。[CSS2]
弹性容器
参见Flex Baselines在[CSS-FLEXBOX-1]。
网格容器
参见Grid Baselines在[CSS-GRID-1]。

要从单一基线为框 生成基线，使用字体设置中的基线表和该框的 首个可用字体， 并将该基线集对齐到给定的单一基线。

如果参与 基线对齐的框没有 基线集， 则其 对齐基线 将根据参与的 格式化上下文 的规则进行 合成。 从矩形（或两条平行线）中 合成基线 ，从 下划线 行合成字母基线， 通过平均两个边缘或线的位置合成中央基线。 有关合成附加基线的规则，请参见 CSS Inline Layout 3 §  附录 A：合成对齐度量。

注意：用于从框 合成 基线的边缘取决于它们的 格式化上下文： 行内级框 从其边距边缘合成 [CSS-INLINE-3]， 表格单元格 从其内容边缘合成 [CSS2]， 网格和弹性项目 从其边框边缘合成 [CSS-GRID-1] [CSS-FLEXBOX-1]。

一般而言，用于确定框、形状或其他对象的 书写模式 被用来确定合成的 下划线 和 上划线 边缘。 然而，当该 书写模式 的 块流方向 与 对齐上下文 的轴平行时， 必须假设与轴兼容的 书写模式：

  • 如果建立对齐上下文的盒子的块流方向与 对齐上下文的轴正交， 请使用其书写模式。

  • 否则，假设是horizontal-tb或vertical-lr 书写模式， 无论哪个与盒子的书写模式正交。

为了找到盒子的基线， 它及其所有具有滚动机制的流内后代（请参见overflow属性） 必须被视为如果已滚动到其初始滚动位置。 此外， 如果滚动容器的第一个/最后一个基线位置在其边框边缘之外， 则该基线的位置会被限制在边框边缘。

9.2. 基线对齐分组

一个基线共享 组由参与基线对齐的盒子组成。 这只有在它们同时满足以下条件时才可能：

  • 沿着一个与它们正在进行基线对齐的轴垂直的对齐上下文共享。 （例如，具有align-self: baseline的网格项在网格的 块轴上进行基线对齐，因此与同一行中的其他项参与对齐。）
  • 具有兼容的基线对齐偏好 （即，想要对齐的基线位于对齐上下文的同一侧）。

注意： 参与基线 内容对齐的盒子和 参与基线自我对齐的盒子可以成为同一个基线共享组的一部分， 从而能够相互对齐。 两种方法之间的区别在于 在执行对齐时额外空间插入的位置 （在盒子自己的边框内或外）。

盒子沿着特定轴共享对齐 上下文， 由特定盒子建立，当它们是：

  • 位于同一行的表格单元格，沿着表格的行（行内）轴，由行盒子建立
  • 位于同一行的网格项，沿着网格的行（行内）轴，由网格容器建立
  • 位于同一列的网格项，沿着网格的列（块）轴，由网格容器建立
  • 位于同一弹性行的弹性项目，沿着弹性容器的主轴，由弹性容器建立

注意： 从概念上讲， 行内级盒子在行盒中也共享自我对齐上下文 并参与基线共享组； 但是它们仅响应于vertical-align属性进行基线对齐， 而不考虑本模块中定义的任何属性。 参见[CSS-INLINE-3]。

如果一个盒子跨越多个共享对齐上下文， 则它将在其最开始/最后的共享对齐上下文 沿着该轴进行首/末基线对齐。 例如，一个跨越三行的表格单元格 在与它跨越的第一行的表格单元格进行首基线对齐， 或者在与它跨越的最后一行的表格单元格进行末基线对齐。

位于基线对齐偏好的两个盒子在一个基线共享组中是兼容的，如果它们满足：

  • 相同的块流方向和相同的基线 对齐偏好

  • 相反的块流方向和相反的基线 对齐偏好

9.3. 通过基线对齐盒子

给定一组对齐主题及其基线 它们都属于一个单一的基线共享组， 对齐主题的基线对齐方式如下：

  1. 从该组的第一个可用字体生成基线共享组的基线表 并将此基线表的镜像覆盖，通过对齐它们的中央基线。 这些是对齐主题将要对齐的基线“网格”。

  2. 根据指定的对齐基线 将每个对齐主题对齐到组的基线表 或其镜像，取决于哪个与对齐主题的行方向匹配。 除非另有说明（例如，通过alignment-baseline属性）， 对齐基线是主基线的对齐上下文。

  3. 将对齐的 基线共享组 根据其 对齐容器 放置在 后备对齐 的位置。 后备对齐 的 基线共享组 是其项的 后备对齐 经过解析到 物理方向 的结果。

  4. 对于首/末基线内容对齐， 然后在对齐容器的开始/结束内容边缘和对齐主题的边缘之间添加必要的最小额外空间 以对齐对齐上下文中所有对齐容器的开始/结束边距边缘，同时在基线共享组内保持基线对齐。

附录 A：静态定位术语

当给定轴上的内嵌属性为auto时，在绝对定位盒子上， CSS2使用其静态位置来解决其大小和位置。 参见层叠样式表第2级修订版1 (CSS 2.1) 规范 § #abs-non-replaced-width。 盒子对齐属性修改这些计算， 正如它们在其他布局模式中的尺寸和定位计算一样。 这些修改涉及到一个静态定位矩形， 其边缘表示盒子从其包含块每一侧的静态位置。

静态定位矩形及其对应的静态位置 由其“假想盒子”的布局模型定义：

块布局
静态位置的块级盒子在[CSS2]第10章中定义。 静态定位矩形是一个零厚度的矩形 跨越盒子静态定位包含块的行轴侧（见CSS2§10.3.7）； 并位于其块开始的静态位置（见CSS2§10.6.4）。
行内布局
静态位置的行内级盒子在[CSS2]第10章中定义。 静态定位矩形是一个零厚度的矩形 跨越包含其“假想盒子”的行盒的行上/行下 边缘（见CSS2§10.3.7）； 并位于其行开始的静态位置。
弹性布局
静态定位矩形的子元素在弹性容器中对应于弹性容器的内容边缘。 见CSS Flexbox 1 § 4.1 绝对定位的弹性子元素。
网格布局
默认情况下，静态定位矩形的子元素在网格容器中对应于网格容器的内容边缘。 然而，如果该网格容器也建立了 盒子的实际包含块， 则由网格区域指定的网格定位属性建立其静态定位矩形。 见网格容器子元素的静态位置在[CSS-GRID-1]。

符合性类别

本规范的符合性定义为三类符合性：

样式表
一个 CSS 样式表。
渲染器
一个 用户代理，它解释样式表的语义并渲染使用它们的文档。
创作工具
一个 用户代理，它编写样式表。

一个样式表符合本规范的要求，如果它所使用的语法根据本模块定义是有效的，并符合该模块定义的各个特性的个别语法。

一个渲染器符合本规范的要求，如果它除了根据适当的规范解释样式表外，还支持本规范定义的所有特性，正确解析并相应地渲染文档。然而，由于设备的限制，用户代理无法正确渲染文档并不使其不符合规范。（例如，用户代理不需要在单色显示器上渲染颜色。）

一个创作工具符合本规范的要求，如果它编写的样式表在语法上符合通用 CSS 语法和该模块中每个特性的个别语法，并满足该模块描述的样式表的所有其他符合性要求。

部分实现

为了让作者可以利用向前兼容的解析规则来分配回退值，CSS 渲染器必须将任何没有可用支持级别的 at-rules、属性、属性值、关键字和其他语法构造视为无效（并适当忽略）。特别是，用户代理不得选择性地忽略不受支持的组件值，并在单个多值属性声明中尊重支持的值：如果任何值被视为无效（因为不受支持的值必须如此），CSS 要求整个声明被忽略。

不稳定和专有特性的实现

为了避免与未来稳定的 CSS 特性发生冲突，CSSWG 建议 遵循最佳实践，以实现 不稳定 特性和 专有扩展。
