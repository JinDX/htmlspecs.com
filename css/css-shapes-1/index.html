摘要

CSS 形状描述用于 CSS 的几何形状。对于第 1 级，CSS 形状可以应用于浮动。浮动上的圆形形状将导致行内内容 环绕 圆形形状，而不是浮动的边界框。

CSS 是一种描述结构化文档（如 HTML 和 XML） 在屏幕、纸张等上呈现的语言。

1. 引言

本节不具规范性。

形状定义了任意几何形状，可以用作 CSS 值。 本规范定义了控制元素的几何形状的属性 浮动区域。 shape-outside 属性使用形状值 来定义浮动的 浮动区域。

注意：未来的 CSS Shapes 将允许在非浮动元素上使用形状。 其他 CSS 模块也可以使用形状， 例如 CSS 掩膜 [CSS-MASKING] 和 CSS 排除 [CSS3-EXCLUSIONS]。

注意：如果用户代理实现了 CSS Shapes 和 CSS Exclusions， shape-outside 属性定义了 排除的区域。

注意：未来的 CSS Shapes 将定义一个 shape-inside 属性， 该属性将定义一个形状来 包裹 内容。

1.1. 模块交互

该模块扩展了在 [CSS2] 第 9 章中定义的浮动特性。

1.2. 值

本规范遵循 CSS 属性定义约定， 来自 [CSS2]，使用 值定义语法，来自 [CSS-VALUES-3]。 本规范未定义的值类型在 CSS 值与单位 [CSS-VALUES-3] 中定义。 与其他 CSS 模块的结合可能会扩展这些值类型的定义。

除了在定义中列出的特定属性值外， 本规范中定义的所有属性 还接受 CSS-wide 关键字 作为其属性值。 为了可读性，它们没有被重复明确列出。

1.3. 动画值

预计 CSS 将包含动画样式之间过渡的方法。 为此， 本模块为每个属性包含一行“可动画”， 指定该属性的值是否以及如何可以被动画化。

1.4. 术语

包裹

本规范使用术语 包裹 来指代 流动内容 绕过 浮动区域， 在 [CSS2] 第 9 章中定义。 内容 包裹 在左浮动框的右侧， 内容 包裹 在右浮动框的左侧。 这种 包裹 的一个结果是，邻近浮动的行框 会根据需要缩短 以避免与 浮动区域 发生交叉。

浮动区域

用于 包裹 内容 绕过浮动元素的区域。 浮动行为的规则 使用 浮动区域 的边界 来确定内容的流动位置。 默认情况下， 浮动区域 是浮动元素的 外边距框（注意这可能 与 浮动区域 不同 由 margin-box 值生成， 该值包括边框半径的曲率）。 本规范的 shape-outside 属性 可用于定义任意的、 非矩形的 浮动区域。

2. 与盒模型和浮动行为的关系

虽然用于将 行内流内容包裹 在浮动之外的边界 可以使用形状定义， 但实际的盒模型并没有变化。 如果元素指定了 边距、边框或填充， 它们将根据 [CSS3BOX] 模块计算和渲染。 此外，浮动定位和堆叠并不受 使用形状定义的 浮动区域 的影响。

当使用形状定义 浮动区域 时， 该形状会被裁剪到浮动的边距框。 换句话说， 形状只能减少 浮动区域， 而无法增加它。 减少的 浮动区域 可能对某些行框 没有影响，这些行框通常会受到浮动的影响。 如果形状不包围任何区域， 形状的边缘仍会用于定义 浮动区域。

由形状定义的 浮动区域 可能会在所有边减少正常的 浮动区域， 但这并不允许内容在 浮动的两侧 包裹。 左浮动带有 shape-outside 仍然只允许内容在右侧 包裹， 而右浮动只允许在左侧 包裹。

在以下示例中， 左右浮动的 img 元素 使用 shape-outside 属性指定了一个三角形形状。

<img class="left" src="hand.svg">
<img class="right" src="hand.svg">
<p>
  有时网页的文本内容看起来
  像是在引导你的注意力
  向页面上的某个点
  以引导你跟随某个特定链接。有时
  你并没有注意到。
</p>

<style type="text/css">
  .left {
    shape-outside: polygon(0 0, 100% 100%, 0 100%);
    float: left;
    width: 40%;
    height: 12ex;
    transform: scaleX(-1);
  }

  .right {
    shape-outside: polygon(100% 0, 100% 100%, 0 100%);
    float: right;
    width: 40%;
    height: 12ex;
  }

  p {
    text-align: center;
  }
</style>

由于形状被裁剪到浮动的边距框， 将该形状添加到上面的左浮动 将导致相同的渲染结果。

shape-outside: polygon(0 0, 500% 500%, 0 500%);

一个不包围任何区域的形状 仍然具有对 浮动区域 的贡献。

这个内嵌形状是一个垂直线，位于 参考框的中点。 这个中点边缘被用作 内容包裹的浮动区域的边缘。

shape-outside: inset(0% 50% 0% 50%);

如果内嵌值的总和大于宽度， CSS 背景 3 § 4.5 重叠曲线 规则用于确定 矩形的边缘。 该形状在参考框的左侧 25% 处形成垂直边缘。

shape-outside: inset(0% 150% 50% 0%);

如果形状仅为一条水平线， 则它是一个空的浮动区域， 对包裹没有影响。 请注意，在此示例中，shape-margin 必须为 0px （否则该线将扩展以包围一个区域）。

shape-outside: inset(50% 0% 0% 50%);
shape-margin: 0px;

一个 shape-outside 可以在 浮动区域 的左侧和右侧创建开放区域。 内容仍然只在浮动一侧 包裹。 在图片中， 形状呈蓝色渲染， 形状外的内容区域呈淡紫色。 

shape-outside: polygon(50px 0px, 100px 100px, 0px 100px);

以下样式创建了一个 形状，远小于 浮动的内容区域， 并在浮动上添加了上边距。 在图片中， 形状呈蓝色渲染， 形状外的内容区域呈淡紫色， 浮动框的边距为绿色。

.float-left {
  shape-outside: polygon(0% 50%, 50% 100%, 0 100%);
  float: left;
  width: 100px;
  height: 100px;
  margin-top: 20px;
}

下一个图片显示了如果这两个浮动并排堆叠的可能结果。 请注意，这些浮动是通过它们的边距框定位的， 而不是 浮动区域。

3. 基本形状

<basic-shape> 类型 可以使用基本形状函数来指定。 当使用这种语法 定义形状时， 每个使用 <basic-shape> 值的属性定义了 参考框。 形状的坐标系统 原点位于 参考框 的左上角， x轴向右延伸， y轴向下延伸。 所有以百分比表示的长度 从 参考框 的使用维度解析。

3.1. 支持的形状

以下形状是支持的。 所有 <basic-shape> 值使用 函数表示法， 并在此处使用 值定义语法 进行定义。

inset() = inset( <length-percentage>{1,4} [ round <'border-radius'> ]? )
定义一个内嵌矩形 通过从每个边缘的 参考框 的内嵌值。

如果提供的 <length-percentage> 值少于四个， 省略的值默认与 margin 简写相同： 省略的第二或第三个值默认与第一个相同， 省略的第四个值默认与第二个相同。

四个 <length-percentage> 定义矩形的顶部、右侧、底部和左侧边缘的位置， 分别作为从对应边缘的 参考框 的内嵌。

在任何一个维度中一对内嵌值 相加超过已用维度 （例如左侧和右侧内嵌各为 75%） 使用 CSS Backgrounds 3 § 4.5 重叠曲线 规则 按比例减少内嵌效果至 100%。

例如，指定 inset(75% 0 50% 0) 的顶部和底部边缘相加为 125% 的 参考框 的高度。 它们按比例减少至总和为 100%, 相同于指定 inset(60% 0 40% 0)。

可选的 <'border-radius'> 参数 定义矩形的圆角 使用 border-radius 简写 语法。

xywh() = xywh( <length-percentage>{2} <length-percentage [0,∞]>{2} [ round <'border-radius'> ]? )
定义一个矩形 通过从 参考框 的顶部和左侧边缘的偏移量， 以及指定的宽度和高度。

四个 <length-percentage> 分别定义， 从 参考框 左侧边缘的内嵌， 从 参考框 顶部边缘的内嵌， 矩形的宽度， 以及矩形的高度。

注意： 此语法受到 viewBox 属性的启发。

可选的 <'border-radius'> 参数 定义内嵌矩形的圆角 使用 border-radius 简写 语法。

rect() = rect( [ <length-percentage> | auto ]{4} [ round <'border-radius'> ]? )
定义一个矩形 通过从 参考框 的顶部和左侧边缘的内嵌。

四个 <length-percentage> 定义 矩形的顶部、右侧、底部和左侧边缘的位置， 分别作为从 参考框 的顶部边缘（第一个和第三个值） 或左侧边缘（第二个和第四个值）的内嵌。

auto 值使边缘与对应的 参考框 边缘重合： 相当于 0% 作为第一个（顶部）或第四个（左侧）值， 相当于 100% 作为第二个（右侧）或第三个（底部）值。

可选的 <'border-radius'> 参数 定义矩形的圆角 使用 border-radius 简写 语法。

circle() = circle( <shape-radius>? [ at <position> ]? )
  • shape-radius 参数表示 r，圆的半径。 负值无效。 此处的百分比值 从 参考框 的使用宽度和高度解析为
    sqrt(width2+height2)/sqrt(2)。
  • position 参数定义 圆的中心。 如果省略，默认为中心。
ellipse() = ellipse( [ <shape-radius>{2} ]? [ at <position> ]? )
  • shape-radius 参数表示 rx 和 ry， 椭圆的 x 轴和 y 轴半径， 按此顺序。 任何半径的负值无效。 此处的百分比值相对于使用宽度（对于 rx 值） 和参考框的使用高度（对于 ry 值）进行解析。
  • position 参数定义 椭圆的中心。 如果省略，默认为中心。
polygon() = polygon( <'fill-rule'>? , [<length-percentage> <length-percentage>]# )
  • <'fill-rule'> - 用于确定多边形内部的填充规则。 有关详细信息，请参见 SVG 中的 fill-rule 属性。 可能的值是 nonzero 或 evenodd。 省略时的默认值为 nonzero。
  • 列表中的每对参数表示 xi 和 yi - 多边形第 i 个顶点的 x 和 y 轴坐标。

用户代理必须通过连接最后一个顶点 与列表的第一个顶点来关闭多边形。

path() = path( [<'fill-rule'>,]? <string> )
  • <'fill-rule'> - 用于确定路径内部的填充规则。 有关详细信息，请参见 SVG 中的 fill-rule 属性。 可能的值是 nonzero 或 evenodd。 省略时的默认值为 nonzero。
  • <string> 表示一个 SVG 路径数据字符串。 不符合 SVG 1.1 语法和解析规则的路径数据字符串， 或者符合但定义为空路径的字符串， 被视为 无效，并导致整个 path() 无效。 初始位置由路径字符串中的第一个“移动到”参数定义。 对于初始方向，请遵循 SVG 1.1。

用户代理必须通过隐式 closepath 命令（“z” 或 “Z”）关闭路径， 如果路径字符串中未包含该命令， 适用于需要闭合循环的属性 （如 shape-outside 和 clip-path）。

未定义的参数定义如下：

<shape-radius> = <length-percentage [0,∞]> | closest-side | farthest-side
定义圆或椭圆的半径。如果省略，默认为 closest-side。
  • closest-side 使用形状中心到 参考框 最近边的长度。 对于圆形，这在任何维度上都是最近边。 对于椭圆，这在半径维度上是最近边。
  • farthest-side 使用形状中心到 参考框 最远边的长度。 对于圆形，这在任何维度上都是最远边。 对于椭圆，这在半径维度上是最远边。

此外，这三个矩形形状函数为方便起见被归为一个生成：

<basic-shape-rect> = <inset()> | <rect()> | <xywh()>

3.2. 基本形状的计算值

在 <basic-shape> 函数中的值根据指定的方式计算，以下为例外情况：

  • 省略的值被包含并计算为其默认值。
  • 在 circle() 或 ellipse() 中的 <position> 值计算为从左上原点的偏移对（水平然后垂直）， 每个值作为 <length-percentage> 给出。
  • 在 <basic-shape-rect> 函数中的 <'border-radius'> 值计算为所有八个 <length-percentage> 值的扩展列表。
  • 所有 <basic-shape-rect> 函数计算为等效的 inset() 函数。

    注意： 给定 rect(t r b l)， 等效函数为 inset(t calc(100% - r) calc(100% - b) l)。 给定 xywh(x y w h)， 等效函数为 inset(y calc(x + w) calc(y + h) x)。

3.3. 基本形状的序列化

要序列化 <basic-shape> 函数， 根据其各自的语法序列化， 按照语法书写的顺序， 尽可能避免 calc() 表达式， 避免 calc() 变换， 在不改变含义的情况下省略组件， 用一个空格连接以空格分隔的标记， 并在每个序列化的逗号后加一个空格。

由于 <position> 关键字代表百分比，没有偏移的关键字会变成百分比。

circle(at left bottom)

序列化为 "circle(at 0% 100%)"

省略组件意味着某些默认值不会出现在序列化中。但是由于 <position> 始终使用 2 值或 4 值形式，默认 <position> 不会被省略。

circle(closest-side at center)

序列化为 "circle(at 50% 50%)"

使用语法顺序意味着 <position> 值总是先给出水平组件，然后是垂直组件。

circle(at bottom left)

序列化为 "circle(at 0% 100%)"

避免 calc() 表达式意味着一些可以简化为 2 值形式的 <position> 值必须以 4 值形式序列化。

circle(at right 5px bottom 10px)

序列化为 "circle(at right 5px bottom 10px)"

而不是 "circle(at calc(100% - 5px) calc(100% - 10px))"

避免 calc() 变换意味着如果指定的（或计算的） calc() 必须保持在 calc() 形式中，它将被按原样使用，而不是以不同的原点重新公式化或简化。

bottom calc(10% + 5px)

序列化为 "bottom calc(10% + 5px)"

而不是 "top calc(90% - 5px)" 或 "calc(90% - 5px)"

在必须提供省略的偏移量时，优先使用 0% 而不是零长度。

circle(at right 5px top)

序列化为 "circle(at right 5px top 0%)"

优先使用左上原点意味着某些百分比偏移量将归一化到这些原点（当可以避免 calc 时）。

circle(at right 5% top 0px)

序列化为 "circle(at 95% 0%)"

3.4. 基本形状的插值

在一个基本形状和第二个形状之间插值时， 应用以下规则。 形状函数中的值根据 计算值进行插值。 列表值尽可能地插值为 长度、百分比或 calc。 如果列表值不是这些类型 但相同 （例如在两个列表中找到相同位置的 nonzero） 那些值就会插值。

  • 两个形状必须使用相同的 参考框。
  • 如果两个形状是相同类型， 该类型是 ellipse() 或 circle()， 并且没有半径使用 closest-side 或 farthest-side 关键字， 在形状函数中的每个值之间插值。
  • 如果两个形状是 inset() 类型， 在形状函数中的每个值之间插值。
  • 如果两个形状是 polygon() 类型， 两个多边形具有相同数量的顶点， 并使用相同的 <'fill-rule'>， 在形状函数中的每个值之间插值。
  • 在所有其他情况下不指定插值。

4. 从图像生成形状

定义形状的另一种方法是指定一个源 <image>， 其 alpha 通道用于计算形状。 形状计算为封闭区域的路径或路径 这些区域中指定图像的透明度 大于 shape-image-threshold 值。 如果没有任何像素的 alpha 值 大于指定阈值， 则会产生一个空的浮动区域，不会影响换行。 如果未指定 shape-image-threshold， 则要考虑的初始值为 0.0。

图像的大小和位置 处理方式如同它是一个替换元素， 其指定的宽度和高度 与元素的使用内容框大小相同。

对于动画光栅图像格式（如 GIF）， 使用动画序列的第一帧。

一幅图像浮动在段落的左侧。 图像显示的是具有透明背景的 CSS 标志的 3D 版本。 该标志使用了 alpha 通道的阴影。

图像通过 浮动区域 定义其 shape-outside 属性。


  <p>
    <img id="CSSlogo" src="CSS-logo1s.png">
    blah blah blah blah...
  </p>

  <style>
    #CSSlogo {
      float: left;
      shape-outside: attr(src url);
      shape-image-threshold: 0.1;
    }
  </style>

shape-outside 属性重用了 img 元素的 src 属性中的 url。

完全可以显示一幅图像 并为其 浮动区域 使用不同的图像。

在下图中，alpha 通道阈值 由 CSS 标志周围的虚线表示。

然后可以通过三种方式影响段落的行开始位置：

  1. 修改图像中的 alpha 通道
  2. 改变 shape-image-threshold 属性的值
  3. 改变 shape-margin 属性的值（见 示例 8）

5. 从盒子值生成形状

形状可以通过参考 CSS 盒子模型 中的边缘来定义。 这些边缘包括 边框半径曲率 [CSS3BG]，由使用的边框半径值决定。 <shape-box> 值扩展了 <box> 值 以包含 margin-box。 其语法为： 

<shape-box> = <box> | margin-box

这些值的定义为：

margin-box 值定义了外部边距边缘所包围的形状。 该形状的角半径由相应的边框半径和边距值决定。 如果 border-radius/margin 的比例为 1 或更大， 或边距为负值或零， 则边距框角半径为 max(border-radius + margin, 0)。 如果 border-radius/margin 的比例小于 1， 并且边距为正值， 则边距框角半径为 border-radius + margin * (1 + (ratio-1)^3)。

border-box 值定义了外部边框边缘所包围的形状。 该形状遵循所有正常的边框半径形状规则 用于边框外部。

padding-box 值定义了外部填充边缘所包围的形状。 该形状遵循所有正常的边框半径形状规则 用于边框内部。

content-box 值定义了外部内容边缘所包围的形状。 此框的每个角半径为 0 或 border-radius - border-width - padding 的较大者。

给定下面的 100px 正方形，带有 10px 的填充、边框和边距， 盒子值定义了这些形状：

  • margin-box：包含所有黄色像素的形状
  • border-box：包含所有黑色像素的形状
  • padding-box：包含所有紫色像素的形状
  • content-box：包含所有蓝色像素的形状

同样的定义适用于一个更复杂的示例，具有相同的 100px 正方形，但具有以下边框、填充和边距属性：


  border-radius: 20px 20px 20px 40px;
  border-width: 30px 10px 20px 10px;
  padding: 10px 20px 10px 10px;
  margin: 20px 10px 10px 10px;

普通浮动换行与围绕由 margin-box 值定义的形状的换行之间的区别在于 margin-box 形状包括角落形状。 取一个 100px 正方形，带有 10px 填充、 边框和边距， 但边框半径为 60px。 如果你将其设置为左浮动， 内容通常会以这种方式换行：

如果你为浮动添加一个 margin-box 形状， 则内容会围绕圆角的 margin-box 进行换行。


  shape-outside: margin-box;

6. 声明形状

形状通过 shape-outside 属性声明， 可能通过 shape-margin 属性进行修改。 由 shape-outside 和 shape-margin 属性定义的形状 改变浮动元素的 浮动区域 的几何形状。

6.1. 浮动区域形状：shape-outside 属性

名称： shape-outside
值： none | [ <basic-shape> || <shape-box> ] | <image>
初始值： none
适用对象： 浮动
继承： 否
百分比： 不适用
计算值： 如为 <basic-shape> （如果提供则跟随<shape-box>）；否则计算为<image>；否则为指定的关键字
规范顺序： 按语法
动画类型： 如定义的为<basic-shape>， 否则为离散

该属性的值具有以下含义：

none
浮动区域 不受影响。
<shape-box>
如果单独指定了这些值之一，形状是基于 margin-box、border-box、padding-box 或 content-box 计算的，这些使用其各自的框 包括 border-radius 的曲率，类似于 background-clip [CSS3BG]。
<basic-shape>
该形状是基于以下一个值计算的：inset()、circle()、ellipse() 或 polygon()。如果提供了 <shape-box> ，这定义了 参考框 用于 <basic-shape> 函数。如果未提供 <shape-box> ，则 参考框 默认为 margin-box。
<image>
该形状是根据指定的 <image> 的 alpha 通道提取和计算的，如 shape-image-threshold 所定义的。

用户代理必须使用 CORS 协议，该协议由 [FETCH] 规范定义 用于 shape-outside 值中的所有 URL。 在获取时，用户代理必须使用“匿名”模式，将引用源设置为样式表的 URL 并将来源设置为包含文档的 URL。 如果因此导致网络错误，使得没有有效的后备图像， 则效果就像指定了值 none 一样。

6.2. 选择图像像素：shape-image-threshold 属性

shape-image-threshold 定义用于提取形状的 alpha 通道阈值。 值为 0.5 表示形状将包围所有超过 50% 不透明的像素。

名称： shape-image-threshold
值： <alpha-value>
初始值： 0
适用对象： 浮动元素
继承： 否
百分比： 不适用
计算值： 指定数字，限制在 [0,1] 范围内
规范顺序： 按语法
动画类型： 按计算值

该属性的值具有以下含义：

<number>
设置用于从图像提取形状的阈值。 形状由其 alpha 值大于阈值的像素定义。 阈值在 0.0（完全透明）到 1.0（完全不透明）范围外的值将被限制在此范围内。

注意： CSS Shapes 的未来版本可能会定义一个开关 以使用图像的亮度数据而不是 alpha 数据。 当这种情况发生时， shape-image-threshold 将扩展 以根据开关状态应用其阈值 到 alpha 或亮度。

6.3. 扩展形状：shape-margin 属性

shape-margin 属性为 shape-outside 添加了一个边距。 这定义了一个新的形状， 是包含所有点的最小轮廓 （在收缩包的意义上）， 这些点在从基础形状的一个点 垂直方向向外的shape-margin 距离内。 这包括基础形状的任何边缘或线段。 请注意，在 垂直线未定义的点 （例如尖点或线端） 取所有点 在以该点为中心 并具有半径为shape-margin的圆上。 此属性仅接受非负值。

名称： shape-margin
值： <length-percentage>
初始值： 0
适用对象： 浮动元素
继承： 否
百分比： 参考包含块的 内联大小
计算值： 计算的<length-percentage>值
规范顺序： 按语法
动画类型： 按计算值
<length-percentage>
将形状的边距设置为指定值。

注意： 添加 shape-margin 不允许浮动区域 扩展到浮动边距框之外。 可能需要在使用 shape-margin 的同时 应用额外的边距以避免裁剪。

一个 shape-margin 从多边形 shape-outside 创建偏移。 浅蓝色区域显示在 100x100px 的浮动中形状， 深蓝色区域显示 10px 的偏移。


  .float {
      width: 100px;
      height: 100px;
    shape-outside: polygon(10px 10px, 90px 50px, 40px 50px, 90px 90px, 10px 90px);
    shape-margin: 10px;
  }

如果 shape-margin 被添加到示例 6 中的 CSS 标志， 包裹在形状周围的行框 将进一步缩短。 如果图像的 alpha 通道 运行到图像的右边缘， 应该应用一些额外的右边距 以确保形状不被边距框裁剪。


  #CSSlogo {
    shape-margin: 35px;
    margin-right: 35px;
  }
