CSS 规范

本页面列出了由CSS WG完成的所有已完成规范和草案。如果你想了解 CSS 的发展动态，这里是你的起点。有想法？想贡献？请查看本页的“如果你想参与”部分。

规范不是使用手册。 草案写得不好不是借口，如果你发现了，请尽管提出意见。但规范确实面向特定读者。可参考 fantasai 的如何理解 CSS 规范。。J. David Eisenberg 也写了另一份有用的如何阅读 W3C 规范。或你可以阅读关于“模块”、“级别”、“快照”以及 CSS 流程的介绍。

如果你想参与贡献

请通过 GitHub 提出问题。（你需要先注册 GitHub 账号。）GitHub 上包含了CSS 规范和‘Houdini’ API的编辑草案副本。

贡献者指南详细说明了 GitHub 问题在 CSS 规范开发中的作用。

如果你针对某个具体的 CSS 模块提出问题，请在问题标题前加上对应规范代码（见“本文档状态”部分），如“[css3-flexbox] margin 计算有误”。这有助于编辑们查找和跟踪你的意见。

如果你在W3C 会员组织工作，你还可以加入CSS 工作组并参加会议。参与需承诺每周平均投入一天时间。请联系Chris Lilley和你们组织的 W3C AC 代表。工作组会议纪要是公开的，发布在CSS WG 博客上。

有多种方式可以及时了解 CSS 工作组的新发布内容。上方“最新动态”部分展示了最新草案，并且有一个 Atom 订阅。 新发布会在CSS WG 博客及其 Atom 订阅,以及工作组的 Mastodon账号同步。所有 W3C 工作组的首批草案会发布在public-review-announce 邮件列表及其 RSS 订阅。 所有 W3C 工作组的最新发布都在技术报告页面顶部，并配有

归档邮件列表 www-style@w3.org中有工作组会议的议程和纪要。你可以自行订阅。

关于测试套件

CSS 工作组投入大量时间开发CSS 测试套件 及 CSS 规范。为每个模块在发布时即提供测试套件，有助于实现 CSS 的实现提前符合规范，也更便于大家理解规范的正式内容。

推荐通过GitHub 提交问题：参见Web-platform tests 仓库。
