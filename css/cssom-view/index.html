摘要

本规范引入的 API 为作者提供了一种检查和操作文档可视视图的方式。这包括获取元素布局框的位置，通过脚本获取视口的宽度，以及滚动元素。

CSS 是一种描述结构化文档（如 HTML 和 XML）在屏幕、纸张、语音等上呈现的语言。

文档约定

一致性要求通过描述性声明和RFC 2119术语的组合来表达。关键词“必须”（MUST）、“不得”（MUST NOT）、“要求”（REQUIRED）、“应当”（SHALL）、“不应当”（SHALL NOT）、 “应该”（SHOULD）、“不应该”（SHOULD NOT）、“推荐”（RECOMMENDED）、“可以”（MAY）和“可选”（OPTIONAL）在本文档的规范部分中按照RFC 2119中的描述进行解释。 然而，为了可读性，这些词在本规范中并未全部以大写字母出现。

除明确标记为非规范性部分、示例和注释的部分外，本规范的所有文本均为规范性文本。[RFC2119]

本规范中的示例以“例如”开头，或通过class="example"与规范性文本区分开来，例如：

这是一个说明性示例。

说明性注释以“注”开头，并通过class="note"与规范性文本区分开来，例如：

注：这是一个说明性注释。

建议性内容是引起特别关注的规范性部分，通过<strong class="advisement"与其他规范性文本区分开来，例如： 用户代理必须提供可访问的替代方案。

一致性类别

本规范的一致性定义了三类一致性：

样式表
CSS样式表。
渲染器
用户代理，解释样式表的语义并渲染使用它们的文档。
创作工具
用户代理，用于编写样式表。

如果样式表中使用了此模块中定义的语法，并且符合通用CSS语法和该模块中每个特性定义的语法，则样式表符合本规范。

如果渲染器除了按照相关规范解释样式表外，还支持本规范定义的所有特性，能够正确解析它们并相应地渲染文档，则该渲染器符合本规范。然而，用户代理由于设备限制无法正确渲染文档并不意味着其不符合规范。（例如，用户代理在单色显示器上不需要渲染颜色。）

如果创作工具编写的样式表语法上符合通用CSS语法和本模块中每个特性的语法，并符合本模块中描述的样式表的一致性要求，则该创作工具符合本规范。

部分实现

为了使作者能够利用向前兼容的解析规则分配回退值，CSS渲染器必须将没有可用支持级别的任何@规则、属性、属性值、关键字和其他语法结构视为无效 （并适当忽略它们）。特别是，用户代理不得选择性地忽略单个多值属性声明中的不支持值并仅保留支持值： 如果某个值被视为无效（因为不支持的值必须被视为无效），CSS要求忽略整个声明。

不稳定和专有特性的实现

为了避免与未来稳定的CSS特性发生冲突，CSS工作组建议遵循最佳实践，以实施不稳定特性和专有扩展。

CR级别特性的实现

当规范达到候选推荐阶段时，实施者应发布任何已达到CR级别的功能的无前缀实现，只要他们能够证明其已按规范正确实现，并应避免公开该功能的带前缀变体。
