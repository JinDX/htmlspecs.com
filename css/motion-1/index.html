1. 简介

本节不具有规范性。

作者可以通过相对彼此或绝对地在自身坐标系统中定位对象，如盒子或容器。CSS 变换（Transforms）扩展了这些功能，提供了一组变换函数，使作者能够改变对象的本地坐标系统。通过 CSS 动画和 CSS 过渡，静态的定位可以在给定的时间内发生变化。这两种动画方式对于表达时间上的过渡非常强大，但并不适合描述对象随时间变化的位置信息。

本规范允许作者指定路径。对象可以沿着这条路径进行定位、过渡和动画化，并在给定的时间内完成。如果未指定动画，时间可以是静态的。

在以下示例中，飞机的示意图沿路径进行动画化。飞机在路径的每个位置上都面向路径的方向。

2. 模块交互

本规范定义了一组影响应用这些属性的元素视觉渲染的 CSS 属性。这些效果是在根据 可视化格式化模型 [CSS21] 调整盒子的大小和位置后应用的。某些 offset-path 和 offset-position 的值可能会创建堆叠上下文和包含块。

本规范中的某些 CSS 属性通过变换来操纵元素的用户坐标系统。这些变换会在通过 transform 属性指定的变换之前进行预乘，并在通过 CSS 变换模块第 1 级 [CSS-TRANSFORMS-1] 定义的属性或派生属性后进行后乘。这些变换还会在通过个别变换属性 translate、scale 和 rotate 后进行后乘，如 CSS 变换模块第 2 级 中所述。

3. 值

本规范遵循CSS 属性定义约定 [CSS21]。基本形状在 CSS 形状模块第 1 级 [CSS-SHAPES] 中定义。这些规范中未定义的值类型在 CSS 值和单位模块第 3 级 [CSS3VAL] 中定义。

除了在定义中列出的特定于属性的值外，本规范中定义的所有属性还接受 CSS 全局关键字，例如 initial 和 inherit 作为其属性值 [CSS3VAL]。为便于阅读，这里未重复说明。

4. 运动路径

运动路径模块允许指定盒子的位置，其位置是盒子的 offset-distance 沿几何路径（offset-path）的距离，该路径是相对于其包含块的坐标（offset-position）定义的。可以选择将盒子的方向指定为相对于该点路径方向的旋转（offset-rotate）。

4.1. 定义路径：offset-path 属性

名称： offset-path
值： none | ray() | path() | <url> | [ <basic-shape> || <geometry-box> ]
初始值： none
适用于： 可变换元素
继承： 否
百分比： 不适用
计算值： 按指定
规范顺序： 按语法
媒体： 视觉
可动画： 如 <angle>，<basic-shape> 或 path()

指定偏移路径，即盒子被定位在的几何路径。偏移路径可以是指定的一个或多个子路径，也可以是未样式化的基本形状的几何体。

路径可以由ray()或<basic-shape>（如<circle()>、<inset()>或<polygon()>）组成。为了允许作者指定曲线和子路径，本规范除了现有的<basic-shape>函数之外，还添加了path()。path()接受SVG路径字符串作为输入[SVG2]。作为另一种选择，作者可以通过<url>引用SVG形状元素，用作路径。

在本规范中，0度的方向（或旋转）等同于对象本地坐标系统中的负y轴方向。例如，如果对象及其祖先没有应用任何变换，0度旋转指向UA的上侧。

值的含义如下：

ray() = ray( [ <angle> && <size> && contain? ] )
<angle>

offset path 是一条从盒子位置开始并沿指定的 <angle> 前进的线段。与 CSS 渐变 类似，<angle> 值被解释为方位角，其中 0deg 指向上方，正角度表示顺时针旋转。

注意： 定义带有 offset path 的 <angle> 时，盒子可以用极坐标系统定位。极坐标系统是一个二维坐标系统，每个点在平面上的位置由距离某个固定点的距离和与固定线的夹角决定。固定点类似于笛卡尔坐标系中的原点，称为极点，而从极点发出的射线称为极轴。这个坐标系统通过极点到点之间的距离和极轴到该段的夹角来指定一个点的位置。在数学理论中，极轴通常定义为 x 轴的正方向，但为了与其他 CSS 规范（如 [CSS3VAL]）保持一致，我们将极轴视为 y 轴的正方向。

<size>

当 offset-distance 表示为百分比时，确定使用到包含框的路径长度。对于 <size> 中的其他值（除 sides 外），路径长度与 <angle> 无关。

定义为：

<size> = [ closest-side | closest-corner | farthest-side | farthest-corner | sides ]

closest-side

测量初始位置与盒子最近边之间的垂直距离。

closest-corner

测量初始位置与盒子最近角之间的距离。

farthest-side

测量初始位置与盒子最远边之间的垂直距离。

farthest-corner

测量初始位置与盒子最远角之间的距离。

sides

测量初始位置与射线与盒子相交处的距离。如果初始位置不在盒子内，则距离为 0。

注意： 当初始位置位于包含块的边缘之一时，closest-side 选择初始位置所在的边，因此用于百分比 offset-distance 值的路径长度为 0。

注意： 当使用 closest-side 或 farthest-side 时，如果初始位置在盒子外，则认为这些边无限延伸。

contain

用于的 offset-distance 值被限制为盒子完全包含在路径内。

如果没有任何 offset-distance 可以使盒子被路径包围，则路径大小将被最小化地增加，以便存在这样的 offset-distance。

这里有一些示例。第一个示例显示了一些盒子的部分超出了 offset path。

        transform-style: preserve-3d;
        width: 200px;
        height: 200px;
    }

        width: 50px;
        height: 50px;

        offset-distance: 100%;

    }

    }
    #blueBox {
        background-color: blue;

    }

<basic-shape> || <geometry-box>

偏移路径 是由CSS Shapes中定义的基本形状 [CSS-SHAPES]。

offset-position 在具有显式中心位置的圆形和椭圆形基本形状中被忽略，以及其他类型的基本形状中也是如此。如果一个圆形或椭圆形基本形状没有显式中心位置，形状将居中于路径的 初始位置，如 offset-position 中所述。

路径的 初始位置 和 初始方向 描述了 offset-distance 从 0% 开始增加时盒子沿路径的位置。

基本形状的 初始位置 定义如下：

<circle()>
<ellipse()>

初始位置 由圆形/椭圆形的水平切线与顶部垂直位置的交点定义。

<inset()>

初始位置 是顶部水平线的左端，紧邻任何 border-radius 圆弧的右边。

<polygon()>

初始位置 由多边形的第一个坐标对定义。

初始方向 由连接的向量定义。

该向量连接 初始位置 和下一个不等于 初始位置 的坐标对。

结合 <geometry-box> 和 <basic-shape> 为 <basic-shape> 提供参考框。如果未指定参考框，将使用 border-box 作为参考框。

如果提供了 <geometry-box> 而没有 <basic-shape>，初始位置 位于顶部水平线的左端，紧邻任何 border-radius 弧的右侧，初始方向 向右。

除了长度不为零的多边形之外，初始方向 是 90 度（即向右）。

注意：当 offset-rotate 设置为 auto 时，此设置会提供 0deg 的旋转。

path() = path(<string>)

<string> 表示一个SVG路径数据字符串。该路径数据字符串必须符合SVG 1.1的语法和解析规则 [SVG11]。初始位置 由路径字符串中的第一个“移动到”参数定义。有关 初始方向 的信息，请参见SVG 1.1 [SVG11]。

<url>

引用一个SVG 形状元素，并使用其几何图形作为 偏移路径。有关 初始位置 和 初始方向 的更多信息，请参见SVG 1.1 [SVG11]。

none

不会创建 偏移路径。当 offset-path 为 none 时，offset-distance 和 offset-rotate 无效。

计算值为非 none 时，会根据通常的 堆叠上下文 和 包含块 创建 转换，参考 [CSS21]。

无法下载、不是SVG 形状元素 的引用或不存在的引用等同于 path("m 0 0")。

注意： 这是一个长度为零的路径，其 方向 与正x轴对齐。

请参阅 “计算路径变换” 部分，了解如何使用 偏移路径 计算变换。

对于没有关联CSS布局框的SVG 形状元素，使用的值为 内容框、内边距框、边框框 和 外边距框 的 填充框。

对于与CSS布局框关联的 形状元素，使用的值为 填充框、描边框 和 视图框 的 边框框。

4.2. 路径上的位置：offset-distance 属性

名称: offset-distance
值: <length-percentage>
初始值: 0
适用于: 可转换元素
继承: 否
百分比: 参考总路径长度
计算值: 对于 <length> 是绝对值，否则是百分比。
规范顺序: 根据语法
媒体: 视觉
可动画: 是

指定盒子沿 偏移路径 的距离位置。

<length-percentage>

指定从 初始位置 到盒子 偏移路径 上 锚点 位置的距离。

百分比相对于 偏移路径 的长度，即从 初始位置 到 偏移路径 结束位置的距离。

4.2.1. 沿路径计算计算的距离

根据 偏移路径 的性质，沿着路径处理距离的方式有所不同：

  • 引用具有包含的 <angle> 偏移路径 是未闭合区间。

  • 引用不包含的 <angle> 偏移路径 是无界射线。

  • 所有基本CSS形状都是闭合回路。

  • 偏移路径（包括对SVG路径的引用）仅在路径列表的最后一个命令是闭合路径命令（"z"或"Z"）时为闭合回路，否则为未闭合区间。

  • 引用SVG圆形、椭圆形、图像、多边形和矩形是闭合回路。

  • 引用SVG线条和折线是未闭合区间。

为了确定给定 偏移路径 和 偏移距离 的 使用的偏移距离：

  1. 让 总长度 等于具有所有子路径的 偏移路径 的总长度。

  2. 将 偏移距离 转换为像素，100% 被转换为 总长度。

  3. 如果 偏移路径 是无界射线：

    让 使用的偏移距离 等于 偏移距离。

    否则，如果 偏移路径 是具有包含的 <angle> 路径：

    让 使用的偏移距离 等于 偏移距离，并进行限制以确保盒子完全位于路径内。

    如果 偏移路径 是任何其他未闭合区间：

    让 使用的偏移距离 等于 偏移距离，并限制在0和路径总长度之间。

    否则，偏移路径 是一个闭合回路：

    让 使用的偏移距离 等于 偏移距离 模除路径的总长度。如果路径的总长度为0，使用的偏移距离 也为0。

4.3. 定义路径的起点：offset-position属性

名称: offset-position
值: auto | <position>
初始值: auto
适用对象: 可转换元素
继承: no
百分比: 参照包含块的大小
计算值: 对于<length>，为绝对值，否则为百分比。
规范顺序: 按语法
媒体: 视觉
可动画: 如position

指定offset-path的起始位置。如果position属性的值为static，则忽略offset-position。

值的定义如下：

auto

表示起始位置是由position属性指定的位置。

注意： 这类似于绝对定位，但offset-position不会阻止盒子影响后续兄弟元素的布局。

计算值如果不是auto，将创建一个堆叠上下文[CSS21]和包含块，按常规用于变换。

4.4. 定义锚点：offset-anchor属性

名称: offset-anchor
值: auto | <position>
初始值: auto
适用于: 可转换元素
是否继承: no
百分比: 相对于盒子的宽度和高度
计算值: 对于<length>的绝对值，否则为百分比。
语法顺序: 根据语法
媒体: 视觉
可动画: 与<position>一样

定义沿着offset-path定位的盒子的锚点。锚点指定了沿着offset-path移动的盒子的点。

auto

<position>
<percentage>

<length>

长度值表示从盒子内容区域左上角的偏移。

4.5. 旋转点：offset-rotate 属性

名称： offset-rotate
值： [ auto | reverse ] || <angle>
初始值： auto
适用于： 可变换元素
是否继承： 否
百分比： 不适用
计算值： 计算的 <angle> 值，可选地加上 auto
语法顺序： 按语法
媒体： 视觉
是否可动画： 是

定义了盒子沿着 offset path 时的方向。

值的含义如下：

auto

表示对象随着 offset-distance 动画（如果有）根据 offset path 的方向向量相对于正 x 轴的角度旋转。如果与 <angle> 一起指定，则 <angle> 的计算值会加到 auto 的计算值上。

注意：对于射线路径，auto 的旋转比射线的方位角少 90 度 <angle>。

reverse

表示对象随着 offset-distance 动画（如果有）根据 offset path 的方向向量相对于正 x 轴的角度加 180 度旋转。如果与 <angle> 一起指定，则 <angle> 的计算值会加到 reverse 的计算值上。

注意：这与指定 auto 180deg 是相同的。

<angle>

表示对象应用了一个以指定旋转角度为常量的顺时针旋转变换。如果与 auto 或 reverse 一起指定，请参见相关定义。

当 offset path 是一个零长度路径时，offset-rotate 的值为 0 度，方向为正 x 轴。

如果 offset path 由多个线段组成，那么在段与段的连接处方向与上一段的方向相同。

注意：此处描述的旋转不会覆盖或替代由 transform 属性定义的任何旋转。

以下示例使用飞机形状。形状中央的红点表示形状的 anchor point。当未设置 offset 属性时，形状不会沿路径平移或旋转。

当形状的 anchor point 放置在路径的不同位置，且 offset-rotate 为 0deg 时，形状不会旋转。

如果将 offset-rotate 属性设置为 auto，并将形状的 anchor point 放置在路径的不同位置，形状将根据当前位置的梯度进行旋转，面对该位置路径的方向。

在此示例中，offset-rotate 属性设置为 reverse。飞机在路径的每个位置都朝向路径的相反方向。

最后一个示例将 offset-rotate 属性设置为 -45deg。形状逆时针旋转 45 度一次，并在路径的每个位置保持旋转。

此示例展示了 auto 或 reverse 与 <angle> 结合使用时的效果。<angle> 的计算值会加到 auto 或 reverse 的计算值上。

4.5.1. 路径变换的计算

  1. 为盒子的局部坐标系创建一个补充变换矩阵 T1。

  2. 根据 初始位置 找到由 offset-position 指定的 offset path 的 T2。

  3. 设 P 为沿着 使用的 offset 距离 的 offset path 上的点。

  4. 找到盒子的平移，使其 anchor point 位于 P，并将其应用到 T2。

  5. 将 T2 按 offset-rotate 指定的旋转进行后乘。

  6. 将 T2 后乘到 T1。

  7. 将 T1 后乘到盒子的局部坐标系。

变更

本节非规范性。

自 2015年4月9日首次公开工作草案以来，进行了以下变更。

  • 将 motion-path 重命名为 offset-path，以便与 polar-angle 进行整合。

    • 添加了 ray()，用于定义 offset path 为一个通过 <angle> 指定方向的线段。

    • 为 ray() 添加了 <size> 和 contain 值。

  • 将 motion-offset 重命名为 offset-distance，以便与 polar-distance 进行整合。

  • 将 motion-rotation 重命名为 offset-rotate。

  • 添加了 offset-position，用于通过合并 polar-origin 来指定路径的 初始位置。

  • 添加了 offset-anchor，用于通过合并 polar-anchor 来指定元素的原点。

  • 将简写属性 motion 重命名为 offset。

  • 使 offset-rotate 能够通过 auto 或 reverse，与 <angle> 结合来指定旋转变换。
