摘要

此注册被录入到 [webcodecs-codec-registry] 中。 它描述了线性 PCM 的：(1) 完全限定的 编解码器字符串、 (2) 特定于编解码器的 EncodedAudioChunk [[internal data]] 字节，(3) AudioDecoderConfig.description， 以及 (4) EncodedAudioChunk [[type]] 的值。

线性 PCM 是 WebCodecs 中使用的原始音频格式， 不需要解码。注册线性 PCM 编解码器的动机是启用 PCM 内容的 透传“解码”，作为 WebCodecs 应用的一种架构简化。

此注册无意包含关于某种 编解码器格式是否受知识产权权利主张约束的任何信息。如果实现者和 作者打算实现或使用某种特定的编解码器格式，建议他们就此事项 寻求适当的法律顾问意见。

此注册是非规范性的。

1. 完全限定的编解码器字符串

此 编解码器字符串以前缀 "pcm-" 开头，后跟表示 采样格式的后缀。编解码器字符串及其关联 格式的完整列表如下。

  • pcm-u8，使用格式 u8

  • pcm-s16，使用格式 s16

  • pcm-s24，使用三个 byte（24-bit）samples，并采用 interleaved channel arrangement。

  • pcm-s32，使用格式 s32

  • pcm-f32，使用格式 f32

注： [WEBCODECS] 未定义 24-bit AudioSampleFormat。 24-bit samples 允许存在于 EncodedAudioChunk 中， 但这类 samples 将会 在 AudioData 对象中被“解码”为 s32 或 f32。 请 参阅 WebCodecs § 9.3.2 音频样本的幅度以了解更多细节。

2. EncodedAudioChunk 数据

线性脉冲编码调制（linear PCM）描述了一种格式，其中音频 值以固定间隔采样，并且两个连续值之间的量化级 线性均匀。

EncodedAudioChunk [[internal data]] 预期为 任意长度的字节序列，其中每 N 位出现一个 sample， 其中 N 由 编解码器 字符串定义。对于多声道 PCM，来自不同声道的 samples 会被interleaved。

3. AudioDecoderConfig description

description 预期从 AudioDecoderConfig 中省略。

4. EncodedAudioChunk type

对于包含 线性 PCM 的 EncodedAudioChunk， 其 [[type]] 始终是 “key”。
