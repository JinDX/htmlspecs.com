📄 HTTP 文档

HTTP 由这些 IETF RFCs 和 IANA 注册表 定义。

核心规范

HTTP 协议的“核心”语义由以下内容定义：

  • RFC 9110: HTTP Semantics HTTP 语义
  • RFC 9111: HTTP Caching HTTP 缓存

这些语义以三种方式“在传输中”表达：

  • RFC 9112: HTTP/1.1 HTTP/1.1
  • RFC 9113: HTTP/2 HTTP/2
  • RFC 9114: HTTP/3 HTTP/3

较新的 HTTP 版本提供字段压缩：

  • RFC 7541: HPACK Header Compression for HTTP/2 HTTP/2 的 HPACK 头部压缩
  • RFC 9204: QPACK Field Compression for HTTP/3 HTTP/3 的 QPACK 字段压缩

这些 RFC 综合废止了所有此前定义 HTTP 的 RFC，包括 RFC 1945, RFC 2068, RFC 2616, RFC 2617, RFC 7230-5, 和 RFC 7540.

另一个相关的是结构化字段值规范（Structured Field Values），该规范被越来越多的首部和尾部使用：

  • RFC 9651: Structured Field Values for HTTP (test suite) 结构化字段值

构建在 HTTP 之上的协议应遵循此处记录的最佳现行实践：

  • RFC 9205: Building Protocols with HTTP 使用 HTTP 构建协议

扩展注册表

有大量的 HTTP 扩展，包括方法、状态码和首部。重要的是要注意，只要它们被列在相应的注册表中，它们就是 HTTP 的“组成部分”：

  • HTTP 方法注册表 - 更多信息
  • HTTP 状态码注册表 - 更多信息
  • HTTP 字段名注册表 - 更多信息
  • HTTP/2 参数 - 帧类型、设置、错误代码
  • HTTP/3 参数 - 帧类型、设置、错误代码

单个首部有其各自的注册表来规范允许的取值，包括：

  • HTTP 认证方案注册表 - 更多信息
  • HTTP 缓存指令注册表 - 更多信息
  • HTTP 参数 - Content-codings、transfer-codings、Forwarded、Prefer 和与 Range 相关的首部
