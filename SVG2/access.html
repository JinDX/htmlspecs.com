C.1. SVG 无障碍功能

本附录强调了 SVG 的无障碍功能以及支持 SVG 的相关规范和相关的 W3C 网站内容无障碍指南 (WCAG) 2.0 [WCAG2] 要求。

[WCAG2] 创建可以以不同方式呈现的内容（例如更简单的布局），而不会失去信息或结构：信息和关系。
  • 支持 aria-labelledby 的标签关系。
  • 支持 aria-describedby 的描述关系。
  • 支持 aria-owns 的结构关系。
  • 支持 aria-controls 的控制关系，表示一个元素控制另一个元素的内容和/或行为。
  • 为定义的分组提供 ‘g’。
  • 支持 WAI-ARIA 组 和 单选组 角色。
[WCAG2] 基于时间的媒体：为基于时间的媒体提供替代方案。
  • 支持 HTML ‘track’ 元素，以提供基于时间的媒体的替代方案
[WCAG2] 可键盘访问：确保所有功能可通过键盘使用。
  • 焦点支持 和 ‘tabindex’ 属性的顺序焦点导航与 HTML 对齐。
  • 键盘事件属性支持以及鼠标事件支持 为脚本作者提供帮助。
  • 键盘事件 为脚本作者提供支持。
  • 脚本支持在 DOM 中为每个 SVG 元素 设置焦点。
  • 脚本支持获取 DOM 中每个 SVG 元素 的 ‘tabindex’ 属性。
  • 脚本支持在 文档接口 中的 HTML activeElement 属性。
[WCAG2] 可导航：提供帮助用户导航、查找内容和确定其位置的方法。
  • 为了使用户能够跳过内容块，SVG 支持 WAI-ARIA 地标 角色。
  • 支持 ‘title’ 提供页面标题。
  • 支持 ‘tabindex’ 提供顺序焦点导航顺序。
  • 支持 ‘a’ 元素，使作者能够提供链接目的。
  • 通过使用 标题 角色及 aria-level、aria-labelledby 和 aria-label 支持标题和标签。
  • 通过在选中的元素上呈现可见焦点来支持可见焦点。
[WCAG2] 可读性：使文本内容可读且易于理解。
  • 通过 ‘lang’ 属性支持页面及其部分的语言。
[WCAG2] 兼容性：最大化与当前和未来用户代理的兼容性，包括辅助技术。
  • 通过使用 WAI-ARIA 属性 和 ‘title’ 元素支持名称、角色和值。
WAI-ARIA 支持和文本替代
  • WAI-ARIA 属性
  • 描述和标题文本元素替代方案
